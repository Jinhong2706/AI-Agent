# 进化闭环 · 经验提取→知识沉淀→智能检索→执行验证→自动改进

## 闭环五步

```
① 经验提取    ② 知识存储    ③ 智能检索    ④ 执行验证    ⑤ 自动改进
Agent会话复盘 → 知识文件 → 按需检索 → 测试验证 → 更新技能
     ↑                                                    │
     └──────────────── 反馈驱动 ──────────────────────────┘
```

---

## ① 经验提取

**数据源：**
| 来源 | 提取什么 | 方法 |
|------|---------|------|
| Agent会话历史 | 成功案例、优秀回答、用户采纳的方案 | sessions_history |
| 用户纠正/不满 | 错误模式、改进点 | lcm_grep |
| memory/日志 | 决策、教训、新模式 | read memory/ |
| knowledge/ | 已有知识（避免重复） | read knowledge/ |

**提取流程：**
1. 扫描目标Agent最近N次会话（默认10次）
2. 用 `lcm_grep` 搜索关键词（领域名+错误+改进）
3. 提取：新模式、新知识、用户采纳的方案、错误教训
4. 去重：与现有 knowledge/ 和 references/ 对比

---

## ② 知识存储

**存储层级：**
```
skills/<name>/references/
├── knowledge-base.md    ← Agent输出的领域知识（从会话提取）
├── best-practices.md    ← 成功案例+最佳实践（经验沉淀）
├── common-errors.md     ← 常见错误+避免方法（教训固化）
└── user-feedback.md     ← 用户反馈+改进记录（迭代追踪）
```

**存储原则：**
- 每条知识必须有来源标注（日期+Agent+会话ID）
- 去重后再写入，不重复存储
- 知识文件不超过200行，超了拆分

---

## ③ 智能检索

**检索时机：**
- Phase 2（领域深挖）时：检索现有知识，避免重复调用Agent
- Phase 4（内容生成）时：检索最佳实践和常见错误
- 技能合并时：检索多个技能的知识库

**检索方法：**
- `lcm_grep`：在历史会话中搜索相关片段
- `memory_search`：搜索记忆中的相关经验
- `read` references/：直接加载已有知识文件

---

## ④ 执行验证

**验证方法：**
| 验证类型 | 方法 | 通过标准 |
|---------|------|---------|
| 功能验证 | spawn子Agent用测试用例运行 | 输出格式正确、内容完整 |
| Token验证 | 数SKILL.md行数+description字数 | ≤100行 + ≤120字 |
| 一致性验证 | 对比references/与SKILL.md | 无矛盾、无重复 |
| 用户验证 | 交付后观察用户反馈 | 用户未纠正=通过 |

---

## ⑤ 自动改进

**改进触发条件：**
- 验证失败 → 立即修复
- 用户纠正 → 24小时内更新
- 新知识积累≥5条 → 触发技能版本更新
- 周期性（每月）→ 全面审查

**改进动作：**
1. 更新 SKILL.md / references/
2. 更新 description（市场竞争力和可信度）
3. 更新版本号（记录在 knowledge-base.md 头部）
4. 写入 `memory/evolution-log.md`

---

## 进化记录格式

```markdown
## [日期] [技能名] v[X.X]
- **触发**：用户反馈/自动积累/周期审查
- **变更**：具体改了什么
- **验证**：通过/未通过
- **来源会话**：agent:xxx:session-xxx
```
