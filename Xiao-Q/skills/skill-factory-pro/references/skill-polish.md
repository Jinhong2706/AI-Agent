# 技能包装 · 命名与描述优化

## 命名原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 短而精准 | ≤3个单词，一眼看懂 | ❌ "tcm-constitution-analyzer" → ✅ "sai-huatuo" |
| 有辨识度 | 与竞品区分，有IP感 | ✅ "赛华佗"比"中医助手"更有记忆点 |
| 含功能暗示 | 名字暗示核心能力 | ✅ "destiny-master"（玄学大师）✅ "skill-factory"（技能工厂）|
| 避免通用词 | 不用helper/tool/assistant | ❌ "pdf-helper" → ✅ "pdf" |

## description 优化规范

### 结构公式

```
[一句话定位] · [差异化优势]。触发：[3-5个用户自然语言场景]。
```

### 字数控制

- **总字数 ≤ 120字**（强制）
- 前80字：功能+差异化
- 后40字：触发场景（用户会说什么）

### 优化技巧

| 技巧 | 说明 |
|------|------|
| 数字量化 | "9种体质"比"多种体质"更可信 |
| 权威背书 | "北京中医药大学"比"中医专家"更有说服力 |
| 场景具象 | "久坐+熬夜"比"现代生活方式"更精准（示例，实际根据目标用户调整） |
| 结果导向 | "给出具体行动建议"比"提供分析"更吸引人 |
| 排除竞品 | "基于生辰八字+五行喜忌"比"命理分析"更专业 |
