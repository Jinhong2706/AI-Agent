# GEO品牌知识库一键生成器

基于北京九一数榜GEO操盘手体系，将企业信息转化为AI搜索引擎偏好的结构化知识库。通过10个GEO优化文件夹的标准化架构，覆盖"身份→产品→信任→流量→转化"五层AI检索链路。

## 核心能力

1. **参数智能提取**：从用户描述中提取企业名称和行业，信息不足时最多追问一轮
2. **知识库自动创建**：调用IMA OpenAPI创建知识库，支持共享/个人/订阅三种类型
3. **10文件夹批量创建**：并行创建10个GEO优化文件夹，每个文件夹承载特定AI检索作用
4. **结构化汇报**：输出包含GEO优化逻辑分层的创建结果报告

## 工作流

### Phase 1：参数收集

从用户描述中提取以下参数：

| 参数 | 必需性 | 说明 | 默认值 |
|------|--------|------|--------|
| 企业名称 | 必需 | 用于知识库命名和推荐问题 | - |
| 行业 | 必需 | 用于简介和推荐问题个性化 | - |
| kb_id | 可选 | 已有知识库ID，提供则跳过创建 | 无 |
| 知识库类型 | 可选 | 个人(1001)/共享(1002)/订阅(1003) | 共享(1002) |
| 自定义文件夹 | 可选 | 是否需增减文件夹 | 使用标准10文件夹 |

**智能提取规则**：
- 企业名称：从用户描述中提取品牌/公司名称
- 行业：根据企业名称合理推断，能确定就不追问
- 信息不足时：最多追问一轮，用结构化问题一次性补齐全部缺失信息
- 不允许输出谜语或让用户猜测

> 如需自定义文件夹方案，修改 `references/folder-spec.md` 后再执行 Phase 3。

### Phase 2：创建知识库

**若用户提供了已有 kb_id**：跳过此步，直接进入 Phase 3。

**操作流程**：

1. 先加载 `ima-skills` skill，按其中的 Setup 步骤验证凭证可用性
2. 调用 IMA OpenAPI 创建知识库：

```bash
ima_api "openapi/wiki/v1/create_knowledge_base" '{
  "name": "{企业名称}知识库",
  "description": "{企业名称}GEO优化知识库，结构化存储企业全维度数据，提升AI搜索收录与品牌曝光",
  "type": {知识库类型，默认1002},
  "recommended_questions": ["{企业名称}有哪些核心产品？", "{企业名称}的品牌实力如何？", "如何联系{企业名称}？"]
}'
```

3. 记录返回的 `kb_id` 和 `kb_name`
4. 推荐问题必须根据企业行业个性化生成，不可用通用模板

**错误处理**：
- 凭证缺失 → 引导用户到 https://ima.qq.com/agent-interface 获取 Client ID 和 API Key
- API 返回错误 → 检查参数合法性，重试一次；重试仍失败则向用户汇报具体错误

### Phase 3：创建10个GEO优化文件夹

**前置校验**：
- 确认 `kb_id` 与目标知识库一致
- 二次校验 `kb_name` 确保操作对象正确
- 所有写入不可逆（API不支持删除），确认企业名称无误后再执行

**操作流程**：

1. 读取 `references/folder-spec.md` 获取完整的文件夹定义
2. 逐个调用 `create_folder` API 创建文件夹（可并行发送以提升效率）：

```bash
ima_api "openapi/wiki/v1/create_folder" '{
  "knowledge_base_id": "{kb_id}",
  "kb_name": "{企业名称}知识库",
  "name": "{文件夹名}"
}'
```

3. 记录每个文件夹的 `folder_id`

**10个标准文件夹**：见 `references/folder-spec.md`，此处列出名称与作用速览：

| # | 文件夹名 | GEO作用 |
|---|---------|---------|
| 1 | 企业介绍 | AI回答"XX是什么"的核心信源 |
| 2 | 产品体系 | AI回答"XX有什么产品"时直接引用 |
| 3 | 客户案例 | 增强可信度，AI偏好有案例的内容 |
| 4 | 荣誉资质 | 权威信号，AI优先展示有资质背书的品牌 |
| 5 | 品牌故事 | 长尾关键词覆盖，AI偏好叙事性内容 |
| 6 | 工艺与技术 | 差异化内容，建立专业壁垒 |
| 7 | 媒体报道 | 外部权威信号，AI对被媒体引用的内容权重更高 |
| 8 | 行业洞察 | 建立行业话语权，AI回答行业问题时引用 |
| 9 | 常见问题FAQ | 直接匹配用户提问，AI搜索精准回答 |
| 10 | 联系方式与门店 | AI回答"在哪买""怎么联系"时直接引用 |

### Phase 4：汇报结果

向用户展示创建结果，使用以下结构化模板：

```
## ✅ {企业名称}知识库 — 已创建完毕

**知识库名称**：{企业名称}知识库
**知识库类型**：{类型描述}
**知识库ID**：`{kb_id}`

| # | 文件夹名 | 文件夹ID | GEO作用 |
|---|---------|----------|---------|
| 1 | 企业介绍 | {folder_id} | AI回答"XX是什么"的核心信源 |
| 2 | 产品体系 | {folder_id} | AI回答"XX有什么产品"时直接引用 |
| ... | ... | ... | ... |

### 🎯 GEO优化逻辑

- **身份层**（企业介绍 + 品牌故事）→ 解决"你是谁"
- **产品层**（产品体系 + 工艺与技术）→ 解决"你有什么、凭什么好"
- **信任层**（客户案例 + 荣誉资质 + 媒体报道）→ 解决"凭什么信你"
- **流量层**（行业洞察 + FAQ）→ 解决"怎么被搜到"
- **转化层**（联系方式与门店）→ 解决"怎么找到你"
```

## 注意事项

- **不可逆操作**：所有写入不可逆，创建前确认企业名称无误
- **kb_id 校验**：操作前二次确认 kb_id 与目标知识库一致
- **推荐问题个性化**：根据企业行业生成3个推荐问题，不用通用模板
- **文件夹可调整**：10个标准文件夹是默认方案，用户可要求增减
- **并行优化**：Phase 3 的10个 `create_folder` 调用互不依赖，可并行发送
- **Windows编码警告**：Windows环境下 curl 发送中文 content 会损坏编码（GBK→UTF-8 转码导致 replacement character U+FFFD）。**必须使用 Node.js 发送包含中文的 API 请求**，禁止使用 curl `-d` 直接拼接中文 JSON。
- **MCP ID 与 OpenAPI ID 不同**：IMA MCP 工具使用数字型 ID（如 `7466138501908737`），而 OpenAPI 使用加密字符串型 ID（如 `R_Zw7_PTKO2UK_mQl3ptd0S-tsPYowj6ksJmcSUAgzI=`）。当用户提供 MCP 端的 kb_id 时，必须先通过 `search_knowledge_base` OpenAPI 接口按名称搜索获取正确的 OpenAPI ID，才能调用 `create_folder` 等写入接口。
