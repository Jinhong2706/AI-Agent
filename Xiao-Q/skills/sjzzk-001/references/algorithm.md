# 石家庄中考志愿填报助手 — 模型计算方法

## 一、整体架构

```
输入（分数） → 位次映射 → 概率计算 → 结果输出
                   ↑            ↑
              一分一档表     学校分数线 + 波动参数 + 反向查询
```

四步流程来自产品架构骨架（输入→生成→校验→交付），在志愿填报场景下具体化为：

| 步骤 | 内容 | 数据来源 |
|------|------|---------|
| 输入 | 用户输入预估分数（支持2023/2024/2025三年切换，650/800双分制） | 用户 |
| 位次映射 | 分数→全市排名 | 官方一分一档表（三年完整数据） |
| 概率计算 | 位次→录取概率 | 46所学校统招线（全真实数据）+ 逐校σ模型 + 分配生逻辑 |
| 结果输出 | 概率 + 风险等级 + 反向查询 + 组合推荐 | 冲/稳/保三档 + 智能组合 |

## 二、位次映射

### 2.1 一分一档表

将用户输入的分数转化为全市排名，是概率计算的前提。

| 年度 | 总分 | 考生数 | 官方发布范围 | 工具覆盖范围 | 数据质量 |
|------|------|--------|-------------|-------------|---------|
| 2023 | 650 | ~124,921 | 400-635（主城区）+ 500-628（17县区） | 300-635（含合并） | 主城区236点位 + 17县区129点位，均为官方数据 |
| 2024 | 650 | 78,402 | 500-646（官方完整发布） | 300-646（含低分推算扩展） | 高分段（500+）为官方真实数据；300-499为按2025年分布比例推算，仅用于低分场景兜底 |
| 2025 | 800 | 97,628 | 300-780（官方完整发布） | 300-780 | 全部为官方真实数据，481个数据点 |

### 2.2 位次查询算法

```
给定：分数 N
1. 在 SCORE_TABLE[N] 中查对应的累计人数 C
2. 位次 = C（即全市排名第 C 名）
3. 如果 N 在两个已知点位之间，线性插值计算
4. 如果 N 超出已知范围：上限用 TOTAL_CANDIDATES，下限用 0
```

### 2.3 2024年低分段扩展

由于2024年官方一分一档表发布范围为500-646分（低于500分不公布），为让工具覆盖低分场景（如部分民办校录取线可能在500分以下），对300-499分段做了扩展推算：

```
1. 计算2025年300-499分段占总考生的比例：P_low_2025
2. 计算2025年500-646分段占总考生的比例：P_high_2025
3. 推算2024年300-499分段人数 = 2024总考生 × (P_low_2025 / P_high_2025) × P_high_2024
4. 按2025年300-499分段的位次分布比例，等比分配到2024年
```

**说明**：此扩展段为推算数据，非官方发布。推算假设两年低分段分布比例一致，实际可能有偏差。但低分段对核心决策（中高分段学校选择）影响有限，主要用于工具边界情况的兜底处理。

## 三、概率计算核心模型

### 3.1 基本假设

录取概率 = 1 - 正态累积分布函数(Normal CDF)

核心思想：将"分数到录取线的距离"标准化为 Z-score，通过正态分布查表得到概率。

### 3.2 标准化

```
Z = (学生分数 - 学校统招线) / σ_combined

其中：
  Z              = 标准化的分数差距（衡量"超过录取线几个标准差"）
  σ_combined     = 合成标准差 = √(σ_估分² + σ_school²)
  σ_估分          = 估分误差标准差（固定为 12 分）
  σ_school       = 该校历史统招线年际波动标准差（逐校独立，从三年数据校准）
```

### 3.3 逐校 σ_school 校准（v2.0）

v2.0 的核心升级：每一所学校使用独立的 σ_school，而非统一的 10 分。

```
校准方法：
1. 收集该校三年（2023/2024/2025）统招线数据
2. 跨年分数标准化：2023 和 2024 为 650 分制，2025 为 800 分制
   - 标准化到目标年份量尺：分数 / 当年满分 × 目标年份满分
3. 计算标准化后分数的标准差（样本标准差，ddof=1）
4. σ_school = max(标准差, 3.0)  -- 保底 3 分，防止过度稳定或单年数据
5. σ_combined = √(12² + σ_school²)
```

**46校 σ_school 范围（2023-2025 三年数据校准）**：

| 类型 | σ_school 范围 | 代表学校 | 含义 |
|------|-------------|---------|------|
| 极度稳定 | 3.0 ~ 5.0 | 二中(4.1)、一中(3.0)、二十七中(4.6) | 顶尖公办，统招线几乎不波动 |
| 较稳定 | 5.0 ~ 10.0 | 二十四中(6.7)、九中(8.0)、二中西(5.6) | 中上公办，小幅年际波动 |
| 中等波动 | 10.0 ~ 20.0 | 十七中(13.3)、二十二中(14.0)、四中(14.1) | 中段学校，受报考热度影响 |
| 较大波动 | 20.0 ~ 40.0 | 二十三中(26.5)、二十八中(21.9)、六中(36.1) | 边缘校或特色校，分数线跳跃大 |
| 剧烈波动 | 40.0+ | 河北联邦外国语学校(71.5) | 民办校，分数线极度不稳定 |

**与旧版对比**：

| 学校 | 旧版 σ_combined | 新版 σ_combined | 概率影响（601分/2024为例） |
|------|----------------|----------------|--------------------------|
| 二中 | 15.62 | 12.7 | 非常稳定，概率上调 |
| 二十四中 | 15.62 | 13.7 | 稳定，概率上调 |
| 四中 | 15.62 | 18.4 | 波动大，概率下调 17pp |
| 十二中 | 15.62 | 18.9 | 波动大，概率下调 16pp |
| 河北联邦外国语 | 15.62 | 72.5 | 极度不确定，概率大幅修正 |

### 3.4 概率计算

```
录取概率 = 1 - Φ(Z)
        = 1 - P(X ≤ Z)，其中 X ~ N(0, 1)

即：学生分数超过学校统招线的概率。

Z = (学生分数 - 学校统招线) / √(12² + σ_school²)
```

**实现**：Abramowitz & Stegun 多项式逼近 erf，然后 Φ(Z) = 0.5 × (1 + erf(Z/√2))。

完整 JavaScript 实现内嵌于 `webapp/index.html`，纯前端计算，零外部依赖。

### 3.5 风险分级（v2.1 更新）

| 概率区间 | 风险等级 | 颜色 | 建议 |
|---------|---------|------|------|
| ≥ 80% | 保 | 绿 | 保底首选 |
| 50%-80% | 稳 | 黄 | 主力志愿 |
| < 50% | 冲 | 红 | 不能作为保底志愿 |

## 四、反向查询引擎（v2.1 新增）

### 4.1 核心公式

反向查询将"正向引擎"反转：输入目标学校和期望概率 → 输出最低所需分数。

```
所需分数 = 统招线 + σ_combined × Φ⁻¹(1 - 期望概率)

其中 Φ⁻¹ 是逆正态累积分布函数（probit / quantile function）
```

### 4.2 normSInv 实现

使用有理逼近计算逆误差函数（inverse erf），精度 ±0.0004：

```javascript
function normSInv(p) {
  if (p <= 0) return -Infinity;
  if (p >= 1) return Infinity;
  if (p < 0.02425) {
    // 下尾：有理逼近
    var q = Math.sqrt(-2 * Math.log(p));
    return ((((c[0]*q + c[1])*q + c[2])*q + c[3]) / (((d[0]*q + d[1])*q + d[2])*q + 1)) - q;
  } else if (p > 0.97575) {
    // 上尾：对称变换
    return -normSInv(1 - p);
  } else {
    // 中心区域
    var q = p - 0.5;
    var r = q * q;
    return ((((a[0]*r + a[1])*r + a[2])*r + a[3])*q / ((((b[0]*r + b[1])*r + b[2])*r + b[3])*r + 1));
  }
}
```

### 4.3 界面交互

```
用户操作：
1. 从下拉菜单选择目标学校
2. 拖动概率滑块（10%-95%）
3. 即时显示：所需最低分数 + 对应的全市位次

验证公式：用正向引擎（normCDF）对反向结果验算，概率偏差 ≤ 0.5%
```

| 场景 | 输入 | 输出示例 |
|------|------|---------|
| 目标导向 | 二十四中 + 70%概率 | 需要 603 分，全市约第 XXXX 名 |
| 实力评估 | 已知估分 590，查二中 | "二中需要 615 分（70%概率），差距 25 分" |

## 五、回测验证（v2.1 新增）

### 5.1 方法

```
训练集：2023 + 2024 数据（650分制）
预测目标：2025 年录取结果（800分制）
方法：线性缩放（650→800，系数 1.2308）+ 逐校σ模型
测试校：46校中具有两年历史数据的学校
```

### 5.2 核心结论

| 指标 | 数值 | 说明 |
|------|------|------|
| MAE（平均绝对误差） | 15.44 分 | 在 800 分制下 |
| 中位误差 | -8.20 分 | 系统性低估 |
| ±20 分内命中率 | 84% | 大多数学校预测可靠 |
| ±15 分内命中率 | 65% | |
| 双年训练 MAE | 12.8 | 远优于单年 MAE=20.4 |
| 系统性偏差 | ~6.6 分低估 | 线性缩放导致分数膨胀不足 |

### 5.3 关键发现

1. **双年数据远优于单年**：使用2023+2024双年训练的 MAE=12.8，单年仅用2024为 MAE=20.4，差距显著
2. **线性缩放的局限**：650→800线性缩放（×1.2308）低估了分数分布形态差异，导致系统性低估约6.6分
3. **民办校异常值**：二中南、润德、精英中学等民办校是最大预测偏差来源
4. **稳步提升路径**：2026年获取第四年数据后，可改用位次映射替代线性缩放，并用四年数据扩大训练集

### 5.4 改进方向

- P0：2026年数据补齐后，用位次映射替代线性缩放（消除系统性偏差）
- P1：民办校引入独立修正因子（学费/招生规模/区位）
- P2：四年数据训练 + 加权滑动窗口（近期年份权重更高）

## 六、志愿组合优化（v2.1 新增）

### 6.1 算法流程

```
输入：学生分数、校内排名(可选)、初中(可选)

Step 1：全量计算
  对46校计算录取概率

Step 2：分级筛选
  冲档（概率 < 50%）：排序取最优1所
  稳档（50% ≤ 概率 < 80%）：排序取最优2所
  保档（概率 ≥ 80%）：排序取最优1所

Step 3：智能去重
  排除同一学校重复出现

Step 4：补位逻辑
  若某档无符合条件的学校：
  - 冲档空缺 → 从稳档降级补位
  - 稳档空缺 → 从保档/冲档补位
  - 保档空缺 → 从稳档升级补位

Step 5：排除过低概率
  概率 < 20% 的选项不进入任何推荐

输出：冲×1 + 稳×2 + 保×1 = 4校志愿组合
```

### 6.2 与旧版对比

| 维度 | 旧版（v2.0） | 新版（v2.1） |
|------|------------|------------|
| 冲档阈值 | < 40% | < 50% |
| 稳档区间 | 40%-70% | 50%-80% |
| 保档阈值 | ≥ 70% | ≥ 80% |
| 稳档数量 | 2 | 2 |
| 去重逻辑 | 无 | 有 |
| 补位逻辑 | 无 | 有 |
| 低概率排除 | 无 | < 20% 不推荐 |

## 七、蒙特卡洛模拟框架

### 7.1 概念

概率计算输出的71.8%不是精确预测——它是一个统计推断。为了给出更可靠的结果，我们在概念上使用蒙特卡洛方法：

```
1. 在学生分数 ± σ_估分 范围内，抽取 N 个随机分数
2. 在学校统招线 ± σ_波动 范围内，抽取 N 个随机录取线
3. 对每一组（随机分数, 随机录取线），判断是否录取
4. 统计录取次数 / 总模拟次数 = 录取概率
```

实际上，由于我们使用正态分布解析解（3.4节），这一步等价于用数学公式一次性算出概率。但向家长解释时，"几百万次模拟"比"正态CDF"更容易理解，也更准确传达了模型的本质——**在不确定性中寻找最可能的答案**。

### 7.2 未来升级方向

- 加入"大小年"修正因子
- 加入考生规模变化修正
- 非正态分布尾部建模（民办校）

## 八、分配生模型

### 8.1 基本逻辑

分配生录取概率 = 条件概率，分三步判断：

```
输入：学生分数、校内排名、学校配额数

Step 1：过线判断
  if (学生分数 < 学校统招线 - 50) → 无分配生资格，概率 = 0

Step 2：配额判断
  if (校内排名 ≤ 配额) → 高概率录取，概率 = 0.92
  else if (校内排名 ≤ 配额 × 1.3) → 线性递减 (0.92 → 0.03)
  else → 概率 = 0.03

Step 3：综合概率
  P_分配生 = P_有资格 × P_配额内 × P_实际录取
```

### 8.2 当前限制

| 问题 | 说明 |
|------|------|
| 固定概率值 | 0.92/0.03 是经验估计，未根据实际使用率校准 |
| 未考虑报考意愿 | 假设所有过资格线的学生都报考该校 |
| 未区分初中差异 | 不同初中的竞争激烈程度差异大 |
| 配额数据 | 2023/2024/2025年分配生指标数据，但使用率未知 |

### 8.3 改进路径

- 收集各初中往年的配额实际使用数据
- 引入"生-校匹配度"因子（学生层次与学校层次的匹配度）
- 分层建模：区分重点初中和普通初中的配额竞争差异
