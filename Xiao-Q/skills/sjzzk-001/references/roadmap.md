### 1.1 反向查询引擎 ✅

**状态**：v2.1 已上线（2026-06-03）

**实现方案**：有理逼近逆正态CDF（normSInv），精度 ±0.0004

```
反向引擎 = normSInv 解析解（非二分搜索）

所需分数 = 统招线 + σ_combined × Φ⁻¹(1 - 期望概率)
```

**界面**：
- 独立卡片区域：学校下拉 + 概率滑块（10-95%）
- 即时显示所需最低分数 + 对应位次
- 正向反向互验：用正向引擎对反算结果验算，偏差 ≤ 0.5%

**验收**：
- [x] 输入"二十四中 + 70%"，秒出所需分数

- [x] 支持2023/2024/2025年度切换

### 1.2 逐校 σ_波动 校准 ✅

**状态**：v2.0 已上线（2026-05-30），旧版统一 σ_波动=10 已被逐校独立 σ 替代。

**实现方案**：

```python
# 逐校 σ 计算（cross-scale normalization）
def compute_school_sigma(school, scores_2023, scores_2024, scores_2025):
    # 三年分数标准化到同一量尺（650→800 或 800→650）
    normalized = [normalize(s, yr) for s, yr in scores]
    
    # 计算 μ 和 σ
    mu = mean(normalized)
    sigma = std(normalized, ddof=1)  # 样本标准差
    
    # 保底 σ_min = 3（只有1年数据或过度稳定的学校）
    sigma = max(sigma, 3.0)
    
    # 合成标准差 σ_combined = √(12² + σ_school²)
    combined = sqrt(144 + sigma**2)
    
    return sigma, combined
```

**关键结果**：
- 46校逐校 σ 范围：3.0（二中/一中滨河/铭德等新校）~ 71.5（河北联邦外国语学校）
- 9所新校（无2023数据）使用2年数据计算，保底 σ=3
- top校（一中/二中）：σ=3~4，稳健如磐石
- 中段校：σ=5~15
- 波动校（六中/联邦外国语）：σ=36~72

**验收**：
- [x] 46校各校 σ 值计算完成，范围合理
- [x] 二中/一中 σ 明显低于统一值10
- [x] 边缘学校 σ 明显高于统一值10
- [x] v2.0 webapp 已嵌入三年数据 + 逐校 σ

### 1.3 回测验证 ✅

**状态**：v2.1 已完成（2026-06-03），回测脚本 `scripts/backtest_2025_v4.py`

**方案**：

```
训练集：2023 + 2024 数据（650分制）
预测目标：2025 年录取结果（800分制）
方法：线性缩放（×1.2308）+ 逐校σ模型
对比：单年(2024) vs 双年(2023+2024) 训练效果
```

**核心结论**：

| 指标 | 数值 | 评级 |
|------|------|------|
| MAE（平均绝对误差） | 15.44 分 | 可接受 |
| 中位误差 | -8.20 分 | 系统性低估 |
| ±20 分内命中率 | 84% | 良好 |
| ±15 分内命中率 | 65% | 一般 |
| 双年训练 MAE | 12.8 | 远优于单年 20.4 |
| 系统性偏差方向 | 低估 ~6.6 分 | 线性缩放导致 |

**关键发现**：

1. **双年数据远优于单年**：MAE 从 20.4 降至 12.8，三年数据的价值已验证
2. **线性缩放的局限**：650→800线性缩放低估了分数膨胀的实际幅度
3. **民办校异常值**：二中南、润德、精英中学是最大预测偏差来源
4. **稳步提升路径**：2026年获取第四年数据后，改用位次映射替代线性缩放

## Phase 2：决策系统 🔄

### 2.1 志愿组合优化 ✅

**状态**：v2.1 已上线（2026-06-03）

**核心算法**：

```
输入：学生分数、校内排名（可选）、初中（可选）

Step 1：全量计算 → 46校录取概率
Step 2：分级筛选 → 冲<50% / 稳50-80% / 保≥80%
Step 3：智能去重 → 排除同一学校重复
Step 4：补位逻辑 → 某档空缺时从相邻档补位
Step 5：低概率排除 → 概率<20%不推荐

输出：冲×1 + 稳×2 + 保×1 = 4校志愿组合
```

**与旧版变更**：

| 维度 | 旧版（v2.0） | 新版（v2.1） |
|------|------------|------------|
| 冲档阈值 | < 40% | < 50% |
| 稳档区间 | 40%-70% | 50%-80% |
| 保档阈值 | ≥ 70% | ≥ 80% |
| 去重逻辑 | 无 | 有 |
| 补位逻辑 | 无 | 有 |
| 低概率排除 | 无 | < 20% |

**验收**：
- [x] 任意分数输入，输出 冲×1+稳×2+保×1 组合
- [x] 风险梯度不跳档（冲→稳→保合理排列）

### 2.2 分配生双轨博弈（待开发）

**用户故事**：学生有分配生资格 → 系统建议分配生名额和统招志愿的最优搭配。

**核心问题**：

一个学生只能被录取一次。如果分配生被录取，统招志愿自动失效。因此分配生名额是**稀缺资源**——用在哪所学校最划算？

**建模**：

```
输入：学生分数、校内排名、所属初中、可选分配生学校列表

对每所"校内分配生覆盖学校"：
  Step 1：判断校内资格
    校内排名 ≤ 该校配额 → 高概率（0.92）
    校内排名 在配额1.0-1.3倍之间 → 递减概率
    校内排名 > 配额1.3倍 → 低概率（0.03）

  Step 2：比较两条路径
    路径A（分配生）：P_分配生 = P_资格 × P_配额内
    路径B（统招）：P_统招 = 正向引擎(分数, 学校)

  Step 3：决策建议
    if P_分配生 > P_统招 + 0.15:
      → 建议用分配生，统招留给其他学校
    elif P_统招 > P_分配生 + 0.15:
      → 建议走统招，分配生名额用在高一档学校
    else:
      → 两条路径概率接近，建议统招（更稳妥）
```

**改动范围**：
- `engine/calculators.py`：新增 `AssignmentPathAnalyzer` 类
- `webapp/index.html`：新增"分配生分析"Tab 或区域
- 输入表单增加：校内排名、所属初中

**验收标准**：
- [ ] 输入校内排名+初中 → 每个学校显示分配生 vs 统招两条路径概率
- [ ] 给出"分配生用在哪个学校最优"的建议
- [ ] 与统招组合推荐联动（不冲突）
