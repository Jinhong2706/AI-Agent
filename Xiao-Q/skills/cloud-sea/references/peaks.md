## API 调用示例

### Open-Meteo 天气预报
```
GET https://api.open-meteo.com/v1/forecast
?latitude={纬度}
&longitude={经度}
&hourly=temperature_2m,dew_point_2m,relative_humidity_2m,
     precipitation_probability,precipitation,
     wind_speed_10m,wind_direction_10m,cloud_cover,
     weather_code,surface_pressure
&daily=temperature_2m_max,temperature_2m_min
&timezone=Asia/Shanghai&forecast_days=7
```

### Open-Meteo 空气质量
```
GET https://air-quality-api.open-meteo.com/v1/air-quality
?latitude={纬度}
&longitude={经度}
&hourly=pm10,pm2_5,nitrogen_dioxide,ozone,aerosol_optical_depth,us_aqi
&timezone=Asia/Shanghai&forecast_days=7
```

## 注意事项

### ⚠️ dew_point_2m 生死线
- 必须从 API 实测获取，绝不假设
- RH=100% 且 T≈Td 时，LCL_ag≈0（雾在地面）

### ⚠️ station_elev vs summit_elev
- Open-Meteo 返回的 `elevation` 是模型格点海拔，非山峰实际海拔
- **权威雾顶公式（2026-04-20 统一）：`fog_top_msl = station_elev + LCL_ag + 100m`**
  - `LCL_ag = 125 × max(T − Td, 0.1)`（地面以上云底高度）
  - `fog_thickness` 固定 **100m**（辐射雾典型厚度，与 weather_fetch.py v3.0 默认值对齐）
  - ⚠️ 不得使用 summit_elev − station_elev 作为雾厚（2026-04-16 已修复）

### ⚠️ JSON 峰名编码
- weather_data_*.json 中峰名作为 dict key 时可能变成乱码
- 解决方案：遍历 peaks 取 `.peak` 字段匹配
