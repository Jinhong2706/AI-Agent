> **让 AI 像体制内笔杆子一样。**

`Official Document Skill` 是一个面向中文公文、政务应用文和机关材料写作的 Codex Skill。它会根据用户提供的文种、场景、对象和材料，生成结构规范、语气稳妥、内容具体、少 AI 味、可直接交付的文本。

V1.2 在原版基础上进一步整合了多篇优秀公文案例、公文写作标准、法定公文格式规则，以及 100 多篇人民日报文章蒸馏结果。它不是高级套话生成器，而是把优秀机关材料里的底层规律提炼出来：文种要对、事实要实、判断要稳、结尾要落地。

---

## 你有没有遇到过这种情况？

让 AI 写一份工作报告，它经常给出这样的文字：

```text
为深入贯彻落实高质量发展要求，进一步凝心聚力、开拓创新、真抓实干，
我单位持续强化责任担当，不断完善工作机制，全面推动各项工作走深走实、
提质增效，为谱写新时代事业发展新篇章贡献更大力量。
```

看起来很正式，仔细一看却有几个问题：

- 谁负责？做了什么？通过什么机制？到什么节点？有什么结果？

都没有。

这就是典型的 **AI 味**：抽象词很多，工作信息很少。

这个 Skill 要解决的就是这件事：把正确但空的文字，改成具体、稳妥、能用的公文表达。

## V1.2 相比原版升级了什么？

### 1. 升级为公文全流程写作引擎

原版重点解决 AI 腔、套话、空话问题。V1.2 增加了完整公文生成链路：

```text
识别场景 → 判断文种 → 提取事实 → 匹配结构 → 生成正文 → 降 AI 味 → 质量评分 → 交付
```

这意味着它会先判断这到底该写成通知、请示、报告、函，还是方案。

### 2. 蒸馏了多篇优秀公文案例和写作标准

V1.2 整合了公文写作资料中的法定公文格式、应用文模板、常见文种写法、格式错误、行文关系和交付校对规则。

新增强化内容包括：

- 通知、请示、报告、批复、函、纪要、通报、决定、通告等法定公文规则；
- 工作方案、工作总结、调研报告、汇报材料、讲话稿、简报、宣传稿等事务文书规则；
- 标题、主送机关、附件、落款、日期、上行文/下行文/平行文的格式约束；
- 请示与报告、通知与通告、批复与复函等易混文种辨析。

### 3. 整合 100 多篇人民日报文章蒸馏结果

V1.2 吸收了人民日报文章分析中的表达规律，但只保留可迁移到公文写作的部分：

- 事实承接判断；
- 背景不脱离具体任务；
- 问题切入要有真实表现；
- 段落要有明确功能；
- 强判断必须有强证据；
- 普通公文结尾回到责任、时限和办理要求。


### 4. 文种覆盖从 11+ 扩展到 20+

V1.2 支持更多实际工作场景：

| 类别 | 支持文种 |
|---|---|
| 法定公文 | 通知、通报、报告、请示、批复、函、纪要、决定、通告、公告、意见、议案 |
| 事务文书 | 工作方案、工作计划、工作总结、调研报告、汇报材料、简报/信息稿 |
| 会议材料 | 讲话稿、发言稿、表态发言、经验材料 |
| 面向公众 | 倡议书、公开信、感谢信、宣传稿、信访回复、政策解读 |
| 评论类 | 理论评论文章，但会严格限制其向普通公文迁移 |

### 5. 升级为“单文件 SKILL.md”

当前版本已把核心规则整合进一个 `SKILL.md`，减少调用时遗漏规则的风险。辅助文件只保留：

- `README.md`：说明和版本介绍；
- `examples.md`：示例输入输出；
- `tests.md`：人工回归测试；

## 它能做什么？

### 1. 从零生成公文

例如：

```text
用 $official-document-skill 写一份通知：区教育局通知各学校开展暑期校园安全排查，
重点查消防、用电、食堂、实验室危化品，6月30日前报送隐患台账。
```

Skill 会自动判断这是“通知”，重点写清对象、事项、要求、时限，而不是写成宣传稿。

### 2. 把口语材料转成正式公文

你可以直接给会议记录、工作要点、零散材料，它会整理成正式结构。

### 3. 降 AI 味、压缩套话

典型改写方向：

| AI 味表达 | V1.2 改写方向 |
|---|---|
| 凝心聚力、开拓创新 | 写清参与主体和共同目标 |
| 强化责任担当 | 明确责任单位、责任人、完成时限 |
| 走深走实、提质增效 | 写清落实到哪个环节、产生什么变化 |
| 形成闭环机制 | 写清发现、办理、反馈、复盘步骤 |
| 谱写新篇章 | 回到下一步任务、报送节点或工作目标 |

### 4. 检查文种和格式

它会识别这些常见问题：

- 报告里夹带请示事项；
- 请示多头主送；
- 通知写成讲话稿；
- 函写成命令；
- 标题文种和正文目的不一致；
- 提到附件但没有附件说明；
- 结尾过度宏大，脱离办理事项。

## 看一个简短例子

原句：

```text
我单位高度重视窗口服务工作，持续推进服务能力提升，全面优化服务流程，
不断增强群众满意度，推动政务服务提质增效。
```

V1.2 改写：

```text
我单位围绕窗口受理、材料审核、结果反馈三个环节改进服务流程，
统一一次性告知清单，增设线上预审入口，减少群众重复提交材料和现场等待时间。
```

区别在于：

| 原句问题 | 改写动作 |
|---|---|
| “高度重视”没有信息 | 删除空泛表态 |
| “持续推进”没有宾语 | 改成具体环节 |
| “提质增效”没有证据 | 改成减少重复提交、等待时间 |
| 抽象判断多 | 增加对象、动作、机制 |

## 四项核心能力

### 1. 文种识别

先判断文种，再匹配结构。

| 用户意图 | 推荐文种 |
|---|---|
| 要求下级办理、告知事项 | 通知 |
| 向上级请求批准、支持、经费 | 请示 |
| 向上级汇报情况、进展、问题 | 报告 |
| 不相隶属单位商洽、询问、答复 | 函 |
| 答复下级请示 | 批复 |
| 安排专项行动 | 工作方案 |
| 复盘阶段工作 | 工作总结 |
| 分析情况并提出建议 | 调研报告 |
| 会议动员部署 | 讲话稿 |
| 快速反映动态 | 简报 / 信息稿 |

### 2. 事实密度增强

V1.2 会优先补足七个维度：

```text
谁负责 → 做什么 → 针对什么问题 → 通过什么机制 → 到什么时间完成 → 如何反馈 → 有什么结果
```

### 3. 语气克制

到什么程度，说什么话。

| 表达强度 | 使用条件 |
|---|---|
| 已启动、正在开展 | 只有部署或起步动作 |
| 有序推进、稳步推进 | 有计划、有步骤，但结果有限 |
| 取得进展、初见成效 | 有阶段性结果或对象反馈 |
| 取得明显成效、形成机制 | 有数据、制度、流程或稳定运行 |
| 重大突破、历史性成就 | 有权威认定、关键指标、行业比较或重大节点 |

## 一分钟快速测试

把下面这段复制给已加载 Skill 的 AI：

```text
用 $official-document-skill 写一份通知。场景：区教育局要求各中小学开展暑期校园安全排查。重点查消防、用电、食堂、实验室危化品。6月30日前报隐患台账和整改计划
```

合格输出应当：

- 写成通知，不是宣传稿；
- 有主送机关；
- 如实写整改进展和存在问题；
- 下一步安排可执行；
- 不用“谱写新篇章”等万能结尾。

## 内置评分

生成后按六个维度自检：

| 维度 | 分值 | 看什么 |
|---|---:|---|
| 事实密度 | 20 | 有对象、动作、机制、数据或案例吗？ |
| 文种适配 | 20 | 结构、语气、结尾符合文种吗？ |
| 语气克制 | 15 | 判断强度和证据匹配吗？ |
| 句式自然 | 15 | 是否机械排比、句式单一？ |
| 抽象词控制 | 15 | 抽象词后是否有具体承接？ |
| 结构清晰 | 15 | 段落功能是否明确，推进是否自然？ |

处理规则：

| 分数 | 处理 |
|---|---|
| 90 分以上 | 可直接输出 |
| 80—89 分 | 局部压缩套话并增强具体性 |
| 70—79 分 | 重写问题段落 |
| 70 分以下 | 按文种结构整体重构 |

## 设计理念

V1.2 的核心不是写得更像 AI 认为的公文，而是让文本回到机关材料最朴素的标准：

> **文种对、事实实、结构清、语气稳、责任明、结尾落地。**

人民日报语料提供的是表达规律，不是可复制的套话；优秀公文案例提供的是结构标准，不是机械模板。真正可交付的公文，最后一定要落到具体工作。
