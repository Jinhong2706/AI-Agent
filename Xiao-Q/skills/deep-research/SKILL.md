# 深度研究技能（Deep Research）

## 技能描述

每次用户提出问题或需求时，自动执行以下流程：

1. **本地知识库搜索**：搜索工作空间内的所有知识库文件，提取相关信息
2. **互联网多平台搜索**：从多个互联网平台搜索相关内容
3. **整合分析**：结合搜索信息和自身认知，进行深度分析和整合
4. **完整推送**：生成结构完整、内容详实的回答推送给用户

## 核心原则

### 1. 问题优先解决
- **遇到问题立即解决**，而不是立即换其他方案
- 问题排查路径：检查配置 → 检查依赖 → 检查权限 → 检查代码 → 解决问题
- 每个问题至少尝试3种不同的解决方案
- 记录问题和解决方案，建立知识库

### 2. 深度优先于速度
- 宁可花更多时间获取完整信息，也不推送不完整的内容
- 信息不足时，主动告知并继续搜索
- 始终提供有深度、有洞察的分析

### 3. 完整性优先于简洁
- 提供结构完整的内容：背景 → 分析 → 结论 → 建议
- 包括数据来源、引用和参考
- 多角度分析，不同观点对比

## 工作流程

### 步骤1：接收用户问题
- 理解用户的核心需求和意图
- 识别关键概念和关键词
- 确定搜索范围和深度

### 步骤2：本地知识库搜索

#### 2.1 知识库文件列表
搜索以下知识库文件：
- `02_projects/knowledge-base/产品知识.md`
- `02_projects/knowledge-base/销售话术库.md`
- `02_projects/knowledge-base/销售技巧库.md`
- `02_projects/knowledge-base/泰康销售话术库.md`
- `02_projects/knowledge-base/产品推荐系统.md`
- `02_projects/knowledge-base/培训材料索引.md`
- `02_projects/knowledge-base/保险法律法规知识库.md`
- `02_projects/knowledge-base/常见问题.md`
- `02_projects/knowledge-base/查询索引.md`

#### 2.2 搜索方法
1. 使用关键词在知识库文件中搜索
2. 提取相关段落和上下文
3. 记录来源文件和位置

### 步骤3：互联网多平台搜索

#### 3.1 搜索平台
1. **网页搜索**：使用 web_search 工具进行通用搜索
2. **网页抓取**：使用 web_fetch 工具获取详细页面内容
3. **Twitter**：使用07技能搜索相关推文（如已配置）
4. **YouTube**：使用07技能搜索相关视频内容（如已配置）
5. **小红书**：使用07技能搜索相关笔记（如已配置）
6. **其他平台**：根据需求扩展搜索范围

#### 3.2 搜索策略
1. **关键词优化**：根据知识库搜索结果优化搜索关键词
2. **多轮搜索**：第一轮获取概览，第二轮深入特定方向
3. **来源交叉验证**：对比不同来源的信息
4. **时效性优先**：优先获取最新信息

### 步骤4：整合分析

#### 4.1 信息整理
1. **分类整理**：按主题、来源、相关性分类
2. **去重合并**：去除重复信息，合并相关内容
3. **时间排序**：按时间顺序整理信息
4. **可信度评估**：评估信息来源的可信度

#### 4.2 深度分析
1. **核心观点提取**：提取信息中的核心观点和关键数据
2. **多角度分析**：从不同角度分析问题
3. **趋势判断**：判断行业趋势和发展方向
4. **关联分析**：分析信息之间的关联和影响

#### 4.3 知识整合
1. **知识库信息 + 互联网信息**：整合两方面的信息
2. **自身知识补充**：结合自身的知识和理解
3. **交叉验证**：验证信息的准确性和一致性
4. **补充完善**：补充信息中的空白和不足

### 步骤5：生成完整回答

#### 5.1 回答结构
```
## [标题]

### 摘要
[2-3句话的简要概述]

### 背景分析
[问题背景、相关概念、历史沿革]

### 核心内容

#### 1. [主题1]
[详细内容]

#### 2. [主题2]
[详细内容]

#### 3. [主题3]
[详细内容]

### 深度洞察
[独特见解、趋势判断、关联分析]

### 实用建议
[具体的行动建议、操作指南]

### 参考资料
- 知识库来源：[文件名] - [位置]
- 互联网来源：[标题] - [URL]
- 其他来源：[描述]

### 补充说明
[时间说明、免责声明、后续建议]
```

#### 5.2 质量标准
1. **完整性**：所有相关信息都涵盖
2. **准确性**：所有信息都经过验证
3. **深度性**：有深度分析和独到见解
4. **实用性**：有具体的行动建议
5. **可读性**：结构清晰，表达流畅

### 步骤6：问题处理机制

#### 6.1 问题识别
- 搜索工具返回错误
- 搜索内容不相关或质量低
- 搜索结果为空
- 知识库文件不存在
- 文件读取失败
- 信息整合异常

#### 6.2 问题解决路径
1. **检查配置**：检查工具配置是否正确
2. **检查依赖**：检查相关依赖是否安装
3. **检查权限**：检查文件访问权限
4. **检查代码**：检查执行代码是否有误
5. **尝试替代方案**：
   - 搜索工具失败 → 更换搜索关键词 → 更换搜索工具 → 手动搜索
   - 文件读取失败 → 检查文件路径 → 检查文件编码 → 使用替代方法读取
   - 信息不足 → 扩大搜索范围 → 调整搜索策略 → 告知用户信息有限
6. **记录问题**：记录问题和解决方案到知识库
7. **反馈用户**：告知用户遇到的问题和已采取的解决措施

#### 6.3 禁止行为
- ❌ 遇到问题立即放弃当前方案
- ❌ 不记录问题和解决方案
- ❌ 推送不完整或未经验证的信息
- ❌ 忽略用户反馈和指正

### 步骤7：持续改进

#### 7.1 反馈收集
- 收集用户对回答质量的反馈
- 记录用户指出的问题和错误
- 分析改进空间

#### 7.2 知识更新
- 将新知识补充到知识库
- 更新过时信息
- 建立新的知识关联

#### 7.3 技能优化
- 优化搜索策略和关键词
- 优化回答结构和格式
- 优化问题处理机制

## 使用示例

### 示例1：产品咨询
**用户**：泰盈人生2026庆典版适合什么人买？
**执行流程**：
1. 搜索知识库中的泰盈人生2026相关信息
2. 搜索互联网上的最新评价和对比
3. 整合信息分析目标客户群体
4. 推送完整的产品分析报告

### 示例2：行业趋势
**用户**：当前保险行业的最新趋势是什么？
**执行流程**：
1. 搜索知识库中的行业分析信息
2. 搜索互联网上的行业报告和新闻
3. 整合信息分析行业趋势
4. 推送完整的行业趋势分析

### 示例3：比较分析
**用户**：泰康和其他保险公司相比有什么优势？
**执行流程**：
1. 搜索知识库中的泰康公司信息
2. 搜索互联网上的竞品分析和对比
3. 整合信息进行多维度对比
4. 推送完整的对比分析报告

## 工具依赖

### 必需工具
- `read`：读取知识库文件
- `web_search`：互联网搜索
- `web_fetch`：网页内容获取

### 可选工具
- `07技能`：多平台互联网访问（如已安装配置）

### 注意事项
- 搜索时注意API调用频率限制
- 注意信息的时效性，优先使用最新信息
- 注意信息来源的可信度
- 注意保护用户隐私和数据安全
