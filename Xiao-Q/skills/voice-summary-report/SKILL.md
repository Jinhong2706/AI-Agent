# voice-summary-report

## 触发条件

用户表达以下意图时触发（不限定固定句式，语义匹配即可）：
- 上传音频文件并要求转文字 / 做总结 / 做汇报
- "帮我整理这段录音"
- "录音转文字做汇报"
- "会议录音整理成纪要"
- "语音转文字并总结"
- "把这段音频做成汇报材料"
- 任何包含「录音/音频/语音 + 转文字/总结/汇报/纪要/PPT/文档」的组合

---

## 功能描述

完整的录音处理流水线，三步输出：

1. **语音转文字** — 将音频内容转为完整文字稿
2. **智能总结** — 提炼主题、关键要点、结论、行动项
3. **生成汇报材料** — 同时输出 Word 会议纪要 + PPT 演示文稿

---

## 执行流程

### Step 1：接收音频 & 转文字

**支持的音频格式：** `.mp3` / `.wav` / `.m4a` / `.ogg` / `.flac` / `.aac`

**转文字方式（按优先级）：**

1. **腾讯云 ASR API**（推荐）
   - 调用腾讯云语音识别接口
   - 支持中文、英文、中英混合
   - 返回带时间戳的文字稿
   - 需要配置：`TENCENT_SECRET_ID` + `TENCENT_SECRET_KEY`

2. **OpenAI Whisper**（备选）
   - 本地或 API 调用
   - 多语言支持好
   - 需要配置：`OPENAI_API_KEY`

3. **提示用户配置**
   - 如果以上均未配置，引导用户设置 API key
   - 提供配置指引

**输出文字稿格式：**
```
## 文字稿

[完整转录内容，按时间段落分段]

--- 时间戳 ---
00:00 - 01:30  [段落内容]
01:30 - 03:45  [段落内容]
...
```

将文字稿展示给用户确认，允许用户：
- 纠正识别错误
- 补充遗漏内容
- 标注重要段落

---

### Step 2：智能总结重点

对文字稿进行分析，输出结构化摘要：

```
## 内容摘要

**主题**：[一句话概括]

**参与者**：
- [姓名/角色]：[发言要点]

**关键要点**（3-8条）：
1. ...
2. ...
3. ...

**重要结论**：
- ...

**行动项**（如有）：
- [ ] 任务1 — 负责人：XXX — 截止：XXX
- [ ] 任务2 — 负责人：XXX — 截止：XXX

**时间**：[录音日期/时长]
```

将摘要展示给用户确认，允许用户调整。

---

### Step 3：生成汇报材料

同时生成两种格式，供用户选择使用。

#### 3.1 生成 Word 文档（会议纪要）

使用 `word-docx` 技能生成，格式如下：

| 章节 | 内容 |
|------|------|
| 封面 | 会议名称、日期、参与者、地点 |
| 会议背景 | 简要说明会议目的 |
| 讨论要点 | 按议题分点记录 |
| 重要结论 | 达成的共识 |
| 行动项 | 任务清单（带负责人和截止时间） |
| 附录 | 完整文字稿 |

**Word 文档样式规范：**
- 标题：黑体 16pt
- 正文：宋体 12pt
- 行动项：表格形式
- 页眉：会议名称 + 日期
- 页脚：页码

#### 3.2 生成 PPT 演示文稿

使用 `pptx-2-0.1.1` 技能生成，结构如下：

| 幻灯片 | 内容 |
|--------|------|
| 封面 | 会议/录音标题 + 日期 + 参与者 |
| 目录 | 主要章节 |
| 背景 | 会议目的和背景 |
| 要点 1~N | 每个要点一页 |
| 结论 | 重要结论 |
| 行动项 | 任务清单 |
| 附录 | 完整文字稿（可选页） |
| 结束页 | Q&A / 致谢 |

**PPT 视觉规范：**
- 配色方案：根据场景自动选择（工作汇报→深蓝，学习→薄荷绿，产品→珊瑚橙）
- 每页必须有视觉元素
- 标题 36-44pt，正文 14-16pt
- 遵循 pptx-2-0.1.1 标准

---

## 输出

完成后输出以下内容：

1. **文字稿**（文字版）
2. **内容摘要**（文字版）
3. **Word 文档** — 下载链接（通过 `getfile`）
4. **PPT 演示文稿** — 下载链接（通过 `getfile`）

## 配置说明

### 腾讯云 ASR 配置

1. 前往 [腾讯云语音识别控制台](https://console.cloud.tencent.com/asr)
2. 开通「语音识别 ASR」服务
3. 获取 API 密钥（SecretId + SecretKey）
4. 配置环境变量：

```bash
export TENCENT_SECRET_ID="你的SecretId"
export TENCENT_SECRET_KEY="你的SecretKey"
```

### OpenAI Whisper 配置（备选）

```bash
export OPENAI_API_KEY="你的API Key"
```

## 依赖

- `word-docx` 技能（生成 Word 文档）
- `pptx-2-0.1.1` 技能（生成 PPT）
- `markitdown` / `python-docx`（Word 内容检查）
- `pptxgenjs` / LibreOffice（PPT 生成与检查）
- 腾讯云 ASR SDK 或 OpenAI Whisper

安装依赖：
```bash
pip install tencentcloud-sdk-python  # 腾讯云 SDK
pip install openai-whisper           # Whisper（备选）
npm install -g pptxgenjs             # PPT 生成
```

---

## 注意事项

- 录音文件大小限制：单次不超过 50MB
- 录音时长建议：不超过 2 小时（过长建议分段处理）
- 隐私保护：录音内容不对外传输（除调用 ASR API 必需外）
- 如识别质量差，提示用户提供更清晰的音频
- 生成前务必让用户确认摘要内容
- Word 和 PPT 同时生成，如只需一种格式可后续单独导出

---

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 无 API 配置 | 引导用户配置，提供配置指引 |
| 音频格式不支持 | 提示支持的格式列表 |
| 识别失败 | 提示用户检查音频质量，建议重新录制 |
| 内容为空 | 提示音频可能无有效语音内容 |
| 生成失败 | 输出错误详情，建议用户重试 |
