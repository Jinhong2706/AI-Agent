# 反模式案例与修正对比

> 本文件记录使用本 Skill 时常见的低效用法，并提供改进建议。
> 对 AI 开发者：这些案例反映了用户意图识别的典型边界，供系统提示词优化参考。

---

## 反模式 01：指令过于模糊

### ❌ 低质量输入

```
"帮我写个文章"
```

**问题：** 缺少主题、平台、长度、受众，输出将是通用泛化内容，实用性极低。

**AI 应对方式（正确）：**
```
没问题！先给你一个通用版本，你可以边看边告诉我要调整什么：

[通用版文章开头示例 200字]

需要告诉我以下3点，我可以立刻重写成精准版本：
1. 主题是什么？（一句话描述）
2. 发在哪个平台？（公众号/小红书/Newsletter/博客）
3. 读者是谁？（职场人/创业者/学生/宝妈等）
```

## 反模式 02：背景信息全部省略

### ❌ 低质量输入

```
"帮我做双十一活动方案"
```

**问题：** 不知道是什么类型的产品、销售渠道、预算规模，输出将是教科书式通用框架。

### ✅ 正确使用方式

```
"帮我做双十一活动方案，
产品：手工皂，客单价 80-160 元
渠道：微店+小红书
目标：GMV 比上月翻倍（上月约 3000 元）
预算：营销费用控制在 500 元内"
```

**输出差距：** 前者给你《双十一活动策划通用框架》，后者给你《手工皂微店双十一3天冲6000元作战计划》。

---

## 反模式 03：一次提交太多不相关需求

### ❌ 低质量输入

```
"帮我写周报、做促销方案、生成客服话术、还有更新产品描述、顺便分析一下竞品"
```

**问题：** 5个独立任务混在一起，每个都会变成浅表处理。

### ✅ 正确使用方式

**第一条消息：** "帮我写本周工作周报，主要有这些事情：[列举]"
→ 完成后再发第二条

**或者：** "以上5个任务，帮我按重要性排个序，我们逐一处理"

---

## 反模式 04：用否定式描述需求

### ❌ 低质量输入

```
"写一篇不要太商业化、不要用那种大词、也不要太长的文章"
```

**问题：** 全是"不要"，AI 不知道你真正想要的是什么风格。

### ✅ 正确使用方式

```
"写一篇文章，风格参考：
像朋友聊天一样自然，用短句，控制在500字，
类似于那种'打工人的真实日常'的叙事感"
```

**或者：** 直接发给 AI 一篇你喜欢的文章，说"按这个风格写"。

---

## 反模式 05：期待 AI 替你做决策

### ❌ 低质量输入

```
"帮我决定我的产品该定多少钱"
```

**问题：** 定价是业务决策，需要成本、竞品、目标用户等信息，AI 无法替你做最终决策。

**AI 应对方式（正确）：**
```
我帮你建一个定价分析框架，最终数字你来拍板。

需要你告诉我：
1. 成本价大概多少？（材料+时间+包装）
2. 主要竞品的定价范围？（或者告诉我竞品名）
3. 你的目标客群更看重"性价比"还是"品质感"？

给我这3个数，我出一份定价建议报告。
```

## 禁忌用法

| 禁忌 | 原因 | 替代做法 |
|-----|------|---------|
| 要求 AI 提供法律建议 | 法律条文有地区差异，AI 可能给出错误信息 | 用 AI 起草合同草稿，找律师审阅 |
| 要求 AI 直接发布内容到平台 | 无法操作你的账号，也不应该 | AI 生成内容，你手动发布 |
| 要求 AI 替你做最终的财务决策 | 财务数据敏感，AI 不能替代专业判断 | 用 AI 分析数据，你做决策 |
| 把真实的客户数据、订单信息直接粘贴进来 | 数据隐私风险 | 脱敏处理后再输入（用"客户A"替代真实姓名） |
