# 多渠道内容分发优化器

## 元信息

- **Skill ID**: multi-channel-content-distribution
- **版本**: 1.0.0
- **分类**: 内容运营 / 多平台分发
- **作者**: AI Agent Skills Factory
- **适用场景**: 一篇核心内容需要同时分发到多个中文社交媒体平台

## 触发条件

当用户输入包含以下关键词时触发本技能：

- **中文触发词**: 内容分发、多渠道分发、适配平台、多平台发布、内容改编、全平台分发、跨平台适配
- **英文触发词**: content distribution、multi-channel distribution、cross-platform adaptation

**不触发场景**：
- 单平台发布排期管理
- 社交媒体账号运营与粉丝管理
- 内容数据分析与复盘
- 广告投放与商业化策略

## 技能描述

将用户输入的一篇核心内容（文章 / 视频脚本 / 图文），自动适配微信公众号、小红书、抖音、B站、知乎、微博六大主流中文社交媒体平台的内容格式和风格要求，生成各平台优化版本。确保每个版本符合目标平台的算法偏好、用户阅读习惯和最佳实践。

## 支持平台

| 平台 | 内容类型 | 字数范围 | 核心特征 |
|------|---------|---------|---------|
| 微信公众号 | 长图文 | 1500-3000字 | 深度分析、结构化排版、封面图 |
| 小红书 | 图文笔记 | 300-800字 | 视觉优先、Emoji装饰、标签吸睛 |
| 抖音 | 短视频脚本 | 100-300字 | 前3秒钩子、节奏紧凑、口语化 |
| B站 | 中长视频脚本 | 800-2000字 | 知识性强、弹幕互动点、分P结构 |
| 知乎 | 深度回答/文章 | 2000-5000字 | 逻辑严密、数据支撑、引用标注 |
| 微博 | 短图文 | 100-300字 | 话题标签、热点结合、传播性强 |

## 五步工作流

### 第一步：内容分析（Content Analysis）

- 识别输入内容的核心主题、目标受众、内容类型（科普/观点/教程/评测/故事）
- 提取关键信息点（3-7个核心论点或信息节点）
- 判断内容适配性评分（0-100分），低于40分建议用户调整素材
- 输出：内容画像摘要 + 关键信息点清单

### 第二步：平台适配（Platform Adaptation）

根据目标平台的用户画像和内容消费习惯，对核心内容进行适配：

- **微信公众号**：保留完整论证结构，增加小标题分段，补充背景引言和总结
- **小红书**：提炼3-5个精华要点，重新组织为「总分总」结构，加入Emoji装饰
- **抖音**：提取最吸引人的1个核心观点作为钩子，脚本按「钩子-展开-反转-升华」结构组织
- **B站**：保持知识深度，增加互动节点（弹幕触发点），规划分P结构
- **知乎**：强化论据和数据支撑，增加引用来源标注，按「现象-分析-结论」结构展开
- **微博**：提炼1句话核心观点，配合2-3个话题标签，适当加入反问句引发讨论

### 第三步：格式转换（Format Conversion）

严格按各平台格式规范执行转换：

- **字数控制**：各平台版本严格控制在对应字数范围内
- **标题优化**：每个版本生成2-3个备选标题，符合目标平台标题风格
- **配图建议**：根据内容主题生成配图描述（数量、比例、风格）
- **标签策略**：按平台规则生成标签列表
  - 小红书：5-8个标签，含热门+垂直标签
  - 微博：2-3个话题标签（#格式）
  - 抖音：3-5个话题标签
  - B站：5-10个标签
  - 知乎：1-3个话题
  - 微信：无标签系统，通过SEO标题优化

### 第四步：内容优化（Optimization）

- **钩子检测**：确认前3行（抖音）/ 首段（图文）具有足够的吸引力
- **CTA优化**：根据平台特性添加适当的行动号召（关注/点赞/收藏/转发）
- **敏感词检查**：标记可能的平台敏感词并建议替换方案
- **可读性评估**：评估每个版本的阅读/观看体验得分

### 第五步：交付输出（Delivery）

输出格式为结构化报告，包含：

- 各平台适配版本全文
- 标题备选列表
- 配图建议清单
- 发布时间建议（基于各平台流量高峰）
- 内容优化评分卡
- 后续互动话术建议

## 约束与限制

1. **内容保真原则**：适配过程中不得改变核心观点和关键数据，允许调整表述方式和叙事结构
2. **平台合规原则**：所有输出必须符合目标平台的内容规范和社区公约
3. **原创性原则**：不做简单的复制粘贴，每个版本必须针对平台特性进行实质性改写
4. **质量均等原则**：不因平台字数限制而牺牲内容质量，通过精炼表达保持信息密度
5. **时效性提醒**：提供发布时间建议时需考虑热点时效性，时效性内容应在24小时内发布

## 输入格式

用户可提供以下任一形式的输入：

- 直接粘贴文章/脚本文本
- 提供文件路径（Markdown/TXT/DOCX）
- 提供URL链接（自动抓取内容）
- 提供内容要点（由技能辅助扩写后分发）

## 边界情况处理

本技能对以下边界情况有明确处理策略：

- **输入内容过短（< 50字）**：提示「输入内容过短，无法有效适配多平台。请提供至少200字的原始内容。」并停止适配流程，等待用户补充内容。
- **内容适配评分过低（< 40/100）**：启用降级策略，仅输出最适合的2个平台版本，并在报告中标注「建议优化原始内容后重新分发」。
- **多媒体输入（图片/视频）**：提示「当前版本仅支持文本内容适配，图片/视频请单独提供标题和描述文字作为适配输入」，引导用户提供文本素材。
- **单一平台适配请求**：提示「如需仅适配单一平台，此流程仍然适用（Step 4跳过跨平台对比）」，自动调整工作流，保留单平台深度优化。

## 依赖工具

- 内容分析：文本处理（纯 Python stdlib）
- 格式验证：`scripts/validate.sh`
- 适配计算：`scripts/content-adapter.py`
- 平台规范：`references/多平台内容标准.md`
- 配置参数：`assets/config.json`

### 可扩展性说明

- **平台配置完全由 `assets/config.json` 驱动**，新增平台仅需在 config.json 的 `platforms` 数组中添加条目（name, char_limit, image_ratio, hashtag_limit, best_post_times, algorithm_prefs）。Python 脚本 `scripts/content-adapter.py` 自动加载配置，无需修改代码。
- **用户自定义标签库和标题模板存放在 `references/templates/`**，通过 config.json 的 `custom_templates` 字段引用，支持按行业垂直领域定制适配策略。
