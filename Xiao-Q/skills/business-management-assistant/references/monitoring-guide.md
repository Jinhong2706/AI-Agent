## 概览
本文档提供了对话式业务管理工具（BMA）外部信号监控的配置指南和操作规范。通过合理配置监控源和关键词，确保及时捕获与业务相关的外部变化。

## 监控配置概览

### 配置文件位置
`./sdp_data/config.yaml`

### 配置结构
```yaml
# 监控配置
monitoring:
  enabled: true  # 是否启用监控
  frequency: daily  # 搜索频率：daily/weekly
  sources:  # 信号源列表
    - name: [来源名称]
      type: [类型]
      keywords: [关键词列表]
      url: [URL，如果是RSS]
  global_keywords: [全局关键词]
```

### 启用/禁用监控
- **启用**：设置 `monitoring.enabled: true`
- **禁用**：设置 `monitoring.enabled: false`（不执行自动搜索）

## 信号源配置方法

### 信号源类型
1. **web_search**：使用web_search工具搜索关键词
2. **rss**：从RSS源读取（暂未实现，预留）
3. **manual**：手动添加信号，不自动搜索

### 配置示例

#### 竞品监控
```yaml
monitoring:
  sources:
    - name: 百度秒哒
      type: web_search
      keywords:
        - 百度秒哒
        - 百度AI
        - Baidu Miaoda
    - name: OpenAI动态
      type: web_search
      keywords:
        - OpenAI
        - GPT
        - ChatGPT API
        - OpenAI Blog
```

#### 行业趋势监控
```yaml
monitoring:
  sources:
    - name: AI工具行业
      type: web_search
      keywords:
        - AI工具市场
        - 对话式应用
        - H5生成工具
        - 无代码开发
    - name: 技术趋势
      type: web_search
      keywords:
        - LLM应用
        - AI生成界面
        - 自然语言UI
```

#### 政策法规监控
```yaml
monitoring:
  sources:
    - name: 政策法规
      type: web_search
      keywords:
        - AI监管政策
        - 互联网内容审核
        - 数据安全法规
```

### 最佳实践
1. **分层配置**：将信号源按类型分组（竞品、行业、技术、政策）
2. **关键词多样性**：每个信号源配置3-5个相关关键词
3. **避免重复**：确保关键词不重叠，减少冗余搜索
4. **定期更新**：根据业务发展调整关键词列表

## 关键词设置策略

### 关键词分类

#### 1. 竞品关键词
- **直接竞品**：竞争对手产品名称（如"百度秒哒"）
- **间接竞品**：相关领域产品（如"Canva H5"、"易企秀"）
- **竞品功能**：竞争对手的核心功能（如"对话式生成"、"AI模板"）

#### 2. 技术关键词
- **核心技术**：业务使用的关键技术（如"LLM"、"GPT-4"、"自然语言处理"）
- **技术趋势**：相关技术的发展方向（如"AI生成界面"、"无代码开发"）
- **技术突破**：可能影响业务的技术突破（如"多模态模型"、"实时渲染"）

#### 3. 行业关键词
- **市场趋势**：行业发展趋势（如"AI工具市场增长"、"H5需求"）
- **用户需求**：用户关注的问题（如"快速制作H5"、"无代码设计"）
- **平台动态**：相关平台的更新（如"微信小程序"、"支付宝小程序"）

#### 4. 政策关键词
- **监管政策**：行业监管相关（如"AI监管"、"内容审核"）
- **数据安全**：数据保护相关（如"数据安全法"、"隐私保护"）
- **合规要求**：业务合规相关（如"互联网信息服务"）

### 关键词优化技巧

#### 同义词扩展
为每个核心关键词配置同义词和变体：
- "LLM" → "大语言模型"、"Large Language Model"、"GPT"
- "对话式应用" → "Conversational App"、"聊天机器人"、"Chatbot"
- "H5" → "HTML5"、"移动网页"、"微信H5"

#### 中英文混合
对于技术术语，同时配置中英文：
- "自然语言处理" + "NLP"
- "无代码开发" + "No-code"
- "AI生成界面" + "AI-generated UI"

#### 时间限定
在搜索时添加时间限定，避免过期信息：
- "百度秒哒 2026"
- "AI工具 最新"
- "H5生成工具 新闻"

#### 排除词
在搜索时添加排除词，过滤无关信息：
- "百度秒哒 招聘" -排除
- "AI工具 教学" -排除
- "H5生成 教程" -排除

### 全局关键词配置
在 `config.yaml` 中配置全局关键词，用于所有方向的通用监控：

```yaml
monitoring:
  global_keywords:
    - AI
    - 人工智能
    - 对话式
    - 无代码
    - 低代码
```

全局关键词的作用：
- 作为补充搜索，捕获更广泛的信号
- 在特定方向关键词结果不足时使用
- 提高信号覆盖率

### 业务方向关键词配置
在每个业务方向中配置特定关键词：

```yaml
## 方向 D001：对话式界面生成
- **外部信号关键词**: 百度秒哒, 对话式应用开发, LLM UI生成
```

业务方向关键词的特点：
- 针对性强，直接关联业务方向
- 数量较少，3-5个即可
- 与全局关键词配合使用

## 搜索频率与时机

### 频率选项
- **daily**：每日搜索一次（推荐用于快速变化的市场）
- **weekly**：每周搜索一次（推荐用于稳定的市场）
- **biweekly**：每两周搜索一次（推荐用于长期战略）

### 配置示例
```yaml
monitoring:
  frequency: daily
```

### 频率选择建议

| 业务场景 | 推荐频率 | 理由 |
|---------|---------|------|
| 快速变化的市场（AI、互联网） | daily | 技术更新快，竞争激烈 |
| 稳定行业（传统软件） | weekly | 变化较慢，无需高频监控 |
| 长期战略规划 | biweekly | 关注长期趋势，无需频繁更新 |

### 搜索时机
- **主动触发**：用户执行"搜索外部信号"命令时立即搜索
- **定时触发**：根据 frequency 配置自动搜索
- **事件触发**：重大事件发生时（如竞品发布）主动搜索

### 搜索执行流程
1. 读取 `config.yaml` 获取监控配置
2. 读取 `business_framework.md` 获取各方向的关键词
3. 合并全局关键词和方向关键词
4. 调用 web_search 搜索关键词
5. 评估搜索结果的关联性
6. 写入信号日志
7. 高关联度信号主动提醒

## 信号评估标准

### 关联度分级

#### 高关联度（High Relevance）
**定义**：直接影响业务方向或路径的信号

**判断标准**：
- 直接竞争对手发布新产品或新功能
- 关键技术有重大突破或更新
- 政策法规变化直接影响业务
- 行业标准或平台规则变更
- 用户投诉或负面反馈激增

**处理方式**：
- 立即写入信号日志
- 主动推送提醒
- 提供具体建议
- 记录到 decisions.log

**示例**：
```
⚠️ 百度秒哒发布企业级API，与D001核心能力重叠
→ 建议暂停P001，创建竞品分析路径
```

#### 中关联度（Medium Relevance）
**定义**：间接影响业务方向的信号

**判断标准**：
- 行业趋势报告
- 相关技术讨论
- 间接竞品动态
- 市场规模变化
- 用户需求变化

**处理方式**：
- 写入信号日志
- 提供摘要（用户主动查询时）
- 记录为参考信息

**示例**：
```
✅ 行业报告显示对话式应用需求增长30%
→ 验证了业务方向的合理性，无需立即行动
```

#### 低关联度（Low Relevance）
**定义**：对业务方向影响很小或不确定的信号

**判断标准**：
- 一般行业新闻
- 通用技术更新
- 不相关领域的AI新闻
- 宏观经济新闻

**处理方式**：
- 写入信号日志
- 不主动推送
- 作为背景信息记录

**示例**：
```
📝 AI工具市场整体增长10%
→ 记录即可，无需立即响应
```

### 评估流程

#### 步骤1：提取关键信息
从搜索结果中提取：
- 信号标题
- 发布时间
- 来源
- 核心内容
- 相关关键词

#### 步骤2：匹配业务方向
检查信号与业务方向的关联性：
- 是否匹配业务方向的关键词？
- 是否影响业务方向的成功指标？
- 是否影响业务方向的路径执行？

#### 步骤3：确定关联度
根据上述标准确定关联度：
- 满足高关联度任一标准 → 高
- 满足中关联度任一标准 → 中
- 其他 → 低

#### 步骤4：生成建议
为高关联度信号生成具体建议：
- 分析影响范围
- 提供可选方案
- 推荐最优方案

### 评估示例

#### 示例1：竞品发布（高关联度）
**信号内容**：百度秒哒发布企业级API，支持对话式应用开发

**评估过程**：
1. 提取关键信息：竞品发布新功能
2. 匹配业务方向：匹配D001关键词"百度秒哒"
3. 确定关联度：直接竞品 → 高
4. 生成建议：暂停P001，创建竞品分析路径

#### 示例2：行业趋势（中关联度）
**信号内容**：行业报告显示对话式应用需求增长30%

**评估过程**：
1. 提取关键信息：市场需求增长
2. 匹配业务方向：匹配D001关键词"对话式应用"
3. 确定关联度：市场趋势 → 中
4. 生成建议：无需立即行动，继续观察

#### 示例3：通用新闻（低关联度）
**信号内容**：AI工具市场整体增长10%

**评估过程**：
1. 提取关键信息：市场整体增长
2. 匹配业务方向：不直接匹配
3. 确定关联度：通用新闻 → 低
4. 生成建议：记录即可

## 主动提醒触发条件

### 触发条件

#### 条件1：高关联度信号
- **触发**：评估出高关联度信号时
- **时机**：信号评估完成后立即
- **内容**：信号摘要、影响分析、建议行动

#### 条件2：里程碑即将到期
- **触发**：里程碑截止日期剩余3天时
- **时机**：每日定时检查
- **内容**：里程碑信息、进度提醒、风险提示

#### 条件3：任务逾期
- **触发**：任务截止日期已过但状态非completed时
- **时机**：每日定时检查
- **内容**：逾期任务列表、建议操作

#### 条件4：连续失败
- **触发**：某路径连续3周无进展或关键里程碑失败时
- **时机**：每周复盘时检查
- **内容**：失败分析、调整建议

### 提醒内容模板

#### 高关联度信号提醒
```
⚠️ **新信号提醒**

**信号内容**：[信号摘要]

**关联方向**：D[编号] [方向名称]

**影响评估**：[影响分析]

**建议行动**：
1. [行动项1]
2. [行动项2]
3. [行动项3]

是否需要我帮您执行这些调整？
```

#### 里程碑到期提醒
```
⏰ **里程碑即将到期**

**路径 P[编号]：[路径名称]**
- 里程碑 M[编号]：[里程碑名称]
- 截止日期：[日期]（剩余3天）
- 当前进度：[百分比]%

**风险提示**：[风险分析]

**建议操作**：[建议行动]
```

#### 任务逾期提醒
```
⚠️ **逾期任务提醒**

以下任务已逾期，需要关注：

1. **T[编号]：[任务名称]**
   - 所属路径：P[编号]
   - 截止日期：[日期]（逾期 [天数] 天）
   - 状态：[当前状态]
   - 建议操作：[延期/关闭/重新指派]

是否需要我帮您调整这些任务？
```

#### 连续失败提醒
```
⚠️ **路径执行异常**

**路径 P[编号]：[路径名称]**
- 状态：连续 [天数] 无进展
- 问题：[问题描述]

**建议操作**：
1. 重新评估路径可行性
2. 调整负责人或资源
3. 关闭路径，停止投入

是否需要我帮您评估并调整？
```

### 提醒频率控制
- **避免重复提醒**：同一信号只提醒一次
- **批量提醒**：同类信号批量提醒，避免频繁打扰
- **用户控制**：用户可以通过 `config.yaml` 调整提醒设置

## 常见信号类型与处理

### 竞品动态

#### 产品发布
**示例**：竞品发布新功能
**处理**：
- 分析功能对比
- 评估影响范围
- 建议调整产品策略

#### 价格调整
**示例**：竞品降价或推出免费版
**处理**：
- 分析价格策略
- 评估市场影响
- 建议调整定价或增加差异化功能

#### 市场推广
**示例**：竞品加大市场投入
**处理**：
- 分析推广策略
- 评估用户获取成本
- 建议调整市场策略

### 技术突破

#### 新技术发布
**示例**：新的LLM模型发布
**处理**：
- 评估技术优势

- 建议技术调研

#### 技术趋势
**示例**：某技术方向成为趋势
**处理**：
- 分析趋势影响
- 评估是否需要跟进

### 政策法规

#### 新法规出台

**处理**：
- 分析合规要求
- 评估产品影响
- 建议合规调整

#### 监管加强
**示例**：某领域监管加强
**处理**：
- 评估风险
- 建议风险控制措施

#### 数据安全
**示例**：数据安全法规更新
**处理**：
- 分析数据处理流程
- 评估合规风险
- 建议数据安全调整

### 市场变化

#### 需求变化
**示例**：用户需求发生变化
**处理**：
- 分析需求变化原因
- 评估产品匹配度
- 建议产品调整

#### 新场景出现
**示例**：新的应用场景出现

### 用户反馈

## 监控优化建议

### 定期审查
- **频率**：每月审查一次监控配置
- **内容**：
  - 评估关键词有效性
  - 调整信号源配置
  - 优化搜索频率

### 数据驱动
- **分析**：定期分析信号日志，评估监控效果
- **指标**：
  - 高关联度信号占比
  - 信号响应速度
  - 建议采纳率

### 持续改进
- **反馈**：根据用户反馈调整监控策略
- **优化**：根据实际情况优化关键词和提醒频率

### 团队协作
- **共享**：将信号日志与团队共享
- **讨论**：定期讨论高关联度信号的应对策略
- **决策**：基于信号做出数据驱动的决策
