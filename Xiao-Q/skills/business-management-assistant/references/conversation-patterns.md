## 场景2：创建初始业务框架

### 用户输入示例
- "创建业务框架：目标是成为对话式H5生成工具的首选个人方案"
- "我想建立一个AI工具的业务规划"
- "帮我搭建一个产品线的战略框架"

### 智能体响应模板
```
我将帮您创建业务框架。基于您的描述，我需要了解以下信息：

**业务目标**：成为对话式H5生成工具的首选个人方案

请提供以下信息：
1. **目标用户**：谁是您的目标用户？（如：非技术用户、中小企业、设计师等）
2. **核心价值**：这个业务为用户解决什么核心问题？（如：降低开发门槛、提升创作效率）
3. **初始方向**：您希望从哪些方向入手？（如：对话式界面生成、自然语言UI设计、市场验证等）
4. **成功指标**：如何衡量业务成功？（如：用户数量、转化率、满意度）

您可以一次性提供所有信息，也可以逐项回答。
```

### 执行要点
1. **检查现有框架**：先读取 `./sdp_data/business_framework.md`，确认是否已存在
2. **收集必要信息**：按照模板询问，确保收集到所有必需信息
3. **创建方向**：为每个方向分配编号（D001, D002, ...）
4. **生成关联路径**：询问是否需要为方向创建初始路径
5. **写入文件**：按照 [data-structures.md](data-structures.md) 的格式写入文件
6. **记录决策**：在 decisions.log 中记录方向创建操作

## 场景3：感知外部信号并响应

### 用户输入示例
- "搜索外部信号，关键词是'百度秒哒'"
- "检查最近的市场变化"
- "有没有新的竞争对手发布产品？"

### 智能体响应模板（高关联度信号）
```
⚠️ **新信号提醒**

**信号内容**：[信号摘要]

**关联方向**：D[编号] [方向名称]

**影响评估**：[影响分析]

**建议行动**：
1. [行动项1]
2. [行动项2]
3. [行动项3]

是否需要我帮您执行这些调整？
```

### 智能体响应模板（中低关联度信号）
```
✅ **外部信号摘要**（过去24小时）

**中关联度信号**：[数量]条
- [信号1摘要]
- [信号2摘要]

**低关联度信号**：[数量]条
- [信号1摘要]
- [信号2摘要]

这些信号暂无需立即行动，已记录到信号日志中。
```

### 执行要点
1. **读取配置**：从 `./sdp_data/config.yaml` 读取监控关键词
2. **读取业务框架**：从 `./sdp_data/business_framework.md` 读取各方向的关键词
3. **执行搜索**：调用 web_search 搜索关键词
4. **评估关联度**：根据搜索结果判断与业务方向的关联性（高/中/低）
5. **高关联度处理**：
   - 写入 `./sdp_data/signals/YYYY-MM-DD_signal_log.md`
   - 主动推送提醒
   - 提供具体建议
6. **中低关联度处理**：
   - 写入信号日志
   - 提供摘要（仅在用户主动查询时）

### 关联度评估标准
- **高关联**（主动提醒）：
  - 直接竞争对手发布新产品或新功能
  - 关键技术突破或重大更新
  - 政策法规变化直接影响业务
  - 行业标准或平台规则变更
- **中关联**（记录摘要）：
  - 行业趋势报告
  - 相关技术讨论
  - 间接竞品动态
  - 市场规模变化
- **低关联**（仅记录）：
  - 一般行业新闻
  - 通用技术更新
  - 不相关领域的AI新闻

