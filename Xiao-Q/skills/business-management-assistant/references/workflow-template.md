## 概览
本文档提供了对话式业务管理工具（BMA）的默认使用流程模板。你可以根据实际业务需求修改、跳过或自定义流程。

## 默认流程概览

默认流程包含 6 个阶段，共 18 个步骤：

| 阶段 | 步骤数 | 预计时间 | 说明 |
|------|--------|----------|------|
| 系统初始化 | 2 | 5分钟 | 创建数据目录和核心文件 |
| 构建业务框架 | 4 | 15-30分钟 | 定义业务方向和目标 |
| 拆解实现路径 | 3 | 20-40分钟 | 创建实现路径和里程碑 |
| 创建任务跟踪 | 3 | 持续进行 | 建立任务管理体系 |
| 外部信号感知 | 3 | 持续进行 | 配置监控和主动提醒 |
| 综合视图支持 | 3 | 按需使用 | 建立决策支持体系 |

# 使用流程

## 阶段一：系统初始化
- **状态**: pending
- **进度**: 0/2 步骤
- **预计时间**: 5分钟

### 步骤 1.1：创建数据目录
- **状态**: pending
- **描述**: 创建 sdp_data 数据目录结构
- **操作**: mkdir -p ./sdp_data/paths ./sdp_data/signals
- **提示**: 此步骤将在当前工作目录创建数据存储目录
- **对话示例**:
  ```
  智能体：正在创建数据目录...

  ✅ 已创建目录：
  - ./sdp_data/
  - ./sdp_data/paths/
  - ./sdp_data/signals/
  ```
- **检查条件**: 检查 ./sdp_data 目录是否存在
- **关联文件**: ./sdp_data/

### 步骤 1.2：初始化核心文件
- **状态**: pending
- **描述**: 创建配置文件和日志文件
- **操作**: 创建 config.yaml、decisions.log、tasks.md
- **提示**: 这些文件存储系统配置和决策记录
- **对话示例**:
  ```
  智能体：正在初始化核心文件...

  💡 提示: 此步骤将创建以下配置文件：
  - config.yaml: 系统配置文件
  - decisions.log: 决策记录日志
  - tasks.md: 任务管理文件

  ✅ 已完成：创建所有核心文件
  ```
- **检查条件**: 检查 ./sdp_data/config.yaml 是否存在
- **关联文件**: ./sdp_data/config.yaml

## 阶段二：构建初始业务框架

## 流程自定义指南

### 添加新阶段
如果你需要添加新的流程阶段：

1. 在 workflow.md 中添加新的阶段定义
2. 设置阶段状态为 pending
3. 为新阶段添加步骤
4. 更新阶段进度

示例：
```markdown
## 阶段七：团队协作
- **状态**: pending
- **进度**: 0/2 步骤
- **预计时间**: 持续进行

### 步骤 7.1：配置团队成员
- **状态**: pending
- **描述**: 添加团队成员和权限
- **操作**: 对话引导，配置团队
- **提示**: 团队协作需要明确的角色分工
```

### 修改步骤
如果你需要修改某个步骤的内容：

1. 使用命令"编辑步骤 X.Y"
2. 提供新的描述、提示或对话示例
3. 智能体将更新 workflow.md

示例：
```
用户：编辑步骤 2.3

智能体：📝 **编辑步骤 2.3：设定成功指标**

当前内容：
- 描述: 设定量化的成功指标
- 提示: 具体的指标有助于跟踪进度

请提供新的描述、提示或示例：

用户：提示改为"请设定SMART原则的成功指标"

智能体：✅ 已更新步骤 2.3
```

### 跳过不相关的步骤
如果你的业务不需要某个步骤：

1. 使用命令"跳过步骤 X.Y"
2. 智能体将标记该步骤为 skipped
3. 流程进度会自动调整

示例：
```
用户：跳过步骤 5.3

智能体：✅ 已跳过步骤 5.3：启用主动提醒

流程进度已更新
```

### 重置流程
如果需要重新开始流程：

```
用户：重置流程

智能体：⚠️ 确认重置流程？这将把所有步骤重置为 pending 状态。

用户：确认

智能体：✅ 流程已重置
```

## 流程最佳实践

1. **按需执行**：不必严格按顺序执行，可以根据业务需求跳过不相关的步骤
2. **持续迭代**：流程不是一成不变的，可以根据使用情况不断优化
3. **灵活自定义**：大胆添加、修改、删除步骤，让流程适应你的业务
4. **记录经验**：在 workflow.md 中记录你的修改和优化，便于团队共享
5. **定期回顾**：定期检查流程的有效性，去除不必要的步骤，优化关键环节

## 故障排查

### 问题：流程卡在某个步骤
**解决方案**：
1. 检查检查条件是否满足
2. 尝试"跳过步骤 X.Y"
3. 尝试"重做步骤 X.Y"

### 问题：无法显示流程
**解决方案**：
1. 检查 workflow.md 文件是否存在
2. 尝试"使用默认流程"恢复
3. 手动创建 workflow.md 文件

### 问题：流程进度不准确
**解决方案**：
