# chat-summary-ppt

## 触发条件

用户提到以下意图时触发：
- "总结聊天记录做成PPT"
- "把对话整理成PPT"
- "会议记录生成PPT"
- "把这些内容做成演示文稿"
- 任何包含「聊天/对话/记录/纪要 + PPT/演示文稿/幻灯片」的组合

---

## 功能描述

收集聊天记录或对话内容，智能提炼重点，自动生成精美 PPT 演示文稿。

---

## 执行流程

### Step 1：收集内容

询问用户内容来源（按优先级）：

1. **自动读取**：当前会话的最近对话内容（默认）
2. **指定文件**：用户上传的 `.txt` / `.md` / `.docx` 文件
3. **指定目录**：读取 `memory/` 目录下的日期文件
4. **手动输入**：用户直接粘贴内容

如果用户没有指定，默认使用当前会话上下文。

---

### Step 2：分析 & 总结

对收集到的内容进行分析，输出结构化摘要：

```
## 内容摘要

**主题**：[一句话概括主要话题]

**关键要点**（3-6条）：
1. ...
2. ...
3. ...

**重要结论**：
- ...

**参与者**（如有）：
- ...

**时间范围**（如有）：
- ...
```

将摘要展示给用户确认，允许用户：
- 调整要点数量
- 补充遗漏内容
- 指定 PPT 风格偏好

---

### Step 3：设计 PPT 结构

根据内容自动规划幻灯片结构：

| 幻灯片 | 内容 |
|--------|------|
| 封面 | 标题 + 副标题 + 日期 |
| 目录 | 主要章节概览 |
| 背景/引言 | 为什么讨论这个话题 |
| 要点 1~N | 每个要点一页，配视觉元素 |
| 结论/总结 | 关键结论 + 后续行动 |
| 致谢/Q&A | 结束页 |

**默认配色方案**（根据主题自动选择）：
- 工作汇报 → Midnight Executive（深蓝）
- 学习分享 → Sage Calm（薄荷绿）
- 产品/创意 → Coral Energy（珊瑚橙）
- 正式会议 → Charcoal Minimal（炭灰）

---

### Step 4：生成 PPT

使用 `pptxgenjs` 从零生成 PPT，遵循 pptx-2-0.1.1 技能标准：

- 每页必须有视觉元素（图标/图片/图表）
- 标题 36-44pt，正文 14-16pt
- 配色方案统一，不混用
- 避免纯文字幻灯片

---

### Step 5：质量检查

1. 用 `python -m markitdown output.pptx` 检查内容完整性
2. 转成图片做视觉检查（soffice → PDF → pdftoppm）
3. 修复发现的问题
4. 输出最终文件下载链接

---

## 输出

- 生成的 `.pptx` 文件路径
- 文件下载链接（通过 `getfile` 工具）
- 内容摘要（文字版）

---

## 依赖

- `pptxgenjs`（生成 PPT）
- `markitdown[pptx]`（内容检查）
- LibreOffice + Poppler（视觉检查）
- 可选：`word-docx` 技能（如果内容来源是 .docx）

---

## 注意事项

- 生成前务必让用户确认摘要内容
- PPT 页数控制在 8-15 页（避免过长）
- 如内容过多，提示用户先筛选重点
- 尊重隐私：不将敏感聊天内容外传
