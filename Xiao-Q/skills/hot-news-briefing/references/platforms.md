# 热搜平台搜索策略

本文档记录各平台热搜的搜索方法和注意事项。

## 微博热搜

### 搜索关键词
- `微博热搜榜`
- `微博实时热搜`
- `微博热门话题`

### 搜索技巧
- 优先选择来自 `weibo.com` 官方域名的搜索结果
- 注意区分"微博热搜"和"微博话题"，优先热搜榜
- 热搜榜通常包含：排名、话题词、热度值

### 典型结构
```
1. 话题词1 - 热度值
2. 话题词2 - 热度值
...
```

## B站热门

### 搜索关键词
- `B站热门视频`
- `B站排行榜`
- `B站热门话题`

### 搜索技巧
- 优先选择来自 `bilibili.com` 官方域名的搜索结果
- B站热门包含：视频标题、UP主、播放量、弹幕数
- 注意区分"热门视频"和"热门话题"

### 典型结构
```
1. 视频标题 - UP主 - XXX万播放
2. 视频标题 - UP主 - XXX万播放
...
```

## 跨平台聚合策略

### 去重逻辑
1. 提取每条热搜的**核心关键词**
2. 计算不同平台话题的**文本相似度**
3. 相似度 > 70% 的视为同一事件

### 热度排序
综合考虑：
- 出现在几个平台（平台数 × 权重）
- 各平台的排名位置
- 话题的讨论量/热度值

### 优先级
1. **社会热点** - 涉及民生、政策、重大事件
2. **科技资讯** - AI、数码、互联网
3. **娱乐八卦** - 明星、影视、综艺
4. **其他** - 游戏、体育、生活

## 搜索结果解析

### 使用 web_search 工具
```python
# 示例：搜索微博热搜
web_search(query="微博热搜榜", count=5)
```

### 解析搜索结果
从 `web_search` 返回的 snippets 中提取：
- 话题/标题
- 热度/排名信息
- 来源平台确认

### 兜底方案
如果搜索结果不理想：
1. 尝试不同的搜索关键词
2. 降低 `count` 参数，提高精准度
3. 在 `web_fetch` 中直接访问热搜页面（如果有稳定URL）
