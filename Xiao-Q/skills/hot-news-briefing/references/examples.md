# 全网热搜早报 - 使用示例

本文档提供技能使用的典型场景和对话示例。

## 场景1：主动查看早报

**用户**: "看早报"

**助手执行流程**:
1. 调用 `web_search` 搜索三大平台热搜
2. 解析并聚合结果
3. 输出格式化的早报内容

**输出示例**:
```
📰 **全网热搜早报** - 2026年6月10日

---

### 1. 高考数学难度引热议
网友热议今年高考数学难度适中，多数考生表示发挥稳定
📍 来源：微博、知乎

### 2. AI新模型发布引发关注
某大厂发布最新多模态AI模型，性能提升显著
📍 来源：知乎、B站

### 3. 端午节旅游预订火爆
端午小长假临近，热门景点预订量同比增长50%
📍 来源：微博

### 4. 某某剧集收官引热议
热门剧集迎来大结局，网友讨论剧情走向
📍 来源：微博、B站

### 5. 新能源车企5月销量公布
多家新能源车企公布5月交付数据，市场格局生变
📍 来源：知乎

---

💡 今日关注：教育话题与社会热点并举，科技动态持续受关注
```

## 场景2：设置定时推送

**用户**: "每天早上8点给我推送早报"

**助手执行流程**:
1. 确认用户需求
2. 调用 `schedule-reminder` 技能创建定时任务
3. 返回设置成功提示

**输出示例**:
```
✅ 已设置每日早报推送

⏰ 时间：每天 08:00
📰 内容：全网热搜早报（微博+知乎+B站）
📱 推送方式：当前对话窗口

明天早上8点将自动为你推送热点早报～
```

## 场景3：语音播报

**用户**: "用语音播报今天的早报"

**助手执行流程**:
1. 生成早报文字内容
2. 调用 `tts` 工具转换为语音
3. 返回音频链接

**输出示例**:
```
🎧 正在为你生成语音早报...

[音频播放器]
早报内容已转换为语音，点击播放按钮收听 🔊
```

## 场景5：话题过滤

**用户**: "看早报，不要娱乐新闻"

**助手执行流程**:
1. 正常获取热搜数据
2. 在聚合时过滤掉娱乐相关话题
3. 输出过滤后的早报

**输出示例**:

```
---

### 1. 高考数学难度引热议
...

（仅显示非娱乐类热点）
```

## 场景6：历史对比

**用户**: "昨天有什么热点？"

**助手执行流程**:
1. 检查 `memory/2026-06-09.md` 是否存在
2. 如果存在，提取昨天的早报记录
3. 对比今天和昨天的热点变化

**输出示例**:
```
📅 **昨日热点回顾** (2026-06-09)

1. XXX事件
2. YYY话题
...

📊 **热点变化**:
- 新上榜：高考数学、AI新模型
- 持续热议：端午节旅游（连续2天）
- 已降温：某某明星八卦
```

## 错误处理示例

### 搜索失败

**情况**: 某个平台搜索失败

**输出示例**:
```
📰 **全网热搜早报** - 2026年6月10日

⚠️ 部分平台数据获取失败（知乎），以下为可用数据聚合结果：

1. ...
2. ...
```

### 全部失败

**情况**: 所有平台都搜索失败

**输出示例**:
```
❌ 暂时无法获取热搜数据

可能的原因：
- 网络连接问题
- 平台数据更新中

建议：请稍后再试，或检查网络连接 🔄
```

## 技巧和提示

### 提高搜索质量
- 使用更精确的关键词
- 调整 `web_search` 的 `count` 参数
- 关注搜索结果的时效性（`freshness` 参数）

### 个性化设置
用户可以设置：
- 推送时间（默认8:00）
- 排除的话题类型
- 是否包含语音播报
- 早报详略程度

### 数据缓存
- 不建议缓存热搜数据（时效性强）
- 可以缓存用户偏好设置
- 每日早报可保存到 `memory/` 供回顾

---

📌 **小贴士**: 越早设置定时推送，越早享受每日自动化早报服务！
