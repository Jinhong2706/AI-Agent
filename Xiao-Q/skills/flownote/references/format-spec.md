## 概述

FlowNote 使用纯 Markdown 文件存储笔记内容，辅以 `index.json` 提供快速检索能力。所有文件格式均需遵循本规范，确保智能体能正确读写和处理。

## 目录结构

```
notes/
├── ideas/                    # 灵感碎片（按周聚合）
│   ├── 2026-W13.md

│   ├── article-about-notes.md
│   └── ...
└── index.json               # 元数据索引
```

## Markdown 文件格式

### ideas/YYYY-WXX.md 格式

**用途**：存储灵感碎片，按周聚合到同一文件。

**格式模板**：

```markdown
# YYYY-WXX 灵感

## YYYY-MM-DD HH:MM {标题}
- {内容}
- 来源：{来源}（可选）
- 标签：#{标签1} #{标签2}
- 关联：[[{关联主题}]]（可选）
```

**字段说明**：

| 字段 | 必需 | 说明 |
|------|------|------|
| 标题 | 是 | 核心观点的简短描述，不超过 20 字 |
| 内容 | 是 | 灵感的详细描述 |
| 来源 | 否 | 信息来源（人名、书名、文章名等） |
| 标签 | 是 | 2-5 个相关标签，格式为 `#标签` |
| 关联 | 否 | 关联的主题文件名 |

**示例**：

```markdown
# 2026-W13 灵感

## 2026-04-01 14:32 关于笔记的本质
- 笔记应该是流动的生产资料，而不是静态的存档
- 来源：小强奇谭《我为何最终放弃了所有笔记软件》
- 标签：#笔记方法 #知识管理
- 关联：[[knowledge-management]]

## 2026-04-03 09:15 关于卡片笔记
- 核心不是分类，而是关联
- 来源：《卡片笔记写作法》
- 标签：#卡片笔记 #知识管理
- 关联：[[knowledge-management]]
```

### topics/{主题名}.md 格式

**用途**：存储主题卡片，一个话题一个文件。

**格式模板**：

```markdown
# {主题名}

## 核心观点

### {观点标题}
- {内容}
- 来源：[[YYYY-WXX#YYYY-MM-DD HH:MM]]

## 相关主题
- [[{相关主题1}]]
- [[{相关主题2}]]

## 待探索
- [ ] {待探索项1}
- [ ] {待探索项2}
```

**字段说明**：

| 章节 | 必需 | 说明 |
|------|------|------|
| 核心观点 | 是 | 该主题的核心观点列表 |
| 相关主题 | 否 | 与该主题相关的其他主题 |
| 待探索 | 否 | 需要进一步探索的问题 |

**示例**：

```markdown
# 知识管理

## 核心观点

### 笔记的本质
- 笔记是流动的生产资料，不是静态存档
- 来源：[[2026-W13#2026-04-01 14:32]]

### 卡片笔记法
- 核心不是分类，而是关联
- 来源：[[2026-W13#2026-04-03 09:15]]

## 相关主题
- [[ai-workflow]]
- [[note-taking-methods]]

## 待探索
- [ ] 尝试对话式记录的实际效果
- [ ] 研究更多关于知识关联的方法
```

### drafts/{草稿名}.md 格式

**用途**：存储正在发酵的半成品。

**格式模板**：

```markdown
# {标题}

*创建于 YYYY-MM-DD HH:MM*

{正文内容}

标签：#{标签1} #{标签2}

## 相关笔记
- [[{关联笔记1}]]
- [[{关联笔记2}]]
```

## index.json 数据结构

### 完整结构

```json
{
  "notes": [
    {
      "id": "2026-04-01-1432",
      "type": "idea",
      "title": "关于笔记的本质",
      "content": "笔记应该是流动的生产资料...",
      "file_path": "ideas/2026-W13.md",
      "tags": ["笔记方法", "知识管理"],
      "source": "小强奇谭",
      "created_at": "2026-04-01T14:32:00",
      "relations": ["knowledge-management"]
    }
  ],
  "topics": ["knowledge-management", "ai-workflow"],
  "stats": {
    "total_notes": 10,
    "total_topics": 3,
    "last_updated": "2026-04-03T10:00:00"
  }
}
```

### 字段说明

#### notes 数组

每条笔记包含以下字段：

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，格式：YYYY-MM-DD-HHMM |
| type | string | 是 | 笔记类型：idea/topic/draft |
| title | string | 是 | 笔记标题 |
| content | string | 是 | 内容摘要（最多 200 字符） |
| file_path | string | 是 | 文件相对路径 |
| tags | array | 是 | 标签数组 |
| source | string | 否 | 信息来源 |
| created_at | string | 是 | 创建时间（ISO 格式） |
| relations | array | 否 | 关联笔记 ID 列表 |

#### topics 数组

存储所有主题名称列表，便于快速检索。

#### stats 对象

统计信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| total_notes | integer | 笔记总数 |
| total_topics | integer | 主题总数 |
| last_updated | string | 最后更新时间 |

## 验证规则

### 文件命名验证

| 类型 | 规则 | 正确示例 | 错误示例 |
|------|------|---------|---------|
| ideas | YYYY-WXX.md | 2026-W13.md | 2026-week13.md |
| topics | 小写字母-连字符.md | knowledge-management.md | Knowledge-Management.md |
| drafts | YYYY-MM-DD-名称.md | 2026-04-01-my-draft.md | my draft.md |

### 时间戳验证

- 格式：`YYYY-MM-DD HH:MM`
- 示例：`2026-04-01 14:32`
- 正则：`^\d{4}-\d{2}-\d{2} \d{2}:\d{2}$`

### 标签格式验证

- 格式：`#标签内容`
- 允许字符：中文字符、英文字母、数字、连字符
- 长度：2-20 字符
- 示例：`#知识管理`, `#AI-workflow`

### 关联格式验证

- 格式：`[[文件名]]`
- 文件名不带扩展名
- 示例：`[[knowledge-management]]`

---

## 注意事项

1. **追加写入**：ideas 文件使用追加模式，不覆盖已有内容
2. **索引同步**：每次保存笔记后必须更新 index.json
3. **编码统一**：所有文件使用 UTF-8 编码
4. **换行符**：统一使用 LF（\n）
5. **空行规范**：章节之间保留一个空行
