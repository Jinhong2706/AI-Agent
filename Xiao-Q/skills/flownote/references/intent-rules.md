## 概述

意图识别是 FlowNote 的核心能力之一。智能体需要分析用户对话，判断用户意图并执行相应操作。本文档定义了意图类型、触发词和分类规则。

---

## 意图类型定义

### IntentType 枚举

| 意图类型 | 值 | 说明 | 后续操作 |
|---------|---|------|---------|
| NOTE | note | 记录笔记 | 提取内容并保存 |
| QUERY | query | 查询笔记 | 执行检索并呈现结果 |
| ORGANIZE | organize | 整理笔记 | 检索相关内容并整理 |
| NONE | none | 普通对话 | 正常对话回复 |

### NoteType 枚举（记录类型）

| 类型 | 值 | 说明 | 存储位置 |
|------|---|------|---------|
| IDEA | idea | 灵感碎片 | ideas/YYYY-WXX.md |
| TOPIC | topic | 主题卡片 | topics/{主题名}.md |
| DRAFT | draft | 半成品草稿 | drafts/{草稿名}.md |

## 分类决策规则

### 决策流程

```
用户输入
    │
    ├─► 匹配记录触发词 ──► 意图=NOTE
    │       │
    │       └─► 分析内容类型：
    │            ├─ 包含"主题"、"整理" ──► 类型=TOPIC
    │            ├─ 包含"草稿"、"文章" ──► 类型=DRAFT
    │            └─ 其他 ──► 类型=IDEA
    │
    ├─► 匹配查询触发词 ──► 意图=QUERY
    │
    ├─► 匹配整理触发词 ──► 意图=ORGANIZE
    │
    └─► 无匹配 ──► 意图=NONE
            │
            └─► 判断是否包含有价值洞察：
                 ├─ 是 ──► 询问是否记录
                 └─ 否 ──► 正常对话
```

### 内容类型判断规则

| 条件 | 类型 |
|------|------|
| 包含"主题"、"专题"、"关于XX" | topic |
| 包含"草稿"、"文章"、"写一篇" | draft |
| 包含"想法"、"灵感"、"观点" | idea |
| 默认 | idea |

### 标签提取规则

1. **从内容中提取关键词**：识别专有名词、概念术语
2. **参考已有标签**：查找 index.json 中的现有标签
3. **建议数量**：2-5 个标签
4. **格式规范**：中文短语，2-10 字符

### 来源识别规则

| 来源类型 | 识别模式 | 示例 |
|---------|---------|------|
| 人物 | "XX说"、"XX提到"、"看到XX" | 小强奇谭 |
| 书籍 | "《XX》"、"XX书中" | 卡片笔记写作法 |
| 文章 | "XX文章"、"看到一篇XX" | 如何高效记笔记 |
| 对话 | "我们讨论的"、"刚才说的" | 对话记录 |

## 边界条件处理

### 模糊意图

当无法明确判断意图时：

1. 优先判断为 `none`，正常对话
2. 如果内容明显有价值，提示用户是否记录
3. 不要强制记录，尊重用户意图

### 多意图

当一句话包含多个意图时：

```
用户：记一下这个观点，然后帮我查一下之前记的类似内容
```

**处理方式**：
1. 先执行记录操作
2. 再执行查询操作
3. 分步骤呈现结果

### 意图冲突

当触发词和内容类型不一致时：

```
用户：整理一下这个想法
```

**处理方式**：
- 触发词"整理"倾向 organize
- 但对象是"想法"（单条记录）
- 实际应按 note/idea 处理
- 根据上下文判断，或询问用户

---

## 注意事项

1. **上下文理解**：结合对话历史判断意图
2. **灵活匹配**：触发词列表持续更新，支持模糊匹配
3. **用户偏好**：记录用户的常见操作模式
4. **确认机制**：对于不确定的意图，主动询问确认
5. **失败恢复**：识别失败时，默认按普通对话处理
