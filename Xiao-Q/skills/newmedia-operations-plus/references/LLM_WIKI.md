# LLM Wiki 集成指南

基于 Andrej Karpathy 提出的 LLM Wiki 模式：将知识**编译**为持久化结构化维基页面，而非每次重新检索。核心价值是知识不断积累、交叉引用、持续完善。

---

## 安装步骤（强制）

```
第一步：检查 SkillHub CLI 是否已安装
运行：skillhub --version

若未安装，执行：
请先检查是否已安装 SkillHub 商店，若未安装，请根据 https://skillhub.cn/install/skillhub.md 安装 SkillHub 商店（只安装 CLI）

第二步：安装 llm-wiki 技能
skillhub install llm-wiki

第三步：验证安装
调用 llm-wiki 相关指令，确认可正常执行
```

---

## 三大核心操作

| 操作 | 时机 | 说明 |
|------|------|------|
| **Ingest（摄取）** | 每步完成后自动执行 | 将新数据/发现写入对应 wiki 页面，更新交叉引用 |
| **Query（查询）** | 每步开始前执行 | 基于 wiki 回答问题，有价值的结果写回 wiki |
| **Lint（检查）** | 每周执行一次 | 发现矛盾、补全缺失、整理结构 |

## 新媒体运营 Wiki 目录结构

```
wiki/
├── index.md                    # 总目录（LLM 定位入口）
├── log.md                      # 操作日志（时间线）
├── 行业分析/
│   ├── [行业名].md             # 各行业分析页（含关键词体系/用户画像）
│   └── 行业趋势汇总.md         # 跨行业规律总结
├── 竞品/
│   ├── [账号名].md             # 竞品账号详情页（含历次分析对比）
│   └── 竞品策略矩阵.md         # 跨竞品规律提炼
├── 内容策略/
│   ├── 爆款标题库.md           # 持续积累的标题公式（按行业分类）
│   ├── 违禁词库.md             # 各平台违禁词（持续更新）
│   ├── 内容模板库.md           # 种草/科普/招商等模板
│   └── 内容日历规律.md         # 节日热点与内容融合规律
├── 话术库/
│   ├── 互动钩子.md             # 视频/评论钩子（按行业分类）
│   ├── 私域引导话术.md         # 私信/欢迎语/唤醒话术
│   └── 高效话术案例.md         # 经验证效果好的话术
└── 客户档案/
    └── [品牌名].md             # 客户运营完整档案
```

## 各步骤 Wiki 操作指南

### Step 1 行业分析

**Query（开始前）**：
```
调用 llm-wiki Query：
"[行业名]的用户画像、关键词体系、市场规模是什么？"
参考页面：wiki/行业分析/[行业名].md
```

**Ingest（完成后，自动执行）**：
```
调用 llm-wiki Ingest，写入以下内容到 wiki/行业分析/[行业名].md：
- 本次分析日期和数据来源
- 市场规模/增长数据（更新）
- 用户画像（增量补充）
- 关键词体系（增量补充）
- 平台内容趋势（更新）
- 与上次分析的差异标注

同时更新 wiki/log.md，记录本次 Ingest 操作
```

---

### Step 2 竞品分析

**Query（开始前）**：
```
调用 llm-wiki Query：
"[竞品名]账号现在的内容策略、爆款特征和用户痛点是什么？"
参考页面：wiki/竞品/[账号名].md
```

**Ingest（完成后，自动执行）**：
```
调用 llm-wiki Ingest，写入 wiki/竞品/[账号名].md：
- 本次分析日期和基础数据（粉丝/播放）
- 近期爆款内容拆解（增量追加）
- 用户评论高频词（增量合并，标注新增）
- 内容策略变化（对比上次，标注变化点）
- 与其他竞品的对比（更新竞品策略矩阵）

同时更新 wiki/竞品/竞品策略矩阵.md，提炼跨账号规律
```

---

### Step 3 账号养号

**Query（开始前）**：
```
调用 llm-wiki Query：
"[平台]养号最有效的策略是什么？哪些行为对权重提升最显著？"
参考页面：wiki/行业分析/[行业名].md（内含平台运营规律）
```

**Ingest（完成后，自动执行）**：
```
调用 llm-wiki Ingest，追加到 wiki/客户档案/[品牌名].md：
- 账号基础资料设置记录
- 养号阶段和目标
- 每日养号计划（记录具体执行项）

若发现新的平台规律 → 写入 wiki/行业分析/[行业名].md 的平台运营规律章节
```

---

### Step 4 爆款内容创作

**Query（开始前）**：
```
调用 llm-wiki Query：
"[行业]最有效的爆款标题公式是什么？有哪些违禁词需要避免？"
参考页面：wiki/内容策略/爆款标题库.md、wiki/内容策略/违禁词库.md
```

**Ingest（完成后，自动执行）**：
```
调用 llm-wiki Ingest，更新以下页面：

1. wiki/内容策略/爆款标题库.md：
   - 追加本次提取的新标题公式（注明行业和来源）
   - 标注与已有公式的异同

2. wiki/内容策略/违禁词库.md：
   - 追加本次新发现的违禁词
   - 更新替换建议

3. wiki/内容策略/内容模板库.md：
   - 追加本次创作的优质模板

4. wiki/客户档案/[品牌名].md：
   - 更新内容策略章节（本次内容方向和主题）
```

---

### Step 5 互动钩子设计

**Query（开始前）**：
```
调用 llm-wiki Query：
"针对[行业]用户，哪类互动钩子效果最好？私域引导话术有哪些成功案例？"
参考页面：wiki/话术库/互动钩子.md、wiki/话术库/私域引导话术.md
```

**Ingest（完成后，自动执行）**：
```
调用 llm-wiki Ingest，更新以下页面：

1. wiki/话术库/互动钩子.md：
   - 追加本次生成的新钩子（注明行业/场景）
   - 标注哪些是从高互动数据中提炼的

2. wiki/话术库/私域引导话术.md：
   - 追加欢迎语/唤醒/转人工话术

3. wiki/客户档案/[品牌名].md：
   - 更新私域策略章节
```

## 周期性 Lint 操作

每周运行一次 Lint，检查：

```
调用 llm-wiki Lint，检查以下内容：
1. wiki/内容策略/爆款标题库.md - 是否有重复或矛盾的公式
2. wiki/竞品/ - 哪些竞品页面超过30天未更新（提醒重新分析）
3. wiki/内容策略/违禁词库.md - 是否有过时条目
4. wiki/客户档案/[品牌名].md - 是否有未完成的分析项
5. wiki/index.md - 目录是否完整，有无孤立页面
```

## llm-wiki vs ima-skill 分工

| 维度 | llm-wiki | ima-skill |
|------|----------|-----------|
| 写入时机 | **自动**，每步完成后必须执行 | **询问用户**，选择性保存 |
| 内容形式 | 结构化 markdown 页面，持续编译更新 | 笔记条目，便于快速检索 |
| 知识积累 | 增量编译，含交叉引用和矛盾标注 | 独立笔记，全文检索 |
| 最适合 | 需要持续沉淀的结构化知识（标题库/违禁词/竞品档案） | 一次性报告、快速检索的内容 |
| 互补关系 | wiki = 知识编译层 | ima = 快速检索层 |
