# 查询泛化原则与 5118 关键词挖掘指南

## 核心原则：每次查询必须泛化

**任何信息获取任务，都必须将原始查询词扩展为 3-5 个相似查询，全部执行，以确保数据覆盖全面。**

不允许只搜索单一查询词就结束。每个查询维度都要跑完。

---

## 5118 关键词挖掘工具

**网址**：https://www.5118.com/ci  
**作用**：挖掘长尾词、相关词、流量词、下拉词，支持跨平台（抖音/小红书/淘宝/京东/全网）关键词查询

### opencli 调用方式

```bash
# 打开 5118 关键词挖掘页面
opencli web read "https://www.5118.com/ci"

# 挖掘长尾词（全网）
opencli web read "https://www.5118.com/ci?keyword=[关键词]"

# 按平台挖掘（关键差异）
# 5118 支持切换：全网 / 抖音 / 小红书 / 百家号 / 京东 / 淘宝 / 天猫 / 拼多多
# 在页面上选择对应平台后搜索，或通过以下方式访问：
opencli web read "https://www.5118.com/seo/newwords/[keyword_hash]/"

# 查看最近暴涨词（发现热点）
opencli web read "https://www.5118.com/ci"
# → 查看页面中"最近暴涨词列表"部分

# 查看最近新词（发现新兴话题）
opencli web read "https://www.5118.com/ci"
# → 查看页面中"最近新词列表"部分
```

### 5118 各功能对应新媒体运营场景

| 5118 功能 | 适用场景 | 平台筛选建议 |
|-----------|----------|-------------|
| 挖长尾词 | 内容选题、标题关键词 | 抖音、小红书 |
| 挖相关词 | 扩展话题方向 | 全网 |
| 挖移动流量词 | 移动端用户搜索习惯 | 全网 |
| 挖全网下拉词 | 用户真实搜索行为 | 全网、淘宝 |
| 关键词动态趋势 | 判断关键词热度变化 | 全网 |
| 最近暴涨词 | 发现爆款话题 | 全网 |

## 查询泛化模板（按步骤）

### Step 1：行业分析 — 泛化查询组

以行业词「[品类]」为例，生成 5 个查询维度：

```
维度1 - 原词：           [品类]
维度2 - 推荐型：         [品类] 推荐 / [品类] 哪个好
维度3 - 人群细分型：     [人群特征] + [品类]（如：敏感肌护肤品）
维度4 - 问题/痛点型：    [品类] 怎么选 / [品类] 注意事项
维度5 - 榜单/趋势型：    [品类] 排行榜 / [品类] 2025 趋势
```

**执行要求**：以上 5 个查询词，分别在以下数据源各跑一遍：
- 5118（全网 + 抖音 + 小红书 三个平台维度）
- 小红书（`opencli xiaohongshu search`）
- 抖音（`opencli douyin videos --keyword`）
- 知乎（`opencli zhihu search`）
- 微博（`opencli weibo search`）

---

### Step 2：竞品分析 — 泛化查询组

以竞品「[品牌名]」为例：

```
维度1 - 品牌名：         [品牌名]
维度2 - 评价型：         [品牌名] 怎么样 / [品牌名] 值得买吗
维度3 - 对比型：         [品牌名] vs [同类品牌] / [品牌名] 替代品
维度4 - 差评/负面型：    [品牌名] 踩雷 / [品牌名] 差评 / [品牌名] 缺点
维度5 - 功能/场景型：    [品牌名] [核心功能] / [品牌名] 适合[人群]吗
```

**执行要求**：以上 5 个查询词，分别在：
- 5118（了解用户真实搜索习惯）
- 小红书（用户真实评价）
- 知乎（深度分析）
- 微博（实时舆情）
- 京东/淘宝（电商评价，`opencli jd item` / `opencli smzdm search`）

### Step 4：爆款内容 — 关键词挖掘泛化查询组

**阶段A：选题挖掘**（找内容方向）

```
维度1 - 行业大词：       [品类]
维度2 - 热点融合型：     [品类] + [当前热点/节日]
维度3 - 知识科普型：     [品类] 知识 / [品类] 原理 / [品类] 成分
维度4 - 避坑种草型：     [品类] 踩雷 / [品类] 避坑 / [品类] 种草
维度5 - 效果结果型：     用了[品类]后 / [品类] 多久见效 / [品类] 效果
```

**阶段B：标题关键词挖掘**（5118 重点使用）

```bash
# 在 5118 分别查询以下平台的长尾词：
opencli web read "https://www.5118.com/ci"
# 平台：抖音 → 搜索「[品类]」→ 记录高指数长尾词
# 平台：小红书 → 搜索「[品类]」→ 记录高指数长尾词
# 平台：全网 → 搜索「[品类]」→ 记录下拉词（用户真实输入习惯）
```

**执行要求**：以上查询词组，在以下来源各跑一遍：
- 5118（抖音维度 + 小红书维度 + 全网维度）
- 易撰（WebFetch `https://www.yizhuan5.com/`）
- 小红书爆款（`opencli xiaohongshu search`）
- 微博热搜（`opencli weibo hot`）
- 知乎（`opencli zhihu hot` / `opencli zhihu search`）
- B站（`opencli bilibili ranking`）
- 36氪（`opencli 36kr hot`）

## 全量执行清单（每步必须跑完）

执行任何步骤时，按此清单逐一执行，不得遗漏：

```
□ 5118 查询（全网维度）    - 3-5 个 query
□ 5118 查询（抖音维度）    - 3-5 个 query
□ 5118 查询（小红书维度）  - 3-5 个 query
□ 小红书搜索               - 3-5 个 query
□ 抖音搜索                 - 3-5 个 query
□ 微博热搜 / 搜索          - 行业词 + 2个变体
□ 知乎搜索                 - 痛点词 + 疑问词
□ 36氪 / B站               - 行业资讯 + 排行
□ 内置脚本（批量爬取）     - 补充大量数据
□ WebSearch / WebFetch     - 公开数据兜底
```

**执行标准**：每个数据源至少跑 3 个以上不同 query，合计覆盖 10+ 数据源、30+ query 组合。
