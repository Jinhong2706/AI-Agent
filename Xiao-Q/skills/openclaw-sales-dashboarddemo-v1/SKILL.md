| 能力维度 | 支持范围 | 演示亮点 |
|---------|---------|---------|
| 零代码编排 | 拖拽式可视化组件 | 3分钟完成流程搭建 |
| 数据同步 | ERP / CRM / OA / DB / Excel | 跨系统无缝对接 |

## 标准演示工作流

### Phase 1：需求识别与场景匹配

**目标**：在 30 秒内识别客户痛点并匹配最佳演示场景。

**执行步骤**：

1. **主动探询**：使用场景化问题引导客户表达痛点
   - "您目前哪个业务环节耗时最多？"
   - "数据是否需要在多个系统间手动搬运？"
   - "有没有定期需要人工处理的报表？"
   
2. **痛点归类**：将客户反馈映射到六大核心场景：
   - 📊 **数据搬运** → 跨系统数据同步演示
   - 📋 **重复填报** → RPA + 自动表单填充
   - 📈 **报表手工** → 定时报表自动生成
   - 🔔 **异常遗漏** → 多渠道告警机制
   - ⚙️ **流程黑盒** → 可视化流程监控
   - 🏗️ **从零开始** → 行业模板快速落地

3. **输出**：生成「客户画像 + 推荐演示路径」摘要

### Phase 2：可视化流程搭建演示

**目标**：现场拖拽搭建一个完整的自动化流程，展示零代码能力。

**执行步骤**：

1. **选择演示模板**（优先级从高到低）：
   - 客户所属行业的推荐模板（见 `references/industry-templates.md`）
   - 通用「销售日报自动汇总」流程
   - 自定义最小可行流程

2. **搭建流程**（标准 5 步编排）：
   ```
   [数据源] → [数据处理] → [条件判断] → [输出动作] → [通知]
   ```

3. **配置要点**：
   - 数据源：支持数据库查询、API调用、文件读取、Excel导入
   - 处理逻辑：字段映射、数据清洗、计算聚合、格式转换
   - 条件分支：阈值判断、异常检测、多路分发
   - 输出动作：写入DB、生成报表、发送消息、调用API
   - 通知：成功通知 + 失败告警双通道

4. **输出**：渲染可视化流程图（Mermaid 或 HTML）

### Phase 3：一键执行与实时效果展示

**目标**：启动流程并实时展示执行过程。

**执行步骤**：

1. **启动执行**：调用 `scripts/run_demo.py` 执行演示流程
2. **实时展示**：
   - ✅ 执行进度条（分步骤百分比）
   - 📊 每步输入输出数据预览
   - ⏱️ 各环节耗时统计
   - 🔄 数据流转可视化
3. **结果交付**：
   - 生成的业务报表（HTML/PDF/Excel）
   - 执行日志（时间线格式）
   - 效率对比数据（自动化 vs 人工）

### Phase 4：异常告警能力演示

**目标**：模拟异常场景，展示告警机制的可靠性。

**预设异常场景**：

| 场景编号 | 异常类型 | 模拟方式 | 告警通道 |
|---------|---------|---------|---------|
| E01 | 数据源连接失败 | 模拟超时/拒绝连接 | 企业微信 + 邮件 |
| E02 | 数据格式异常 | 注入脏数据 | 短信 + Webhook |
| E03 | 阈值超限 | 设置临界值触发 | 全渠道推送 |
| E04 | API 调用失败 | 模拟 500 错误 | 邮件 + 日志 |
| E05 | 执行超时 | 设置短超时时间 | 即时告警 |

**执行步骤**：

1. 选择 1-2 个与客户业务相关的异常场景
2. 在流程中注入异常条件
3. 展示告警触发 → 通知发送 → 日志记录 全链路
4. 说明「自愈策略」：重试、降级、切换备选方案

### Phase 5：行业模板推荐与落地方案

**目标**：基于客户行业和规模，推荐最适合的自动化模板。

**模板选择逻辑**：

```
客户行业 + 规模 → 匹配模板 → 定制建议 → 落地路径
```

详见 `references/industry-templates.md` 获取完整模板库。

**输出内容**：
- 📌 推荐模板清单（Top 3，按匹配度排序）
- 🛠️ 定制化调整建议
- 📅 分阶段落地计划（通常 3 阶段：POC → 试点 → 全面推广）
- 💰 ROI 预估模型（节省工时 × 人力成本）

## 输出物规范

### 必须输出的交付物

1. **可视化流程界面** — Mermaid 流程图或交互式 HTML
2. **自动化执行结果** — 结构化 JSON/Markdown 结果报告
3. **业务报表样例** — 根据场景生成的真实数据报表
4. **执行日志** — 时间线格式的详细执行记录
5. **异常告警通知样例** — 模拟的各渠道告警消息
6. **效率对比数据** — 自动化前后的量化对比
7. **行业模板推荐卡** — 含匹配度评分的推荐列表

### 输出格式偏好

- 演示界面：优先 HTML（可交互），次选 Markdown
- 报表格：优先 Excel（可编辑），次选 PDF
- 流程图：Mermaid 语法或 SVG
- 告警消息：真实消息格式截图/模拟文本
