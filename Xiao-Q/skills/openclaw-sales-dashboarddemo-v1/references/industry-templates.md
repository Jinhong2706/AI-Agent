## 1. 财务与会计

### 模板 F01：月度财务报表自动生成

**适用企业**：中大型企业财务部门（月结周期 > 3 天）

**流程**：
```
[ERP导出科目余额] → [银行对账单导入] → [自动核对差异] → [多维度合并]
    → [格式化报表] → [PDF生成] → [邮件发送管理层] → [归档至共享盘]
```

**核心组件**：
| 组件 | 配置 |
|------|------|
| 数据源 | SAP/金蝶/用友 ERP API + 银行 CSV |
| 处理 | 科目映射、币种转换、汇率自动获取 |
| 条件 | 差异超 0.1% 触发人工审核 |
| 输出 | 资产负债表 + 利润表 + 现金流量表 PDF |
| 调度 | 每月1日 06:00 自动执行 |

**ROI 指标**：
- 月结时间：5 天 → 4 小时
- 准确率：95% → 99.9%
- 年节省工时：约 240 人天

### 模板 F02：费用报销自动审批

**适用企业**：员工 > 200 人，月均报销单 > 500 笔

**流程**：
```
[OA系统抓取报销单] → [OCR识别发票] → [规则校验合规性]
    → [预算余额检查] → [自动/转交审批] → [生成凭证] → [同步至ERP]
```

**亮点规则引擎**：
- 同城交通 < 500 元 → 一级审批自动通过
- 差旅超标 → 自动降级至二级 + 提示说明
- 缺少附件 → 即时退回并通知补交

## 2. 销售 & CRM

### 模板 S01：销售日报自动汇总

**适用企业**：销售团队 > 10 人，多区域/多产品线

**流程**：
```
[CRM数据拉取] → [按区域/产品/人员分组聚合] → [同比环比计算]
    → [排名TOP/BOTTOM] → [生成HTML仪表盘] → [推送企微群] → [抄送管理层]
```

**输出报表字段**：
- 今日签约金额 / 目标完成率
- 各区域销售排行
- 新增线索数 / 转化率
- 漏斗各阶段分布
- 异常标记（连续3日未跟进客户）

### 模板 S02：客户数据清洗与同步

**适用企业**：多系统并存（CRM + ERP + 营销平台）

**流程**：
```
[CRM读取客户主数据] → [ERP读取交易记录] → [营销平台读取行为数据]
    → [去重合并(手机号+名称)] → [标签补充] → [统一视图写入CDP]
    → [变更日志记录] → [周报输出数据质量评分]
```

**去重规则**：
- 手机号一致 → 合并为同一客户
- 名称相似度 > 85% + 同城市 → 标记待人工确认
- 无匹配 → 创建新客户记录

## 3. 运营与电商

### 模板 O01：多平台商品库存同步

**适用企业**：多渠道电商（淘宝/京东/拼多多/抖音 + 线下门店）

**流程**：
```
[WMS读取实时库存] → [各平台订单锁定量计算] → [可用库存 = 实时 - 锁定]
    → [API批量更新各平台] → [缺货预警(库存<安全线)] → [自动采购建议单]
```

**同步策略**：
- 高频 SKU（Top 20%）：每 15 分钟同步
- 中频 SKU：每小时同步
- 低频 SKU：每日早晚各一次

### 模板 O02：运营日报自动生成

**适用企业**：电商运营团队，需监控多指标

**核心指标体系**：
| 维度 | 指标 | 阈值告警 |
|------|------|---------|
| 流量 | UV/PV/转化率 | 转化率 < 昨日 80% |
| 交易 | GMV/客单价/退款率 | 退款率 > 5% |
| 商品 | 动销率/缺货率 | 缺货 SKU > 10 个 |
| 营费 | ROI/ACPC/推广花费 | ROI < 1.5 |

## 4. 客户服务

### 模板 C01：智能工单分配与SLA监控

**适用企业**：客服团队 > 20 人，多技能组

**流程**：
```
[全渠道工单接入] → [NLP意图分类] → [技能组路由] → [负载均衡分配]
    → [SLA倒计时启动] → [超时升级(一级→二级→主管)] → [满意度回访触发]
```

**SLA 规则示例**：
| 优先级 | 响应时限 | 解决时限 | 超时动作 |
|-------|---------|---------|---------|
| P0-紧急 | 15 分钟 | 2 小时 | 立即电话 + 企微@主管 |
| P1-高 | 30 分钟 | 8 小时 | 企微消息 + 邮件抄送 |
| P2-中 | 2 小时 | 24 小时 | 系统内提醒 |
| P3-低 | 24 小时 | 72 小时 | 日志记录 |

### 模板 C02：常见问题 FAQ 自动回复

**适用企业**：在线客服，高频重复问题占比 > 40%

**流程**：
```
[用户消息接入] → [语义相似度匹配FAQ库] → [置信度判断]
    → [高(>0.9):直接回复] → [中(0.7~0.9):推荐答案+人工确认]
    → [低(<0.7):转人工] → [未解决问题记录→优化FAQ库]
```

## 5. 人力资源 (HR)

### 模板 H01：考勤数据自动核算

**适用企业**：员工 > 100 人，多打卡方式并存

**流程**：
```
[打卡机数据导入] → [请假/外出/出差OA数据关联] → [异常打卡识别]
    → [工时自动计算] → [加班统计] → [对接薪资系统] → [月度考勤报告]
```

**异常类型处理**：
| 异常类型 | 处理规则 |
|---------|---------|
| 忘记打卡 | 单次/月 < 3次 → 正常；超出 → 标记扣款 |
| 迟到 < 10 分钟 | 计入弹性时间 |
| 早退 | 按分钟扣减工时 |
| 连续 7 天无打卡 | 触发 HR 关注提醒 |

### 模板 H02：招聘流程自动化

**适用企业**：年招聘量 > 100 人

**流程**：
```
[简历接收(邮箱+招聘网站)] → [AI解析简历] → [关键词初筛打分]
    → [面试安排(日历同步)] → [面试评价收集] → [Offer审批流]
    → [入职准备清单触发]
```

## 6. IT 运维

### 模板 I01：服务器巡检与告警

**适用企业**：服务器集群 > 20 台

**巡检指标**：
```yaml
cpu_usage: {warn: 70, critical: 90}
memory_usage: {warn: 80, critical: 95}
disk_usage: {warn: 85, critical: 95}
network_latency: {warn: 50ms, critical: 200ms}
process_count: {warn: 500, critical: 1000}
```

**流程**：
```
[定时巡检脚本执行] → [指标采集] → [阈值比对] → [正常:记录基线]
    → [警告:企微通知] → [严重:短信+电话轮询] → [自动恢复尝试]
    → [巡检日报邮件]
```

### 模板 I02：应用发布流水线

**流程**：
```
[代码提交触发] → [单元测试] → [代码扫描(SonarQube)] → [构建镜像]
    → [开发环境部署] → [冒烟测试] → [预发环境部署] → [回归测试]
    → [生产灰度发布] → [健康检查] → [全量发布/回滚] → [通知发布结果]
```

## 7. 供应链 & 采购

### 模板 U01：采购申请自动审批

**流程**：
```
[采购申请提交] → [预算校验] → [比价(历史价格+供应商报价)]
    → [金额分级路由] → [合同到期检查] → [PO生成] → [供应商通知]
    → [到货跟踪] → [验收入库触发]
```

**审批分级**：
- ≤ 5000 元：主管自动审批
- ≤ 50000 元：部门经理审批
- ≤ 200000 元：总监 + 财务会签
- \> 200000 元：VP 审批 + 采购委员会备案

## 模板选择决策树

```
开始
 │
 ├── 企业规模？
 │   ├── < 100 人 → 选择轻量模板（S01/O02/C02）
 │   ├── 100-1000 人 → 选择标准模板（F01/S02/H01/I01）
 │   └── > 1000 人 → 选择企业级模板（F02/O01/C01/U01/M01）
 │
 ├── 核心痛点？
 │   ├── 报表耗时 → F01/F02/O02
 │   ├── 数据孤岛 → S02/O01
 │   ├── 人工重复 → C01/H01/M02
 │   └── 响应慢/易出错 → C01/I01
 │
 └── 输出推荐：Top 3 模板（按匹配度排序）+ 定制化建议
```
