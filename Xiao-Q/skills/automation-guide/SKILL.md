# Knowledge Reviewer - 通用知识沉淀工具

## 核心定位

这是一个**通用工具 Skill**，它的作用是：

1. **工具性**：提供知识回顾的方法论和模板
2. **通用性**：不包含任何个人数据，可被任何用户使用
3. **可配置**：用户根据自己需求创建回顾机制

## 工具能力

### 能力 1：生成回顾清单模板

根据用户选择的维度，生成对应的回顾清单：

**基础维度（通用）**：
- 基本信息更新
- 任务进展确认
- 偏好与习惯回顾
- 工具使用变化
- 清理过时信息

**职业适配维度**（根据用户职业动态添加）：

| 职业类型 | 适配维度 |
|---------|---------|
| 产品经理 | 需求管理、产品迭代、原型设计、PRD产出 |
| 开发者 | 项目代码、技术栈、开发规范、代码审查 |
| 设计师 | 设计系统、品牌规范、素材库、视觉迭代 |
| 销售 | 客户关系、商机状态、业绩目标、跟进记录 |
| HR | 团队成员、政策法规、招聘进度、员工关系 |
| 管理者 | 团队状态、OKR进度、决策记录、会议要点 |
| 学生 | 课程进度、学习计划、考试准备、知识巩固 |
| 创业者 | 业务进展、资金状况、团队协作、里程碑 |
| ... | 其他职业请告知，动态适配 |

### 能力 2：创建回顾自动化

当用户想要创建定期回顾自动化时：

**步骤 1：询问用户偏好**

```
📅 创建回顾自动化

请告诉我：
1. **回顾周期**：每天 / 每周 / 每月 / 自定义？
2. **回顾时间**：具体几点？（如：每天 21:00 / 每周五 18:00）
3. **回顾维度**：基础版 / 职业适配版 / 自定义组合？
4. **提醒方式**：微信推送 / 无需提醒？

输入示例："每周五18点，基础版，微信推送"
```

**步骤 2：解析并创建**

将用户输入映射为 rrule 规则并创建自动化：

| 周期 | rrule 格式 |
|------|-----------|
| 每天 9:00 | `FREQ=DAILY;BYHOUR=9;BYMINUTE=0` |
| 每天 18:00 | `FREQ=DAILY;BYHOUR=18;BYMINUTE=0` |
| 每周五 18:00 | `FREQ=WEEKLY;BYDAY=FR;BYHOUR=18;BYMINUTE=0` |
| 每周一 9:00 | `FREQ=WEEKLY;BYDAY=MO;BYHOUR=9;BYMINUTE=0` |
| 每月最后一天 | `FREQ=MONTHLY;BYMONTHDAY=-1;BYHOUR=20;BYMINUTE=0` |

**自动化 Prompt 模板**（自动读取工作区记忆文件）：

```markdown
执行知识回顾任务：

1. 读取工作区的记忆文件：
   - .workbuddy/memory/MEMORY.md（长期记忆）
   - .workbuddy/memory/YYYY-MM-DD.md（近期日记）

2. 分析并生成回顾清单，包含：
   - 本周期内新增的信息
   - 待确认或可能过时的信息
   - 建议清理的临时内容

3. 推送提醒用户确认

4. 根据用户回复更新记忆文件

只沉淀稳定且长期有价值的信息，不记录临时上下文。
```

### 能力 3：即时回顾

用户可以随时触发回顾：

```
/回顾 [周期/维度]
```

支持的格式：
- `/回顾` - 执行标准回顾
- `/回顾 简略` - 快速清单
- `/回顾 详细` - 深度分析
- `/回顾 [自定义维度]` - 按需组合

## 回顾清单格式

```markdown
# 📋 [周期] 知识回顾清单

## 📝 待确认信息
| 类型 | 内容 | 操作 |
|------|------|------|
| 用户画像 | xxx | 保留/修改/删除 |
| 任务偏好 | xxx | 保留/修改/删除 |
| 领域知识 | xxx | 保留/修改/删除 |
| 工具习惯 | xxx | 保留/修改/删除 |
| 项目状态 | xxx | 保留/修改/删除 |

## 🗑️ 建议清理
以下信息建议删除：
- [日期] xxx

## ➕ 可补充
建议添加：
- xxx

---
**回复格式**：保留 / 修改 / 删除 / 补充 [具体内容]
```

## 最佳实践原则

1. **工具性**：Skill 只提供方法，不存储个人数据
2. **用户主导**：回顾内容由用户提供，Skill 帮助整理
3. **隐私优先**：不记录敏感信息，尊重用户隐私
4. **灵活适配**：根据用户职业和需求动态调整维度
5. **增量沉淀**：每次回顾只沉淀有价值的变化，不重复记录

## 创建自动化时的配置

当用户创建自动化时，将以下信息记录到 MEMORY.md：

```markdown
## 自动化配置

- 知识回顾：[用户设置的名称]
  - 周期: [周期描述]
  - 时间: [具体时间]
  - 维度: [回顾维度]
  - 创建日期: [YYYY-MM-DD]
```
