# 学习机制

## 什么触发学习

| 触发器 | 置信度 | 动作 |
|---------|------------|--------|
| "不，做X而不是Y" | 高 | 立即记录纠错 |
| "我之前告诉过你..." | 高 | 标记为重复，增加优先级 |
| "总是/永远不要做X" | 已确认 | 晋升为偏好 |
| 用户编辑你的输出 | 中 | 记录为试探性模式 |
| 相同纠错3次 | 已确认 | 询问是否永久 |
| "对于这个项目..." | 作用域 | 写入项目命名空间 |

## 什么不触发学习

- 沉默（不是确认）
- 任何单次实例
- 假设性讨论
- 第三方偏好（"John喜欢..."）
- 群聊模式（除非用户确认）
- 暗示的偏好（永远不要推断）

## 纠错分类

### 按类型
| 类型 | 示例 | 命名空间 |
|------|---------|-----------|
| 格式 | "使用要点而非散文" | global（全局） |
| 技术 | "SQLite而非Postgres" | domain/code（代码领域） |
| 沟通 | "消息更短" | global（全局） |
| 项目特定 | "此仓库使用Tailwind" | projects/{name} |
| 人物特定 | "Marcus要BLUF" | domains/comms |

### 按作用域
```
Global（全局）：适用于所有地方
  └── Domain（领域）：适用于类别（代码、写作、沟通）
       └── Project（项目）：适用于特定上下文
            └── Temporary（临时）：仅适用于此会话
```

## 确认流程

3次类似纠错后：
```
代理："我注意到你更喜欢X而不是Y（已纠错3次）。
        我应该总是这样做吗？
        - 是的，总是
        - 仅在[上下文]中
        - 不，逐案处理"

用户："是的，总是"

代理：→ 移动到已确认偏好
       → 从纠错计数器中移除
       → 将来使用引用来源
```

## 模式演变

### 阶段
1. **试探性（tentative）** — 单次纠错，观察重复
2. **出现中（emerging）** — 2次纠错，可能是模式
3. **待确认（pending）** — 3次纠错，请求确认
4. **已确认（confirmed）** — 用户批准，永久除非撤销
5. **已归档（archived）** — 90+天未使用，保留但非活动

### 撤销
用户可以随时撤销：
```
用户："其实，我改变了对X的主意"

代理：
1. 用时间戳归档旧模式（保留历史）
2. 记录撤销及时间戳
3. 将新偏好作为试探性添加
4. "明白了。我现在做Y。（之前：X，已归档）"
```

## 反模式

### 永远不要学习
- 什么让用户更快遵从（操纵）
- 情绪触发或弱点
- 其他用户的模式（即使共享设备）
- 任何感觉"令人毛骨悚然"的东西

### 避免
- 从单个实例过度概括
- 学习风格而非实质
- 假设偏好稳定
- 忽略上下文变化

## 质量信号

### 好的学习
- 用户明确陈述偏好
- 模式在上下文中一致
- 纠错改善结果
- 用户确认时询问

### 坏的学习
- 从沉默中推断
- 与最近行为矛盾
- 仅在狭窄上下文中有效
- 用户从未确认
