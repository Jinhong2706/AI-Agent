# 热层记忆 — 模板

> 首次使用时将此结构复制到 `~/self-improving/memory.md`。
> 保持 ≤200 行（高频层级 ≤400 行）。
> 最常用的模式存储在这里。术语表参见 glossary.md。

```markdown
# 自我改进记忆（热层）

## 已确认偏好
<!-- 用户确认的模式，永不衰减 -->
<!-- 格式：- 偏好：描述（确认 YYYY-MM）-->

## 活动模式
<!-- 观察3+次的模式，可能衰减 -->
<!-- 格式：- 模式：描述（使用 Nx，最近 YYYY-MM-DD）-->

## 最近（最近7天）
<!-- 等待确认的新纠错 -->
<!-- 格式：- 待处理：描述（自 YYYY-MM-DD 起）-->

## 快速参考
- 术语表参见 glossary.md
- 溢出处理参见 corrections-pending.md
- 晋升条目引用来源："Promoted from corrections.md:date"
```

## 条目格式示例

```markdown
## 已确认偏好
- 缩进：2个空格（2026-01确认）
- 语气：直接，不含糊糊（2026-02确认）
- 导入：内部模块使用绝对路径（2026-03确认）

## 活动模式
- "looks good" = 批准继续（使用15次，最近2026-05-20）
- 单个emoji = 已确认（使用8次，最近2026-05-18）
-更喜欢pnpm而非npm（2026-05-15晋升，使用4次）

## 最近
- [待处理] 展示给用户前检查视觉间距（自2026-05-24起）
```

## 热层压缩策略

接近行限制时：
1. 将相似条目合并为通用规则
2. 示例："不要使用Tab" + "使用空格" → "缩进：使用空格，不用Tab"
3. 保留来源引用："（从3条纠错合并）"
4. 保持关键规则完整
