# 归档：YYYY — 模板

> 复制到 `~/self-improving/archive/YYYY.md` 用于年度归档。
> 包含衰减模式和非活动条目。

```markdown
# 归档：YYYY

归档日期：YYYY-12-31
条目数：N

## 摘要
- 晋升到热层：N
- 降级到温层：N
- 直接归档：N

## 晋升条目（本年度）
<!-- 此期间晋升到热层的条目 -->
- {原始条目} → 晋升到 memory.md

## 降级条目
<!-- 此期间从热层移动到温层的条目 -->
- {原始条目} → 降级到 {文件}

## 归档条目
<!-- 未晋升而归档的条目 -->
### 超时（90+天未使用）
- {条目}：{日期}归档

### 溢出
- {条目}：{日期}归档

### 用户请求
- {条目}：用户要求忘记 {日期}

## 项目归档
<!-- 归档的项目特定条目 -->
### projects/{name}.md（{YYYY-MM-DD}归档）
- {模式摘要}
```

## 单一条目的归档格式

```markdown
## 已归档：YYYY-MM-DD

### 原始条目
- **纠错：** "..."
- **上下文：** "..."
- **计数：** N
- **创建：** YYYY-MM-DD
- **最后使用：** YYYY-MM-DD

**原因：** timeout_without_use（超时未使用）| overflow（溢出）| user_forget（用户要求忘记）| superseded（已取代）

**来源文件：** {原始位置}
**晋升到：** {如果适用则为目标}
```

## 恢复

恢复归档条目：
1. 从归档复制条目
2. 添加到相应层级（corrections.md 或 memory.md）
3. 更新计数以反映历史
4. 从归档中移除或标记为"已恢复 YYYY-MM-DD"

术语表参见 glossary.md。
