# 领域：{domain-name} — 模板

> 复制到 `~/self-improving/domains/{domain-name}.md` 用于领域特定模式。
> 领域适用于工作类别（代码、写作、沟通等）。

```markdown
# 领域：{domain-name}

继承：global（全局）

## 范围
<!-- 此领域涵盖的内容 -->
- 编程语言：{列表}
- 文件类型：{列表}
- 适用项目：{范围描述}

## 模式
<!-- 跨项目适用的领域特定模式 -->
- {模式描述}

## 约定
<!-- 此领域的技术约定 -->
- {约定}

## 纠错
- 总数：N
- 最近：YYYY-MM-DD

## 相关领域
- {领域名}：{关系}
```

## 常见领域名称

| 领域 | 范围 |
|--------|-------|
| code（代码）| 编程、语法、架构 |
| writing（写作）| 文档、文章、邮件 |
| comms（沟通）| 沟通风格、回复格式 |
| git | 版本控制、提交、分支 |
| testing（测试）| 测试策略、覆盖率、框架 |
| security（安全）| 安全模式、凭据处理 |

## 继承

```
global（全局）(memory.md)
  └── domain（领域）(本文件)
       └── project（项目）(projects/{name}.md)
```

术语表参见 glossary.md。
