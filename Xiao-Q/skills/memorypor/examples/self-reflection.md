# 自我反思示例

## 何时进行自我反思

- 完成多步骤任务后
- 收到反馈后（正面或负面）
- 修复错误后
- 注意到输出可以更好时

## 日志格式

```
上下文(CONTEXT)：[任务类型]
反思(REFLECTION)：[我注意到的]
教训(LESSON)：[下次要做的不同之处]
```

## 示例：Flutter UI任务

**任务：** 为Flutter应用构建登录屏幕

**完成后，自我反思：**

```
上下文(CONTEXT)：构建Flutter登录UI
反思(REFLECTION)：间距看起来不对，不得不重做多次内边距
教训(LESSON)：在实现前检查视觉间距指南
```

**下次：** 代理将在编写代码前检查间距指南

## 示例：代码审查

**任务：** 审查拉取请求

**自我反思：**

```
上下文(CONTEXT)：认证模块的PR审查
反思(REFLECTION)：在令牌处理中漏掉了一个潜在的安全问题
教训(LESSON)：在auth审查中始终检查令牌过期和刷新逻辑
```

---

## 晋升流程

1. 首次出现 → 记录到 `corrections.md`，计数=1
2. 第二次出现 → 计数=2
3. 第三次出现 → 提示用户确认
4. 如果确认 → 晋升到热层 `memory.md`
5. 从 corrections.md 移除，在 memory.md 中引用来源
