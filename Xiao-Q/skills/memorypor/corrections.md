# 纠错日志 — 模板

> 首次使用技能时在 `~/self-improving/corrections.md` 创建此文件。
> 保留最近200条纠错（默认配置，约7-10天，高频使用下）。
> 旧条目评估晋升或移动到 `corrections-pending.md`。
> 有关溢出处理程序，参见 `corrections-pending.md`。

## 溢出处理

当 corrections.md 超过200条时：
1. **不要直接删除** 旧条目
2. 将条目移动到 `~/self-improving/corrections-pending.md`（待处理区）
3. 待处理区保留条目14天（观察期）
4. 待处理区中的条目仍累积计数（用于晋升跟踪）
5. 14天后未晋升 → 归档或丢弃

```
~/self-improving/
├── corrections.md         ← 主力缓冲区，200条
├── corrections-pending.md ← 待观察区，约100-300条，14天观察期
└── archive/              ← 最终归档
```

## 条目示例

```markdown
## 2026-02-19

### 14:32 — 代码风格
- **纠错：** "使用2个空格缩进，不是4个"
- **上下文：** 编辑TypeScript文件
- **计数：** 1（首次出现）

### 16:15 — 沟通
- **纠错：** "不要以'好问题！'开头回复"
- **上下文：** 聊天回复
- **计数：** 3 → **已晋升到 memory.md**

## 2026-02-18

### 09:00 — 项目：website
- **纠错：** "对于此项目，始终使用Tailwind"
- **上下文：** CSS讨论
- **动作：** 已添加到 projects/website.md
```

## 日志格式

每个条目包括：
- **时间戳** — 纠错发生时间
- **纠错** — 用户说的内容
- **上下文** — 触发它的场景
- **计数** — 多少次（用于晋升跟踪）
- **动作** — 存储在哪里（如果晋升）
