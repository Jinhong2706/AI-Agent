# 别在本地装 OpenClaw 了，我替你踩了所有的坑

上周末，一个做电商的朋友给我发消息："OpenClaw 到底怎么装？我搞了一下午，报了十几个错。"

我问他装的哪种方式。他说，按网上的教程，一行命令装到电脑上。

我当时就知道问题出在哪了。因为我之前也干过一模一样的事——在一台老 Windows 笔记本上折腾了将近三个小时，最终放弃了。

后来我花了两天时间，把市面上所有能装 OpenClaw 的方法挨个试了一遍。脚本安装、Docker、AI IDE 辅助、腾讯云、阿里云、扣子……每条路我都走了。

这篇东西，就是我踩完所有坑之后想告诉你的。

### 装了 6 种方式，我只想劝你别折腾

OpenClaw 的安装路子，粗分就两条：装在自己电脑上，或者让别人帮你跑。

装在自己电脑上，又叫"本地安装"，主要有三种玩法：

- **脚本安装**——复制一段命令到终端执行。适合有开发经验的人，翻译一下就是：不适合大多数人。
- **Docker 安装**——在你的电脑里起一个虚拟环境来跑。比脚本安全一点，但你得先搞懂 Docker 是什么。
- **AI IDE 辅助安装**——用 Cursor 之类的 AI 编辑器让 AI 带你装。听起来门槛低，但 AI 只能帮你写命令，环境对不上它也没辙。

云端部署就是另一回事了：腾讯云、阿里云，还有几个模型厂商的一键部署方案——KimiClaw、MiniMax，以及扣子。

我先本地装的。后来越装越怀疑人生，才转向云端。

### 本地安装的坑，比你想象的多

先说门槛。

安装过程中涉及 Python 环境、依赖包版本、系统权限配置，随便哪个环节出错，你就卡在那里了。

我自己的经历是这样的：那台老 Windows 装的是 Python 3.8，但 OpenClaw 要求 3.10 以上。好不容易升了 Python，又发现某个依赖包和系统不兼容。改了半天配置，终端里还是一堆红色报错。

三个小时，没装成。

我后来用 AI IDE 试了一次。Cursor 确实能帮你写安装命令，但它解决不了环境层面的事。你的 Python 版本跟要求不一致，你电脑上缺某个 C++ 编译环境，这些 AI 没辙，因为它看不到你电脑的真实状态。

**这不是智商问题，是环境问题。** 你不可能把所有人的电脑环境统一，这就是本地安装最要命的地方。

再说一个大多数教程不会告诉你的事：**安全风险。**

OpenClaw 装到你电脑上之后，它拿到的不是某个文件夹的权限，而是**整台电脑的操作权限。**

它能读你的文件，能执行系统命令，能往外面发网络请求。你可以理解为，你把一把万能钥匙交给了一个不太认识的人。

你电脑上存的那些合同、客户资料、财务报表，它都能访问。如果它执行了一条错误的命令，可能误删你的文件。更极端的情况下，敏感信息可能被传到外部服务器。

> 很多教程只告诉你"怎么装"，从来不提"装了之后意味着什么"。

除非你有独立的开发机器，或者你对 Docker 隔离有足够了解，否则本地安装这件事，我劝你慎重。

### 云端才是普通人的正确打开方式

想明白之后，我把注意力转向了云端部署。

云端的好处很简单：环境是服务商配好的，你不需要操心 Python 版本、依赖包这些东西。打开浏览器，点几个按钮，就完事了。

但云端也有好几个选项。腾讯云、阿里云是传统的云服务，配置项多，功能全，但对非技术用户来说，选项太多反而容易懵。

我最终选了扣子（Coze），两个原因。

**第一，便宜。** 扣子的 OpenClaw 部署成本在几个选项里算很有竞争力的。具体价格可以去官网看，但总的来说，对个人用户和小团队来说负担不大。

**第二，也是最打动我的——它能一键绑定飞书机器人。**

这个功能太关键了。接入飞书之后，OpenClaw 不再是一个你需要打开浏览器才能用的工具，它变成了你飞书里的一个"同事"。

你可以直接在飞书里给它发消息，让它帮你查资料、处理文件、整理数据。不用切换应用，不用额外打开网页，就在你日常办公的界面里完成交互。

对于用飞书办公的人来说，这个体验差距是实打实的。

### 5 分钟，从零到能用

如果你已经被我劝退了本地安装，想试试扣子，流程大概是这样的：

1. 打开扣子官网（coze.cn），找到一键部署 OpenClaw 的入口，点进去
2. 第一次用会跳到付款页面，付完后弹出配置窗口
3. 选模型（豆包或 Kimi 都行），版本选满级版
4. 授权创建飞书机器人，它会自动关联你的飞书账号
5. 打开飞书，搜索你起的机器人名字，发条消息
6. 飞书会给你一个配对码，拿回扣子页面提交，配对完成

正常操作的话，5 分钟以内能搞定。

整个过程不需要写一行代码，不需要打开终端，不需要懂什么是 Docker。你只需要会点鼠标。

### 比安装更重要的，是怎么用

文章写到这里，其实想说的核心就一件事：**别把时间花在安装上。**

OpenClaw 的安装方式有六七种，但最终你用的是它的能力，不是它的安装过程。花三个小时折腾本地安装，和花五分钟在云端搞定，后面你面对的是同一个 OpenClaw。

区别在于，一个让你精疲力竭还没用上，另一个让你五分钟后就开始干活。

还有一点值得提：OpenClaw 有记忆。你用得越多，它越了解你的习惯和偏好。一开始它可能回答得一般般，但用上两周，你会发现它开始知道你喜欢什么格式的报告、习惯怎么组织数据、甚至能猜到你下一步要问什么。

所以，与其纠结"怎么装"，不如先把这 5 分钟花出去，然后开始跟它磨合。

工具好不好用，不是看安装教程写得有多详细，而是看你打开它之后，第一件事能做什么。
