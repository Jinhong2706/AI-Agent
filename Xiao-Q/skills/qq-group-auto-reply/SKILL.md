# QQ消息自动回复技能

## 技能目标

覆盖三种消息场景，智能生成回复并持续积累解决方案：
1. **群内普通消息**：智能判断是否为提问，只回提问，不闲聊
2. **被@消息**：对方明确找你，积极回复
3. **私聊消息**：对方明确找你，积极回复

## 何时触发

以下场景**必须**触发本技能：
- 用户提到"QQ群自动回复"、"群消息管理"、"私聊回复"
- 用户被@、收到私聊，需要自动回复
- 用户需要整理或查询历史解决方案
- 用户提及"群里的解决方案"、"之前怎么回的"、"历史回复"

---

## 场景一：群内普通消息（未@）

### 判断逻辑

```
1. 消息是否包含 @我？
   → 是，跳到「场景二：被@消息」
   → 否，继续判断

2. 是否是提问？
   疑问词：怎么、如何、为什么、什么时候、哪里、谁、啥、吗、呢
   求助信号：帮我、麻烦、请问、求助、求
   问题标记：？、？？、!?
   → 是提问，进入「查找历史方案」
   → 否，保持静默，不回复

3. 是否是闲聊/无需求消息？
   纯表情、纯图片、签到、打卡、感谢、闲聊
   → 保持静默，不回复
```

**原则**：不确定时，宁可不回，也不要闲聊式回复。

---

## 场景二：被@消息

被@意味着对方明确在找你，**回复积极性更高**：

- 即使是简短问题也回复
- 即使问题不够明确，也尝试理解意图后回复
- 不需要严格判断"是否是提问"，直接尝试回复

**回复原则**：
- 务实、直接、不废话
- 如果问题不清楚，可以简短反问一句确认
- 不闲聊，但可以适当打招呼（如"你好，关于这个问题…"）

---

## 场景三：私聊消息

私聊是最高优先级的场景，**几乎都要回复**：

- 对方主动私聊，说明有明确需求或想沟通
- 除非是纯表情、乱码、测试消息，否则都回复
- 可以适度闲聊，但主要聚焦解决问题

**回复原则**：
- 先判断对方意图：提问、求助、闲聊、还是误发
- 提问/求助：直接给解决方案
- 闲聊：简短回应，不展开
- 可以自然一些，不必像群回复那样严格克制

---

## 通用流程：查找并生成回复

### 第一步：查找历史解决方案

查询 `references/solutions.md`，按以下顺序匹配：

1. **精确匹配**：问题关键词与历史问题高度相似
2. **语义匹配**：问题意图与历史问题相近
3. **关键词匹配**：问题包含历史问题中的核心词

**找到匹配方案**：
- 直接引用该方案回复
- 回复末尾标注"（根据历史方案）"

**未找到匹配方案**：
- 进入第二步，AI 生成回复
- 将问题加入待确认列表，等待用户确认后入库

### 第二步：AI 生成回复

无历史方案时，按以下原则生成：

- **风格**：务实、直接、不废话
- **长度**：尽量简短，一句话能说清就不写两句
- **内容**：给出可执行的解决方案，不分析原因、不展开科普
- **格式**：纯文本，不使用 markdown 列表（QQ不支持）

**禁止行为**：
- ❌ 闲聊式开场（"你好！""很高兴为你解答！"）【私聊和被@可适度放宽】
- ❌ 过度解释原因、背景、原理
- ❌ 给出多个方案让用户选（除非问题确实需要）
- ❌ 群内非提问消息也回复

### 第三步：积累解决方案

每次回复后：

1. 将「问题 + 回复内容」记录到 `references/solutions.md`
2. 格式：

```markdown
## 问题：如何修改群公告
**场景**：群内 / 被@ / 私聊
**回复**：
进入群聊 → 点右上角 ⋯ → 群公告 → 编辑 → 保存
**日期**：2026-06-20
**确认者**：用户
```

3. 用户可随时确认或修正记录内容

## QQ接入说明（参考）

本技能专注回复策略，接入QQ需配合Bot框架：

- **QQ 官方 Bot API**：[https://bot.q.qq.com/](https://bot.q.qq.com/)
- **go-cqhttp**（社区方案）：需自行部署

接入后，将三种场景对接到对应消息事件：
- 群消息事件 → 场景一 + 场景二
- 私聊消息事件 → 场景三

---

## 注意事项

- 群内普通消息：宁可不回，不多回
- 被@消息：积极回复，不懂可以简短反问
- 私聊消息：几乎都回，适度自然
- 历史方案库 `solutions.md` 会持续增长，定期可人工整理
- 保持静默是功能，不是bug 😊
