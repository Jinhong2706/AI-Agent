# 诸神皇冠风格 RPG 手游开发助手

基于**扣子低代码平台**的 RPG 手游开发辅助技能，覆盖从设计到实现的完整流程。

---

## 核心能力

### 1. 战斗系统设计
支持两种模式，可独立或混合使用：

#### 战棋模式（Grid Tactical）
- 六边形/方格地图生成（含地形效果：高地、森林、水域等）
- 单位移动范围计算（含地形消耗）
- 攻击范围/技能范围可视化描述
- 战棋战斗逻辑（回合、行动顺序、地形加成）

#### 回合制队列模式（Turn-Based Queue）
- 行动条/速度决定出手顺序
- 技能释放时机与打断机制
- 状态效果（buff/debuff）持续回合管理

#### 输出格式（扣子低代码）
```
- 数据表结构（JSON Schema）
- 逻辑组件描述（触发条件、动作、变量）
- 可视化流程说明（供扣子低代码拖拽配置参考）
```

---

### 2. 角色与养成系统

#### 角色生成
每次生成包含：
- 基础属性（力量/敏捷/智力/体质/幸运）
- 成长率（每级成长范围）
- 职业树（可转职路径，参考诸神皇冠的职业体系）
- 背景故事（可解析剧情脚本格式）

#### 职业系统设计
- 职业树结构（初级→中级→高级→特殊职业）
- 各职业技能列表
- 转职条件（等级、属性、道具）

#### 装备系统
- 武器/防具/饰品分类
- 装备属性与职业限制
- 强化/精炼系统描述
- 套装效果

#### 输出格式示例
```json
{
  "character": {
    "name": "艾莉丝",
    "class": "骑士",
    "level": 1,
    "stats": { "str": 12, "agi": 8, "int": 6, "vit": 10, "luk": 5 },
    "growth": { "str": [1,3], "agi": [0,2], "int": [0,1], "vit": [1,3], "luk": [0,1] },
    "background": "出身于边境骑士家族，父亲在一次边境冲突中失踪..."
  }
}
```

### 3. 深度剧情脚本系统

#### 剧情脚本格式（可解析）
采用自定义脚本语言，支持扣子低代码解析：

```
# 剧情节点定义
NODE: intro_001
TITLE: 边境的警报
CHARACTERS: 艾莉丝, 村长
BACKGROUND: 边境村庄，黄昏
MUSIC: bgm_village_evening

# 对话
SAY: 村长: "艾莉丝，边境哨站传来警报，你父亲留下的部队可能遭遇了袭击！"
SAY: 艾莉丝: "什么？！我这就去！"

# 分支选择
CHOICE:
  - TEXT: "立刻出发去哨站"
    GOTO: battle_001
    FLAGS: { "quest_started": "border_alert" }
  - TEXT: "先回城堡整备"
    GOTO: prepare_001
    FLAGS: { "quest_delayed": true }

# 条件触发
TRIGGER:
  - IF: has_item("父亲的项链") == true
    SAY: 艾莉丝: "这项链...是父亲留给我的，一定会保佑我。"
    FLAGS: { "morale_boost": 10 }
```

#### 剧情生成能力
- 主线剧情大纲（含章节结构）
- 支线任务（可独立可串联）
- 角色个人剧情（好感度触发）
- 随机事件（地图遭遇、NPC互动）

---

### 4. 扣子低代码组件配置

#### 数据表设计
为扣子低代码平台生成标准数据表结构：

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| Character | 角色基础数据 | id, name, class_id, base_stats, growth |
| Class | 职业定义 | id, name, parent_class, skill_tree |
| Skill | 技能数据 | id, name, type, power, range, effect |
| Equipment | 装备数据 | id, name, type, stats, class_limit |
| Quest | 任务数据 | id, title, type, conditions, rewards |
| Dialogue | 剧情节点 | id, node_id, character, text, choices |

#### 逻辑组件描述
为每个游戏系统生成扣子低代码逻辑组件说明：
- 事件触发器（战斗开始、角色升级、任务完成等）
- 动作列表（造成伤害、恢复生命、添加状态等）
- 变量管理（全局变量、角色变量、任务变量）

## 使用流程

### 快速开始
用户只需说：
> "帮我生成一个骑士角色"
> "设计一个战棋战斗场景"
> "写一段主线剧情第一章"

技能会自动判断需求类型，调用对应模块生成内容。

### 完整开发流程
1. **世界观设定** → 生成游戏背景、势力、种族
2. **战斗系统设计** → 选择模式，生成战斗规则
3. **角色系统** → 生成职业树、角色模板
4. **装备/技能系统** → 生成物品和技能数据
5. **剧情创作** → 生成主线+支线剧情脚本
6. **扣子低代码配置** → 生成数据表和逻辑组件说明
7. **迭代优化** → 根据用户反馈调整

## 输出规范

所有输出遵循以下原则：
- **结构化优先**：优先输出 JSON / 表格 / 脚本，便于扣子低代码直接导入
- **可解析性**：剧情脚本必须包含明确的节点ID、条件、跳转，不含歧义
- **模块化**：每个输出可独立使用，也可组合成完整系统
- **诸神皇冠风格**：职业体系、战斗节奏、剧情深度参考诸神皇冠的设计语言

## 注意事项

- 本技能专注于**诸神皇冠风格**的中世纪奇幻RPG，其他风格（科幻、现代等）需明确说明
- 扣子低代码配置说明以**描述性文字+结构化数据**为主，不直接生成扣子平台专有格式文件
- 剧情脚本格式为自定义格式，用户可根据需要调整为其他格式（Yarn、Ink等）
- 所有数值设计参考诸神皇冠的平衡性，避免过于极端
