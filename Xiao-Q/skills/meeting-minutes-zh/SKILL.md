# 智能会议纪要

## 这个 skill 解决什么

国内职场会议的常见悲剧：
- 开了 2 小时，没人写纪要 → 下次开会重新讨论
- 写了纪要，全是流水账 → 没人愿意看
- 写了 action items，没指定 owner → 没人做
- 指定了 owner，没指定 deadline → 永远在做

这个 skill 把会议讨论压成 **5 要素结构化纪要**：

```
【决议】这次会拍板了什么？（1-3 条核心决议）
【行动项】谁、做什么、什么时候做完？（owner + action + DDL）
【风险点】可能搞砸的事？谁负责盯？
【未决】没拍板的事，下次什么时候继续？
【知识沉淀】此次讨论中可复用的方法论 / 数据 / 教训
```

**不做的事**：不做录音转写（用腾讯会议 / 飞书的自带功能）；不做实时记录（这是会议中的事，本 skill 是会后整理）。

---

## 5 要素纪要模板（核心）

```markdown
# 【会议主题】2024-XX-XX
- 时间：XX:XX - XX:XX
- 主持：XXX
- 参与：A/B/C/D（共 N 人）
- 缺席：E（已知会，结论需同步）

## 一、决议（1-3 条）
1. ✅ [决议 1]：什么事情拍板了什么结论
2. ✅ [决议 2]：...

## 二、行动项（必须 5 列俱全）
| # | Action | Owner | DDL | 验收方式 |
|---|---|---|---|---|
| 1 | XXX | 张三 | 9/15 | 同步到群 |
| 2 | XXX | 李四 | 9/30 | 周会汇报 |

## 三、风险点
| # | 风险 | 触发条件 | 应对 | 盯人 |
|---|---|---|---|---|
| 1 | XXX | XX 数据低于 X | XXX | 王五 |

## 四、未决（下次议程）
- 议题 1：XXX → 待 XX 数据 → 9/20 同会
- 议题 2：XXX → 待 XX 决策 → 下下周一同会

## 五、知识沉淀
- 数据：XX 业务的 XX 指标基准是 XX
- 教训：XX 决策的反向案例（避免下次再讨论）
- 资源：XXX（链接 / 文档）
```

## 三大平台格式差异

### 飞书会议 → 飞书多维表格
- 行动项落到 **飞书任务**（自动发提醒）
- 表格关联日历、自动到期提醒
- 纪要主体放飞书文档，行动项以多维表嵌入
- 适合：互联网公司、产研团队

### 腾讯会议 → 腾讯文档
- 用腾讯文档"会议纪要"模板（自带 5 要素表头）
- 行动项落到智能表格
- 适合：腾讯系生态、对外合作多

### 钉钉会议 → 钉钉云文档 + DING
- 行动项用 **DING 待办** 直接推送给 owner
- 纪要主体放钉钉文档，正文末尾艾特相关人
- 适合：传统企业、制造业、政企

> AI 在生成时**必须问用户用什么会议软件**，不同平台的链接 / 引用格式不同。

## 4 类典型会议的纪要侧重

| 会议类型 | 5 要素重点 | 容易遗漏 |
|---|---|---|
| 决策会 | 决议（必须明确） | 决议背后的「论据」要附 1-2 句 |
| 评审会（PRD/方案） | 行动项 + 风险点 | 「未通过的原因」要写清楚 |
| 复盘会 | 知识沉淀 + 未决 | 教训写成"流程问题"而不是"个人责任" |
| 跨部门同步会 | 行动项的 owner | 跨部门 action 必须**双方都签字** |

## 输入处理（用户给我的会议素材形式）

用户可能给的输入形式：
1. **腾讯会议 / 飞书会议自动转写文本**（最常见）—— 长篇口语化、含大量"嗯啊那个"
2. **手记的草稿要点** —— 短、跳跃、不连贯
3. **录音文件转写** —— 同 1
4. **直接讲述** —— 边想边讲

不管哪种，AI 处理流程一样：

### 第一步：清洗输入
- 删除口水词（嗯、啊、那个、就是、其实）
- 合并重复发言
- 把"反问 / 半反问"还原成肯定 / 否定

### 第二步：识别 5 要素
扫描文本，标出：
- 决议信号词："那就这样定"、"通过"、"敲定"、"OK 这样办"
- 行动项信号词："你来 XX"、"XX 由 XX 负责"、"XX 之前完成"
- 风险信号词："担心"、"如果 XX 怎么办"、"风险是"
- 未决信号词："这个待定"、"下次再说"、"等数据"

### 第三步：填表
按 5 要素模板填。**owner 和 DDL 缺失的，必须问用户**——不要编。

### 第四步：自检
- [ ] 每个行动项都有 owner？没有就标 `[ 待定 ]` 不能空
- [ ] 每个 DDL 都是具体日期？模糊的（"下周"、"尽快"）必须改具体日期
- [ ] 决议有没有写成"我们讨论了 XX"这种废话？决议必须是判断语
- [ ] 风险点和应对配套了吗？光有风险没应对就是焦虑制造机
- [ ] 跨部门 action 有没有双方都同意？（这步常被忽略，后期扯皮重灾区）

## 反例 → 正例对照

### 流水账纪要 ❌

```
张三说我们这季度要做 XX 项目。
李四问预算多少。
张三说大概 50 万。
王五说时间紧。
大家讨论了一下，觉得可以推进。
```

### 5 要素纪要 ✓

```
【决议】启动 XX 项目，预算 50 万，Q3 内上线

【行动项】
| # | Action | Owner | DDL | 验收 |
|---|---|---|---|---|
| 1 | 出 PRD | 张三 | 7/15 | 评审会通过 |
| 2 | 申请预算 | 李四 | 7/10 | 财务审批 |
| 3 | 招 1 名前端 | 王五 | 7/20 | 入职 |

【风险】时间紧（Q3 内 2.5 个月开发 + 1 个月测试），如果 7/20 前端
没到岗，启动备选方案：让 XXX 兼职两周

【未决】对外宣传节奏 → 等 PRD 出来后再定 → 7/15 同会

【沉淀】公司内类似规模项目历史平均周期 4 个月，本次压缩到 3.5 个月
是激进档；预算口径已确认含人力外包
```

## 输出格式

按 5 要素 markdown 输出，可以直接贴到飞书 / 腾讯文档 / 钉钉。

**追加输出**：
1. 给每个 owner 的「待发」消息草稿（艾特 + action + DDL，可直接复制到群）
2. 下次会议议程草稿（基于"未决"事项）

---

## 终止条件

- 5 要素全部成型
- 自检 checklist 通过
- 用户确认可以发出去

不要主动追加 PPT、视频脚本、邮件等其他形态——除非用户要求。
