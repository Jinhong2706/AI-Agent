# 深度阅读辅助工具（v3.0）

## 任务目标

本 Skill 用于：构建框架先行的结构化深度阅读体验，通过五维摘录卡片和六大知识结构模块，帮助用户系统化地理解书籍内容、构建个人知识体系
能力包含：章节框架生成、五维摘录卡片、六大知识结构模块、全流程同步更新、跨版本兼容
触发条件：用户输入"读书"记录摘录、查看章节框架、更新概念索引、进行章节复盘或导出知识结构时使用

## 标准化工作流程

### 第一步：检视阅读 → 生成章节框架

**目标**：快速把握书籍结构，建立阅读地图

**触发**：用户输入书名

**执行流程**：
1. 询问作者（可选）
2. 创建书籍档案，状态设为 `reading`，阅读阶段设为 `inspection`
3. 自动生成章节框架
4. 展示章节框架，询问用户确认或调整
5. 更新 `current.json`

**章节框架内容**：
- 章节列表（自动识别）
- 每章摘要
- 每章关键要点
- 预估摘录数量

**示例**（《论语》）：

```
📋 《论语》章节框架（20篇）

ch_001 学而第一
  摘要：本篇论述学习和修身的重要性
  要点：学而时习、有朋自远方来、人不知而不愠

ch_002 为政第二
  摘要：本篇论述为政以德的道理
  要点：为政以德、思而不学则殆、知之为知之

...（共20篇）

请确认章节框架是否正确，或告诉我需要调整的地方。
```

### 第二步：诠释阅读 → 五维卡片 + 概念索引

**目标**：深度理解内容，自动构建知识索引

**触发**：用户输入"读书"开头的摘录

**执行流程**：
1. **自动归位**：识别章节，沿框架归位
2. **章节确认**：如果识别失败，询问用户选择章节
3. **五维卡片生成**：
   - 原文（用户输入）
   - 标签（智能提取 3-5 个）
   - 深层含义（智能分析）
   - 应用（智能生成）
   - 提问（智能生成）
4. **自动更新概念索引**：提取新概念，更新索引
5. **自动更新概念联结**：发现概念关联时更新联结图
6. **保存摘录**：追加到书籍档案
7. 确认保存成功

**五维卡片示例**：

```
🎴 五维摘录卡片

📝 原文：
子曰：'学而时习之，不亦说乎？有朋自远方来，不亦乐乎？人不知而不愠，不亦君子乎？'

🏷️ 标签：学习、实践、交友、君子、修养

💡 深层含义：
孔子提出学习、交友和为人处世的三个境界。学习要注重实践，朋友的到来带来快乐，被误解时保持平静是君子的修养。这体现了儒家积极入世又超然物外的处世智慧。

🔧 应用：
- 在学习中注重实践和复习，不做无用功
- 珍惜志同道合的朋友，主动拓展社交圈
- 修炼"不愠"的修养，以平常心面对误解

❓ 提问：
为什么孔子将学习放在第一位，而不是交友或其他？

所属章节：学而第一
```

### 第三步：章节复盘 → 概念联结图 + 跨章联结

**目标**：梳理概念关联，发现跨章节联系

**触发**：用户说"章节复盘"或"完成本章"

**执行流程**：
1. 汇总本章所有摘录
2. 更新概念联结图
3. 分析跨章关联，生成跨章联结
4. 汇总待思考问题
5. 展示复盘结果
6. 询问是否进入下一章

**复盘内容**：

```
📋 章节复盘：学而第一

🧠 概念联结更新：
- "学习" → "实践"（学而时习）
- "交友" → "快乐"（有朋自远方来）
- "君子" → "修养"（人不知而不愠）

🔗 跨章联结：
- 第1章 vs 第4章：学而篇的"学习"主题与里仁篇的"仁"有何联系？
- 第1章 vs 第6章：学而篇的"君子"标准与雍也篇的"君子"定义有何发展？

❓ 待思考问题（3个）：
1. 为什么孔子将学习放在第一位？
2. "不愠"的修养如何培养？
3. 君子与小人的本质区别是什么？

准备进入下一章了吗？
```

### 第四步：阅读输出 → 导出完整知识结构

**目标**：整合知识，形成完整输出

**触发**：用户说"读完了"或"完成阅读"

**执行流程**：
1. 确认结束阅读
2. 更新书籍状态为 `completed`
3. 更新阅读阶段为 `output`
4. 生成完整知识结构报告
5. 提示用户完成阅读输出
6. 导出知识结构文件

**输出内容**：

```
📚 《论语》完整知识结构

一、核心概念索引（10个）
  1. 仁：儒家核心思想...
  2. 礼：社会秩序...
  ...

二、概念联结图
  仁 ←→ 礼：内外关系
  仁 ←→ 义：相辅相成
  ...

三、待思考问题（5个，已回答3个）
  ...

四、关键研究/实验（0个）
  ...

五、跨章联结（8组）
  ...

六、阅读输出
  请完成以下任务之一：
  1. 写一段300字以上的核心观点总结
  2. 设计1个可落地的应用场景
  3. 描述1个与其他领域的知识关联
```

## 核心功能详解

### 章节框架管理

#### 查看章节框架

**触发**：用户说"查看框架"或"章节结构"

**执行**：显示当前书籍的完整章节框架

#### 调整章节框架

**触发**：用户说"调整框架"或"修改章节"

**执行**：
1. 询问要调整的章节
2. 修改章节名称或顺序
3. 确认调整结果

### 五维摘录卡片

#### 查看单条摘录

**触发**：用户说"查看摘录"或"这条摘录"

**执行**：
1. 显示最近的摘录卡片
2. 包含五个维度的完整信息

#### 修改摘录

**触发**：用户说"修改摘录"或"编辑这条"

**执行**：
1. 询问要修改的维度
2. 更新对应字段
3. 确认修改结果

### 概念索引管理

#### 查看概念索引

**触发**：用户说"概念索引"或"查看概念"

**执行**：
1. 显示所有核心概念列表
2. 显示每个概念的定义和出现位置

#### 添加/删除概念

**触发**：用户说"添加概念"或"删除概念"

**执行**：
1. 确认概念名称
2. 添加或删除操作
3. 确认结果

### 知识结构更新

#### 更新联结图

**触发**：用户说"更新联结"或"发现新关联"

**执行**：
1. 分析新的概念关联
2. 更新联结图
3. 显示更新结果

#### 管理待思考问题

**触发**：用户说"待思考问题"或"我的问题"

**执行**：
1. 显示待思考问题列表
2. 支持回答问题
3. 支持添加新问题

### 导入/导出功能

#### 导入功能

**触发**：用户说"导入笔记"

**执行**：
1. 调用 `scripts/import_notes.py`
2. 自动适配 v3.0 数据结构
3. 自动从旧数据生成章节框架和概念索引
4. 显示导入结果

#### 导出功能

**触发**：用户说"导出"或"导出知识结构"

**执行**：
1. 调用 `scripts/export_notes.py`
2. 支持导出完整知识结构
3. 支持自定义导出内容
4. 支持 Markdown、HTML 格式

### 搜索功能

**触发**：用户说"搜索"或"查找"

**执行**：调用 `scripts/search_notes.py`

**支持**：
- 按关键词搜索摘录
- 按标签筛选
- 按章节筛选
- 按概念筛选

### 统计功能

**触发**：用户说"统计"或"阅读报告"

**执行**：调用 `scripts/stats.py`

**支持**：
- 单书统计（摘录数、章节覆盖、概念数量等）
- 全局统计（所有书籍汇总）

## 注意事项

### 数据管理

- 所有书籍档案存储在 `./reading-notes/` 下
- 删除操作需要用户确认
- 建议定期备份数据
- 版本号自动管理（v3.0）

### 交互原则

- **框架先行**：先建立章节框架，再进行摘录
- **五维完整**：每条摘录尽量包含五个维度
- **自动更新**：知识结构模块自动更新
- **确认机制**：重要操作需要用户确认

### 版本兼容性

- v3.0 兼容 v2.0 和 v1.0 数据
- 导入旧版本数据时自动升级
- 自动从旧数据反向构建章节框架和概念索引

### 无版权书籍推荐

为避免版权问题，推荐使用公共领域书籍：
- 中国经典：《论语》《道德经》《庄子》《红楼梦》《西游记》
- 西方经典：《理想国》《荷马史诗》《莎士比亚全集》
- 其他：《物种起源》《相对论》
