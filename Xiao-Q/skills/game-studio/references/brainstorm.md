# 💡 游戏创意头脑风暴

## 概述

这是一个引导式的游戏概念创意流程，帮助你从零开始（或者从模糊想法）到一个完整的游戏概念文档。

**使用场景**：
- 你完全不知道做什么游戏
- 你有模糊的想法但需要具体化
- 你需要系统化地验证和打磨游戏概念

---

## 🎯 第一阶段：了解你

在开始设计游戏之前，我需要了解一些关于你的信息：

### 1.1 情感锚点
- 什么游戏时刻真正让你感动、兴奋或忘记时间？
- 是什么创造了那种感觉？
- 你在游戏中一直想要但从未找到的幻想或力量体验是什么？

### 1.2 品味档案
- 你玩得最多的 3 款游戏是什么？是什么让你一直玩下去？
- 你喜欢什么类型的游戏？不喜欢什么？为什么？
- 你更喜欢挑战你、让你放松、给你讲故事还是让你表达自己的游戏？

### 1.3 实际约束
- **你想做什么类型的体验？**
  - 挑战与成长
  - 故事与探索
  - 表达与创造
  - 放松与心流
  
- **你的开发时间线是什么？**
  - 几周
  - 几个月
  - 1-2 年
  - 多年

- **你的开发经验？**
  - 第一个游戏
  - 之前做过
  - 有专业背景

## 🎮 第二阶段：概念生成

基于你的反馈，我会生成 **3 个不同方向的概念**，每个概念包括：

| 字段 | 说明 |
|------|------|
| **标题** | 工作用名称 |
| **电梯演讲** | 1-2 句话，必须在 10 秒内让人理解 |
| **核心动词** | 玩家最常做的动作（建造、战斗、探索、解谜等） |
| **核心幻想** | 情感承诺 |
| **独特钩子** | "像 X，但同时 Y" |
| **目标 MDA 美学** | 主要情感（挑战、发现、表达等） |
| **估计规模** | 小/中/大 |
| **为什么能成** | 市场/受众匹配 |
| **最大风险** | 最难的未解答问题 |

### 概念生成技巧

1. **动词优先设计** - 从核心动词开始（建造、战斗、探索、解谜、创造、管理、发现）
2. **混搭方法** - 组合两个意外元素：[类型 A] + [主题 B]
3. **体验优先设计 (MDA 反向)** - 从目标情感出发，反向推到机制

**选择示例**：
- "像《星露谷物语》，但同时是宇宙恐怖"
- "像《杀戮尖塔》，但同时是恋爱模拟"
- "像《城市：天际线》，但同时是即时战斗"

## 🔄 第三阶段：核心循环设计

### 30 秒循环（即时体验）
- 主要操作的感受是什么？
- 这个操作内在满足吗？
- 什么让它感觉良好？（音效反馈、视觉反馈、时机感、战术深度）

### 5 分钟循环（短期目标）
- 是什么把即时游戏组织成循环？
- "再来一回合"/"再来一局"心理从哪里开始？
- 这个层面的玩家做什么选择？

### 会话循环（30-120 分钟）
- 完整的会话是什么样的？
- 自然停止点在哪里？
- 是什么让你不在玩的时候也想着游戏？

### 进步循环（数天/数周）
- 玩家如何成长？（力量？知识？选择？故事？）
- 长期目标是什么？游戏什么时候"完成"？

---

## 🏛️ 第四阶段：支柱与边界

### 游戏支柱（3-5 个）
每个支柱包含：
- **名称** + **一句话定义**
- **设计测试**："如果我们在 X 和 Y 之间争论，这个支柱说我们选择 __"

### 反支柱（3+ 个）
定义这个游戏 **不是** 什么：
- "我们不会做 [某事]，因为它会损害 [某个支柱]"

---

## 👥 第五阶段：玩家类型验证

使用 Bartle 分类和 Quantic Foundry 动机模型：

| 类型 | 描述 | 游戏如何服务 |
|------|------|-------------|
| **成就者** | 追求进度、收集、熟练度标志 | 明确目标、可衡量进度、可见里程碑 |
| **探索者** | 发现系统、隐藏内容、系统深度 | 好奇奖励、涌现互动、知识即力量 |
| **社交者** | 合作系统、共享体验、社交空间 | 互动理由、共享目标、社交认同 |
| **竞争者** | PvP 系统、排行榜、排名 | 公平竞争、技能表现、有意义赌注 |

---

## 📊 第六阶段：范围与可行性

### 目标平台
| 平台 | 引擎推荐 |
|------|---------|
| PC (Steam/Epic) | 任意 |
| 移动 (iOS/Android) | Unity, Godot |
| 主机 | Unreal, Unity |
| 网页/H5 | Phaser, Cocos, Godot |

### MVP 定义
- 什么是绝对最低构建，能测试"核心循环有趣吗"？
- 最大风险：技术风险、设计风险、市场风险
