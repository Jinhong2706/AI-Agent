# 🔍 代码审查与编程辅助

## 概述

这是一个专业的代码审查和编程辅助流程，帮助你提高代码质量、发现潜在问题、并获得专业的实现建议。

**使用场景**：
- 需要审查现有代码
- 遇到 bug 需要调试
- 需要实现新功能
- 需要代码重构建议
- 学习最佳实践

---

## 🎯 代码审查清单

### 1. 代码质量

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| **命名规范** | 变量、函数、类名是否清晰表达意图？ | 🔴 高 |
| **代码复杂度** | 函数是否过长？是否有嵌套过深？ | 🔴 高 |
| **重复代码** | 是否有可以提取复用的代码？ | 🟡 中 |
| **注释质量** | 注释是否解释"为什么"而非"是什么"？ | 🟡 中 |
| **代码风格** | 是否遵循项目或语言的风格规范？ | 🟢 低 |

### 2. 游戏特定检查

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| **硬编码值** | 魔法数字/字符串是否应该提取为常量？ | 🔴 高 |
| **Delta Time** | 物理/动画更新是否使用 delta time？ | 🔴 高 |
| **内存管理** | 是否有内存泄漏风险？ | 🔴 高 |
| **性能考虑** | 热路径是否有性能问题？ | 🔴 高 |
| **数据驱动** | 是否应该使用数据配置而非硬编码？ | 🟡 中 |

### 3. 架构设计

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| **单一职责** | 每个模块是否只做一件事？ | 🟡 中 |
| **松耦合** | 模块间依赖是否最小化？ | 🟡 中 |
| **可扩展性** | 添加新功能是否需要大量修改？ | 🟡 中 |
| **可测试性** | 代码是否易于单元测试？ | 🟡 中 |

### 4. 安全性

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| **输入验证** | 用户输入是否经过验证？ | 🔴 高 |
| **敏感数据** | 密码、API Key 等是否硬编码？ | 🔴 高 |
| **反作弊** | 游戏逻辑是否有被篡改的风险？ | 🟡 中 |

### 5. 错误处理

| 检查项 | 说明 | 优先级 |
|--------|------|--------|
| **异常处理** | 是否捕获并适当处理异常？ | 🟡 中 |
| **边界条件** | 边界情况（空数组、最大值等）是否处理？ | 🔴 高 |
| **日志记录** | 关键操作是否有日志？ | 🟢 低 |

## 🔧 常见问题与修复

### 性能问题

| 问题 | 检测 | 修复建议 |
|------|------|---------|
| 频繁 GC | 每帧分配新对象 | 对象池 |
| 不必要的查找 | 每帧 GetComponent | 缓存引用 |
| 字符串拼接 | 循环内拼接字符串 | StringBuilder |
| 协程泄漏 | 未正确停止协程 | OnDestroy 清理 |

### 游戏逻辑问题

| 问题 | 检测 | 修复建议 |
|------|------|---------|
| 硬编码数值 | 代码中有魔法数字 | 提取为配置常量 |
| 无帧率独立性 | 使用固定帧时间 | 使用 delta time |
| 状态机混乱 | 大量 if/else 状态 | 使用状态模式 |
| 配置数据硬编码 | 属性值写在代码里 | 使用 ScriptableObject |

### 代码组织问题

| 问题 | 检测 | 修复建议 |
|------|------|---------|
| 上帝类 | 单个类超过 500 行 | 拆分为多个类 |
| 循环依赖 | A 引用 B，B 引用 A | 引入中介者 |
| 霰弹修改 | 改一个功能要改很多文件 | 重构以提高内聚 |

## 🎮 引擎特定最佳实践

### Unity (C#)

```csharp
// ✅ 好的实践
[SerializeField] private int health;
public int Health => health;

// ❌ 避免
public int health;

// ✅ 使用对象池
public class ObjectPool<T> where T : new() {
    private readonly Stack<T> _pool = new();
    public T Get() => _pool.Count > 0 ? _pool.Pop() : new T();
    public void Return(T obj) => _pool.Push(obj);
}

// ✅ 帧率独立
void Update() {
    transform.position += velocity * Time.deltaTime;
}

// ❌ 避免
void Update() {
    transform.position += velocity; // 依赖帧率
}
```

### Godot (GDScript)

```gdscript
# ✅ 好的实践
extends Node

@export var speed: float = 100.0
var _velocity: Vector2 = Vector2.ZERO

func _physics_process(delta: float) -> void:
    _velocity = Vector2.ZERO
    if Input.is_action_pressed("move_right"):
        _velocity.x += 1
    _velocity = _velocity.normalized() * speed
    position += _velocity * delta

# ❌ 避免
var speed = 100  # 无类型声明
func _physics_process(delta):
    position.x += 5  # 硬编码速度
```

### JavaScript/TypeScript (H5)

```javascript
// ✅ 好的实践
class Game {
    constructor() {
        this.state = this.loadState();
        this.lastUpdate = performance.now();
    }
    
    update(currentTime) {
        const deltaTime = (currentTime - this.lastUpdate) / 1000;
        this.lastUpdate = currentTime;
        
        // 使用 deltaTime 进行更新
        this.player.x += this.player.speed * deltaTime;
    }
    
    loadState() {
        const saved = localStorage.getItem('gameState');
        return saved ? JSON.parse(saved) : this.getDefaultState();
    }
}

// ❌ 避免
let playerX = 0;
function update() {
    playerX += 5; // 硬编码速度，不使用 deltaTime
}
```
