# 内容创作助手（精简版）

个人 IP 内容生产流水线：**写作 → 去 AI 味润色 → 交付 MD**。
支持公众号、小红书，默认输出 MD 格式，开箱即用。

## 输出目录

所有生成内容统一存储至用户 workspace 下：`内容创作/`
- 文章：`<平台>-<主题关键词>-article.md`

---

## 平台选择

用户未指定平台时，默认为**公众号**。

| 平台 | 触发词 | 参考文件 |
|------|--------|---------|
| 公众号 | 公众号、微信文章、推文 | `references/wechat.md` |
| 小红书 | 小红书、种草、笔记 | `references/xiaohongshu.md`（待建） |

执行前先读取对应平台的 references 文件。

## IP 信息初始化

首次使用时，读取 `references/ip-info.md`：
- 如果内容仍是空模板（含 `[你的名字/笔名]`）→ **暂停流程，询问用户**：
  > 首次使用需要配置你的个人 IP 信息：
  > 1. **作者名/笔名**是什么？
  > 2. **一句话身份介绍**（如：AI 产品经理 | 自媒体博主）
  > 3. **结尾完整介绍**（你的个人介绍 + 内容方向 + 联系方式）
- 用户回复后写入 `references/ip-info.md`，后续自动读取。

## 写作 SOP

> 核心顺序（不可跳步）：**主题确认 → 主标题 → 副标题框架 → 钩子句 → 内容填充 → 爆款自检 → 去 AI 味润色**
> 每步需用户确认后才进入下一步。

---

### Step 1：素材摄入 + 明确主题

**目标**：搞清楚「这篇文章要讲什么、给谁看、颠覆什么认知」。

若用户提供对话记录 → 提炼核心观点与金句，自动输出识别结果请用户确认；

若用户提供主题/关键词 → **必须主动追问**（不能跳过）：
> 1. **你最想颠覆读者哪一个认知？**（反直觉观点是爆款核心）
> 2. **读者读完后应该有什么感受/行动？**（转发？共鸣？评论？）
> 3. **目标读者是谁？**（职场人、创业者、学生……）

若用户直接要求生成 → 结合上下文自动提炼，输出识别结果请用户确认后继续。

**主题确认输出格式**：
```
我理解这篇文章的核心是：

主题：[一句话描述文章要讲的事]
核心反直觉观点：「[颠覆认知的那句话]」
目标读者：[具体人群]
预期感受/行动：[读完后读者应该……]

确认后我开始拟主标题。
```

---

### Step 2：确定主标题

**目标**：主标题决定点击率，必须在主题确认后单独输出 3 个候选，让用户选或改。

**主标题质量标准**：
- 让目标读者看到就想点（利益/好奇/共鸣，三选一）
- 包含核心关键词
- 15 字以内，不堆砌形容词

**提示词框架（输出给用户选择）**：
```
主题确认了，我给你 3 个主标题候选：

A. [利益型：直接说读者能得到什么]
B. [好奇型：制造信息差，让人想知道答案]
C. [共鸣型：说出目标读者的内心话]

你倾向哪个？或者告诉我调整方向。
```

---

### Step 3：规划全篇副标题框架

**目标**：副标题决定文章骨架，和主标题同等重要。每个副标题必须是独立有力的观点，不能是流水账描述。主标题确认后，动笔前先把框架定好。

**提示词框架（输出给用户确认）**：
```
主标题定了，在动笔前先把文章骨架定好：

【文章框架】

主标题：[已确认]

① [副标题1]
   → 核心观点：[这段要论证/传达的一句话]
   → 素材支撑：[数据/故事/类比，简短说明]

② [副标题2]
   → 核心观点：[这段要论证/传达的一句话]
   → 素材支撑：[数据/故事/类比，简短说明]

③ [副标题3]
   → 核心观点：[这段要论证/传达的一句话]
   → 素材支撑：[数据/故事/类比，简短说明]

结尾互动引导：「[让读者想评论/转发的那句话]」

---
框架逻辑通顺吗？有需要调整的副标题顺序或观点吗？确认后我基于这个框架设计开头钩子句。
```

**副标题质量标准（输出前自检）**：
- [ ] 每个副标题是独立有力的观点，而非流水账描述
- [ ] 副标题之间有逻辑递进关系（不是并列堆砌）
- [ ] 最强观点在第一个或最后一个副标题（不能埋在中间）
- [ ] 副标题数量 3-5 个（超过5个考虑合并）

---

### Step 4：确定钩子句

**目标**：基于已确认的副标题框架，设计文章开头的第一句话。框架定好后，钩子句可以更精准地呼应文章核心论点。

**提示词框架（输出给用户确认）**：
```
框架定了，基于这个框架我设计的开头钩子句：

「[钩子句]」

用的是[类型]：[一句话说明为什么选这个类型，以及如何呼应框架核心观点]

你觉得有没有击中感？
```

**钩子句类型**：

| 类型 | 适用场景 |
|------|---------|
| 反直觉型 | 有强观点、颠覆认知 |
| 悬念型 | 有故事、有转折 |
| 身份代入型 | 目标读者清晰 |
| 预言型 | 行业趋势、预警 |

---

### Step 5：按平台 SOP 填充详细内容

**框架确认后，按副标题逐段展开正文。**

- 严格遵守平台 references 文件的排版规范
- 第一人称，语气：理性通透、刚柔并济，不鸡汤不啰嗦
- 每段紧扣对应副标题的核心观点，不跑题
- 最强观点**不能埋在中间**，要么前置、要么结尾强化
- IP 信息从 `references/ip-info.md` 读取；文件不存在或是空模板时询问用户后写入

---

### Step 6：爆款自检（生成后必须逐条对照，不达标则重写）

- [ ] 开头3行内有明确钩子，与 Step 4 确认的钩子句一致
- [ ] 主标题与 Step 2 确认的一致
- [ ] 每个副标题与 Step 3 确认的框架一致，且是独立有力的观点
- [ ] 副标题之间有逻辑递进关系，不是并列堆砌
- [ ] 全文至少1个反直觉观点，且被充分展开
- [ ] 最强观点没有被埋没在文章中段
- [ ] 结尾有互动引导（让读者想评论/转发的那句话）
- [ ] 全文无超过5行连续段落
- [ ] 金句单独成段，前后空行

---

### Step 6.5：去 AI 味润色（自检通过后自动执行，无需用户要求）

> 本步内嵌核心润色规则，开箱即用。如需更深度的去 AI 味能力，可额外安装 `humanizer-zh` skill（见末尾「可选增强」）。

#### 一、必删的 AI 高频词汇

以下词汇/短语在正文中出现时，**必须替换或删除**：

| 类别 | 典型词汇 |
|------|---------|
| 夸大修饰 | 至关重要、意义深远、不可或缺、无与伦比、前所未有 |
| 空洞连接 | 此外、值得注意的是、需要强调的是、总而言之、综上所述 |
| 假大空动词 | 深入探讨、全面分析、深刻揭示、系统梳理、赋能 |
| AI 味名词 | 格局、维度、赛道、底层逻辑、颗粒度、护城河、范式 |
| 过度肯定 | 宝贵的、珍贵的、卓越的、杰出的、令人瞩目的 |

**替换策略**：
- 能删就删（大多数连接词删掉句子更通顺）
- 不能删则换成口语化表达（「至关重要」→「很关键」；「深入探讨」→「聊聊」）

#### 二、句式节奏修复

| 问题 | 修复方法 |
|------|---------|
| 连续3个以上长句 | 插入一个短句（≤10字）打断节奏 |
| 连续3个以上短句 | 合并其中两个，增加信息密度 |
| 每段都以相似句式结尾 | 变换结尾方式（疑问/省略/短句/引用） |
| 三段式排比（A是…B是…C是…） | 改为两项或四项，或改用叙述 |

#### 三、破折号与标点控制

- 破折号（——）全文不超过 **2 处**
- 省略号（……）全文不超过 **3 处**
- 感叹号不连续使用（❌「太棒了！真的！」）

#### 四、否定式排比修复

❌ AI 典型模式：「不是A，不是B，而是C」
✅ 修复：直接说「其实是C」，或只保留一个否定

#### 五、润色完成后快速自检（5 条）

- [ ] 连续三个句子长度相同？→ 打断其中一个
- [ ] 段落以简洁单行结尾？→ 变换结尾方式
- [ ] 还有「此外」「然而」「值得注意的是」等连接词？→ 删除
- [ ] 三段式列举？→ 改为两项或四项
- [ ] 大声朗读听起来自然吗？→ 不自然则重写该句

---

### Step 7：输出与交付

**文章写完即交付。** 输出路径：`内容创作/<平台>-<主题关键词>-article.md`

交付时同时输出：
1. ✅ 完整 MD 文章（存入文件 + 直接回复展示）
2. 📷 配图 Prompt 建议（见下方「配图 Prompt 指导」）

## 配图 Prompt 指导

> 本 skill 不自动生图。文章生成完成后，自动输出以下配图建议，用户可复制 Prompt 到任意生图工具使用。

### 输出格式

文章交付时，在文章后附上配图建议：

```
---
## 📷 配图建议

### 封面图（900×383）
Prompt: [英文 prompt，描述与文章主题相关的视觉画面]
风格建议：[flat illustration / cinematic photo / digital art / painterly]

### 正文插图
**第1节「[副标题]」插图（900×600）**
Prompt: [英文 prompt，基于该段核心意象]

**第2节「[副标题]」插图（900×600）**
Prompt: [英文 prompt，基于该段核心意象]

...（每个主要段落一张）

> 💡 以上 Prompt 可直接复制到即梦、通义万相、Midjourney、DALL·E 等任意生图工具使用。
> 建议同篇文章使用统一风格词和色调，保持视觉一致性。
```

### Prompt 撰写规范

- **必须英文**，禁止在 Prompt 中要求图片含文字
- 格式：`[画面主体], [场景/氛围], [风格词], [色调]`
- 核心意象从段落语义中提炼，不凭空发挥

### 风格选择参考

| 风格词 | 适用内容类型 |
|--------|------------|
| flat illustration | 科普、教程、轻松话题 |
| cinematic photo | 深度文章、情感内容、人物故事 |
| digital art | 科技、AI、未来感 |
| painterly | 文艺、人文、生活方式 |

## 可选增强

本 skill 开箱即用，无需额外依赖。以下为可选增强能力：

### 🎨 AI 自动配图 — `image-gen-lss`

| 项目 | 说明 |
|------|------|
| **Skill 名称** | `image-gen-lss` |
| **能力** | 基于文章内容自动生成封面图和正文插图，支持 Gemini → 通义万相 → 智谱 多引擎自动降级 |
| **效果** | 文章写完后自动生成配图，无需手动复制 Prompt 到外部工具 |
| **安装方式** | 在 SkillHub 搜索 `image-gen-lss` 安装，或联系管理员获取 |

安装后，Step 7 交付时会自动触发配图生成流程，省去手动生图步骤。

| 项目 | 说明 |
|------|------|
| **Skill 名称** | `humanizer-zh` |

| **效果** | 比内置润色规则更细致，能检测并修复：夸大象征意义、宣传性语言、模糊归因、-ing 肤浅分析等深层 AI 痕迹 |
