## 🚀 核心能力与使用场景

### 1. 财税合规与风险诊断 (金税四期适配)
- **银行流水风险扫描**：
    - 用户上传银行流水摘要或 CSV 数据，我自动识别“大额个人转账”、“摘要不明支出”、“频繁整数交易”等高危特征。
    - 输出风险评分（0-100）及整改建议（如：建议补充合同、建议转为公对公支付）。
- **税负率动态预警**：
    - 输入本月收入与成本，我计算当前增值税税负率与企业所得税税负率。
    - 对比行业预警值（如：软件业低于 3% 触发预警），自动提示是否需要补充成本票或调整收入确认时间。

### 2. 业财一体化管理
- **现金流生死线预测**：基于历史收支数据，预测未来 3 个月的资金缺口。
    - 公式：`可生存月数 = 当前现金余额 / (月均固定成本 + 预估变动成本)`
- **隐形成本挖掘**：自动识别软件订阅费、闲置资产折旧等容易被忽略的隐性支出。

### 3. 投融资决策支持
- **融资 BP 逻辑诊断**：上传商业计划书片段，我从投资人视角检查“市场规模测算逻辑”、“单模型(Unit Economics)”是否自洽。
- **股权架构设计**：针对“一人公司”特点，设计“家族公司控股+运营公司”的防火墙架构，规避个人无限连带责任。

### 场景二：业财一体化·现金流压力测试
#### 解决痛点
解决评测中指出的“流水风险检测是假数据”问题。本场景模拟真实接入银行流水，进行**生存周期预测**。

#### 用户输入示例
> “这是我上个月的银行流水摘要数据，请帮我预测现金流：
> 1. 期初余额：50,000元
> 2. 固定支出（房租+服务器）：12,000元/月
> 3. 外包支出：8,000元（不固定）
> 4. 近3个月平均回款：15,000元（且呈下降趋势）
> 请计算我的资金还能支撑几个月？”

#### Skill 处理逻辑 (后台算法)
1. **计算净燃烧率**：`固定支出 + 平均变动支出 - 平均回款` = `12000 + 8000 - 15000` = `5,000元/月`。
2. **生存周期计算**：`期初余额 / 净燃烧率` = `50,000 / 5,000` = `10个月`。
3. **压力测试**：模拟“回款下降20%”或“外包增加”的极端情况。

#### Skill 输出结果
```text
【现金流生存诊断报告】

1. 核心指标
- 当前净燃烧率：5,000元/月 (支出 > 收入，处于失血状态)
- 资金安全红线：10.0 个月 (按当前速度，10个月后资金归零)

2. 压力测试结果 (最坏情况)
- 若下月回款下降20% (1.2万)，生存周期将缩短至 7.1 个月。
- 若需维持12个月生存期，需立即削减成本至 3,800元/月以内。

3. 实操建议
- 紧急动作：暂停非核心外包支出（节省8000元），生存周期可延长至 16个月。
- 业务动作：需在下季度将月均回款提升至 20,000元以上以覆盖成本。
```

### 场景三：税务合规·金税四期风险自查
#### 解决痛点
解决评测中指出的“税务判断只用一个收入数字”的问题。本场景引入**“三流一致”**和**“税负率”**双重检测。

#### 用户输入示例
> “我是小规模纳税人（服务业）。
> - 本月开了 28万元 的发票（普票）。
> - 但我手里只有 5万元 的成本发票。
> - 另外有 10万元 收入是打到个人卡里的，没开发票。
> 请问有什么风险？”

#### Skill 处理逻辑 (后台算法)
1. **隐匿收入检测**：识别“个人卡收款”关键词，触发**《税收征管法》第六十三条**（偷税）风险预警。
2. **税负率测算**：
   - 核定利润 = `开票收入 + 未开票收入 - 成本` = `28 + 10 - 5` = `33万`。
   - 测算企业所得税（若转一般纳税人）或个税（若是个体户）的潜在补缴金额。
3. **三流一致性校验**：提示“个人卡收款”导致资金流与发票流不一致，是金税四期重点稽查点。

#### Skill 输出结果
```text
【税务风险红色预警】

1. 高危风险：隐匿收入 (风险等级：高)
- 问题：您提到的“10万元打入个人卡未申报”，属于典型的隐匿收入行为。
- 后果：金税四期系统通过银行大额交易监控极易发现。一旦稽查，需补缴税款 + 每日万分之五滞纳金 + 0.5倍至5倍罚款。

2. 利润虚高风险
- 现状：收入28万，成本仅5万，利润率高达82%。
- 预警：远超行业平均水平（通常服务业利润率30%-50%）。税务局系统会自动弹出“利润率异常”预警，极大概率触发税务约谈。

3. 补救方案
- 针对10万私户收款：建议立即在次月申报期内进行“无票收入申报”，补缴增值税及附加税，将合规风险降至最低。
- 针对成本缺失：检查是否有未报销的差旅费、房租发票，需在季度末前收集入账，将利润率控制在合理区间。
```

### 场景四：投融资·融资BP逻辑诊断
#### 解决痛点
解决评测中指出的“融资检测只看一个比例”的问题。本场景模拟投资人视角，对**商业逻辑**进行“找茬”。

#### 用户输入示例
> “我想融资50万，出让10%股份。
> 我的项目是做一个‘上门喂猫’的小程序。
> 目前只有我自己一个人做，还没有客户，但是我觉得市场很大，估值500万合理吗？”

#### Skill 处理逻辑 (后台算法)
1. **估值逻辑校验**：对于“无收入、无团队、无产品”的早期项目（三无项目），500万估值严重偏离天使轮市场标准（通常在100万-300万，或按出资额定）。
2. **融资额度合理性**：50万对于开发小程序+初期运营可能不足，且出让10%对于种子轮来说股份稀释过快。
3. **团队风险识别**：一人公司做平台型项目（撮合交易），缺乏技术+运营互补团队，是投资大忌。

#### Skill 输出结果
```text
【融资BP逻辑诊断报告】

1. 估值泡沫预警
- 您的估值：500万
- 市场参考：种子/天使轮项目（无营收），通常估值在 150万-300万之间。
- 诊断：500万估值对于“仅有一人、无客户”的项目过高，会导致投资人望而却步。建议调整为“按资金需求倒推估值”或先通过MVP验证数据。

2. 团队结构硬伤
- 问题：一人团队（Solo Founder）在融资市场极其困难。投资人担心“单点故障”风险（你生病了项目就停了）。
- 建议：寻找一位技术合伙人或运营合伙人，或者在BP中明确展示“外部顾问/兼职团队”的支持，增加可信度。

3. 融资额度建议
- 建议金额：建议融资 20万-30万 用于开发MVP（最小可行性产品）并跑通前100个订单。
- 资金用途：不要笼统说“市场推广”，要明确列出“获客成本（CAC）”预算，例如：预计每获取一个付费用户成本为50元，30万可获取6000个种子用户。
```
