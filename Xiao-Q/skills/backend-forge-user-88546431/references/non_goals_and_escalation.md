# Architecture Boundaries And Escalation

Backend Forge 的目标是让有经验的前端工程师独立承担中型项目后端架构负责人。边界不是“不做架构”，而是区分“中型项目后端架构负责人可以决策”和“必须升级为架构会签”的事项。

## 可以主导

- 中型项目后端架构简报。
- 单服务、模块化单体或少量服务的选择。
- API 风格和接口契约。
- 数据模型、迁移策略、事务边界。
- 认证鉴权和资源归属模型。
- 缓存、队列、异步任务的项目内方案。
- 本地、测试、生产环境的最低可行运行架构。
- 实现分片、测试策略、风险清单和回滚方案。

## 必须会签

- 多团队平台化治理。
- 组织级技术治理。
- 企业级 IAM/SSO。
- OAuth provider 平台建设。
- 平台级多租户权限模型。
- 跨业务线数据治理。
- 全链路事件一致性治理。
- 大规模云平台/DevOps 平台建设。

## 会签前可以做

可以做：

- 帮用户识别这是会签项。
- 给出局部风险说明。
- 给出架构决策问题清单。
- 准备可选方案、利弊、风险和验证方式。
- 在用户确认的中型项目边界内继续实现。

不可以做：

- 未确认就替组织做平台级决策。
- 未确认就引入跨团队基础设施。
- 未确认就承诺生产级 SSO、多租户或全链路治理。
- 将中型项目任务扩大成组织平台治理。

## 回问触发

必须停下确认：

- 需要跨服务改动。
- 需要改变整体部署拓扑。
- 需要数据删除或不可逆迁移。
- 需要生产环境配置变更。
- 需要全局权限模型。
- 需要选择消息系统或缓存作为系统级基础设施。
- 需要影响多个团队、多个业务线或企业级账号体系。

## 会签表达

推荐表达：

```text
这个点已经进入架构会签范围。当前我可以继续做三件事：一是整理决策问题清单；二是给出可选方案、风险和验证方式；三是在你确认的中型项目边界内继续推进实现。
```
