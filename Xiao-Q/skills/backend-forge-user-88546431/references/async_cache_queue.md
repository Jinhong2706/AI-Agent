# Async, Cache, Queue

异步任务、缓存、消息队列属于高级工程能力模块，按需触发，不进入默认项目骨架。

## 异步任务

Spring Boot：

- 简单后台任务可用 `@Async`。
- 定时任务可用 scheduling。
- 需要持久化、重试、可观测性时，不能只用内存线程池草率处理。

FastAPI：

- 简单请求后任务可用 `BackgroundTasks`。
- 复杂任务优先考虑 Celery/RQ 等任务队列。

必须确认：

- 任务失败如何处理。
- 是否需要重试。
- 是否需要幂等。
- 用户是否需要看到任务状态。

## 缓存

可选方案：

- Spring Cache。
- Redis。
- Python Redis 客户端。

必须确认：

- 缓存 key。
- TTL。
- 失效时机。
- 写后是否更新或删除缓存。
- 是否允许短暂脏读。

测试必须覆盖：

- 命中缓存。
- 缓存未命中。
- 数据变更后的失效。
- Redis 不可用时的降级，若项目需要。

## 消息队列

可选方案：

- RabbitMQ。
- Kafka。
- Celery broker。

必须确认：

- 消息语义。
- 生产者和消费者边界。
- 重试策略。
- 死信或失败处理。
- 幂等消费。
- 消息顺序是否重要。

## 架构会签信号

发现以下情况时必须停下确认：

- 跨服务事件一致性。
- 复杂分布式事务。
- 全链路消息治理。
- 多服务拓扑重构。
- 组织级事件规范。
