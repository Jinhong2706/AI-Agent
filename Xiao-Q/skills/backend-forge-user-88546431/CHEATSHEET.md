# Backend Forge Cheatsheet

## 触发词

| 入口 | 用途 |
|---|---|
| `bf-` | 标准流程，关键方案和架构决策先确认 |
| `bf-fast` | 小改动或定位明确任务，发现风险即升级 |
| `bf-a` / `bf a` | 自动补安全默认细节，遇到高风险或架构会签项停下 |
| `/backend-forge` | 显式进入 |

## 路由

1. 先判定项目状态：空目录、中型项目架构负责人、Spring Boot、FastAPI、Django、其他后端、非后端。
2. 再判定任务类型：架构负责人、新项目、API/功能、数据迁移、鉴权、Bug、测试、性能、缓存异步队列、配置部署、审查重构。
3. 最后选择模式和测试闭环。

## 硬门禁

- 需求不清：不实现。
- 架构决策不清：不大规模实现。
- 数据影响不清：不实现。
- 权限边界不清：不实现。
- 测试闭环不清：不实现。
- 迁移未验证：不宣称完成。
- Bug 未复现或未说明无法复现：不直接修。
- 生产配置/数据删除/权限模型变更：必须确认。

## 默认技术策略

| 方向 | 默认 |
|---|---|
| Java | Spring Boot |
| Python | FastAPI |
| Django | 维护支持 |
| 正式数据库 | PostgreSQL |
| Spring 快速验证 | H2 |
| FastAPI 快速验证 | SQLite |
| Spring 迁移 | Flyway 或 Liquibase，优先尊重项目现状 |
| FastAPI 迁移 | Alembic |

## 架构负责人轨道

| 阶段 | 产出 |
|---|---|
| A0 负责人上下文 | 业务目标、团队能力、交付期限、风险承受度 |
| A1 架构范围 | 系统边界、模块划分、API 策略、非目标 |
| A2 数据架构 | 实体、所有权、迁移、事务、增长预期 |
| A3 安全权限 | 认证、授权、资源归属、审计、敏感数据 |
| A4 运行架构 | 环境、配置、可观测性、依赖、回滚 |
| A5 决策冻结 | 架构简报、ADR、风险清单、测试策略 |
| A6 分片交付 | API、数据、权限、业务、集成、测试 |
| A7 架构验证 | 契约、迁移、安全、集成、性能和风险收口 |

## 决策规则库

| 决策 | 默认 |
|---|---|
| 模块化单体 vs 微服务 | 中型项目默认模块化单体，满足独立部署/扩缩容/团队 owner/运维能力才拆服务 |
| PostgreSQL vs MySQL | 新项目默认 PostgreSQL，已有项目尊重现有数据库 |
| JWT vs session | 浏览器同站应用默认 session + 安全 cookie，跨端/API/网关场景再考虑 JWT |
| 同步 vs 异步 | 用户等待结果用同步，长耗时/重试/削峰/事件传播用异步 |
| Redis | 没有明确缓存/session/限流/队列/实时需求不上 Redis |
| ORM | 默认用框架主流 ORM，复杂查询允许显式 SQL |
| 文件存储 | 上传文件默认对象存储，数据库只存元数据 |
| 搜索 | 默认数据库查询/索引，超过能力再引入搜索引擎 |
| 后台任务 | 短任务用框架内能力，可靠任务用队列/批处理 |
| API Gateway | 单服务不默认引入，需统一认证/限流/治理时再上 |
| Observability | 最低结构化日志、指标、健康检查、错误追踪 |
| CI/CD | 至少 CI，CD 依赖测试、回滚、环境和密钥成熟度 |
| 测试金字塔 | 多低层测试，少高层 E2E，不用 E2E 补底层缺口 |

## 前端心智迁移

| 前端熟悉项 | 后端映射 |
|---|---|
| 页面 state/store | 数据模型、持久化字段、派生状态 |
| 点击/提交 | command、事务、幂等、回滚 |
| API type/mock | OpenAPI、status code、契约测试 |
| 隐藏按钮/路由守卫 | 后端认证、授权、资源归属 |
| toast/error boundary | 稳定错误码、日志、trace id |
| React Query/SWR cache | cache key、TTL、失效、脏读窗口 |

## 测试闭环

| 任务 | 最低验证 |
|---|---|
| 新项目 | 核心 API、持久化、异常路径 |
| API/功能 | 单元测试 + 接口/集成测试 |
| 数据迁移 | 迁移执行 + 关键查询 |
| 鉴权 | 未登录、权限不足、越权、正常授权 |
| Bug 修复 | 失败复现测试 + 回归测试 |
| 性能/并发 | 基线 + 优化后对比 |
| 缓存/队列 | 失效、重试、幂等、失败路径 |

## 会签信号

- 多团队平台化治理。
- 组织级技术治理。
- 企业级 IAM/SSO。
- 平台级多租户权限模型。
- 跨业务线数据治理。
- 全链路事件一致性。

命中这些信号时，`backend-forge` 先输出决策问题、可选方案、风险和验证方式，再要求用户确认授权或会签结果。
