# 跨平台质量归一化评分规则

搜索模块 Step 5 使用此文件。**搜索时无需加载此文件**，除非需要查阅具体映射值。
核心公式已内联在 search.md 中：`推荐指数 = 热度×0.4 + 权威×0.35 + 鲜度×0.25`。

---

## 三维评分模型

| 维度 | 权重 | 含义 |
|------|------|------|
| **热度** (Popularity) | 40% | 安装量、下载量、Stars、排名等量化流行度 |
| **权威** (Authority) | 35% | 来源可信度、官方背书、Verified 标识、策展收录 |
| **鲜度** (Freshness) | 25% | 最后更新时间、活跃维护状态 |

## 热度归一化（各平台映射表）

| 平台 | 原始指标 | ⭐5 | ⭐4 | ⭐3 | ⭐2 | ⭐1 |
|------|---------|-----|-----|-----|-----|-----|
| **skills.sh** | 周安装量 | ≥ 10K | ≥ 1K | ≥ 100 | ≥ 10 | < 10 |
| **ClawHub** | 总下载量 | ≥ 50K | ≥ 5K | ≥ 500 | ≥ 100 | < 100 |
| **GitHub** | Stars | ≥ 10K | ≥ 1K | ≥ 100 | ≥ 10 | < 10 |
| **SkillsMP** | 源仓库 Stars | ≥ 100K | ≥ 10K | ≥ 1K | ≥ 100 | < 100 |
| **Awesome lists** | 被收录 | — | — | ✅ 收录 = 3 | — | 未收录 = 1 |

> 无法获取具体数值时，按该平台可获取的最佳信号估算。

## 权威归一化

| 信号 | 分值 | 标记 |
|------|------|------|
| 官方维护（Anthropic / Vercel / Google） | 5 | 🏢 官方 |
| Verified 标识（ClawHub 认证） | 4.5 | ✅ 认证 |
| 平台精选/推荐（skills.sh 精选、SkillsMP 推荐） | 4 | ⭐ 精选 |
| 入选 awesome-list / 策展合集 | 3.5 | 📋 精选 |
| 知名作者/组织（Stars 1K+ 的作者） | 3 | 👤 知名 |
| 一般社区贡献 | 2 | 👥 社区 |
| 未知来源 | 1 | ❓ 未知 |

## 鲜度归一化

| 最后更新时间 | 分值 |
|-------------|------|
| < 1 个月 | 5 |
| 1-3 个月 | 4 |
| 3-6 个月 | 3 |
| 6-12 个月 | 2 |
| > 1 年 | 1 |

> 精选合集中的 Skill 如果无法获取更新时间，默认给 3 分。

## 跨平台排序规则

所有渠道结果混合排序，按推荐指数从高到低。同分时：
1. 官方来源 > 精选 > 社区
2. 安装量/Stars 高的优先
3. 更新时间近的优先

## 推荐指数标记

| 推荐指数 | 标记 | 含义 |
|---------|------|------|
| ≥ 4.0 | 🔥🔥🔥 强烈推荐 | 头部 Skill，经过大量验证 |
| 3.0-3.9 | 🔥🔥 推荐 | 优质 Skill，值得尝试 |
| 2.0-2.9 | 🔥 可用 | 功能匹配但需自行评估 |
| < 2.0 | — | 仅展示，不主动推荐 |

## 多来源加分

| 出现渠道数 | 加分 |
|-----------|------|
| 2 个渠道 | +0.3 |
| 3+ 个渠道 | +0.5 |

> 例如同时出现在 skills.sh（热度 3）和 awesome-openclaw（热度 3），取较高热度 3 + 0.3 = 3.3。

## 各来源热度指标说明

| 来源 | 热度指标 | 说明 |
|------|---------|------|
| skills.sh | **周安装量** | `npx skills find` 返回，非总安装量 |
| ClawHub | **总下载量** | CLI 返回，注意 Verified 标识 |
| GitHub | **Stars 数** | `gh api repos/{owner}/{repo}` 获取 |
| SkillsMP | **源仓库 Stars** | API 返回 `stars` 字段 |
| Awesome lists | **被收录** | 入选 = 通过社区筛选 |
