# 降级规则与三层兜底链

> 单源失败 / 模块失败 / 全系统失败时的统一降级策略，防止"静默 fallback"导致用户对偏差无法定位。

> SKILL.md 主体只声明"降级时必须透明告知"原则，详细 9+3 条规则全部在本文件维护。

## L1：单源失败降级表（9 条）

| 异常情况 | 降级策略 |
|---------|---------|
| **CLI 不可用（无 Node.js）** | skills.sh 搜索降级为 curl API（同一端点），安装统一用 `install_skill.sh`（禁止 `npx skills add`），其余 CLI 渠道跳过并标注 |
| **ClawHub 版本不兼容** | 跳过，标注不可用 |
| **gh CLI 未安装或未认证** | **强烈引导安装 gh**；降级为 WebSearch 兜底（精度显著降低；触发 balanced 条件 D） |
| **GitHub 搜索全部无结果** | 扩展关键词（同义词），最多重试一次 |
| **深度抓取失败/超时** | 跳过，基于已有摘要分类 |
| **安全扫描脚本不可用** | 跳过硬扫描，仅执行 AI 软判断 |
| **全部搜索均无结果** | 自动切到智能推荐 + 引导创建模块；浏览 https://skills.sh |
| **config/sources.yaml 不存在** | 使用内置默认渠道配置 |
| **skills.sh API 超时** | CLI 和 API 是同一端点，一个超时另一个也不行；跳过 skills.sh，继续其他渠道 |
| **自定义渠道响应异常** | 跳过该渠道，标注原因，继续其他渠道 |

## L2：模块失败跨模块降级

L1 覆盖的是**单源失败**。当**多源 / 整模块**都失败时按以下规则跨模块降级：

| 触发条件 | 处理策略 |
|---|---|
| 同模块超过半数渠道全部失败（如所有搜索源都返空 / API Key 全无效） | 自动跨模块降级：搜索 → 推荐（基于已安装 skill）→ 创建引导（`modules/create.md`） |

**典型 L2 场景**：
- 用户搜"PPT skill"，但 skills.sh / SkillsMP / SkillHub 三源同时返空 → 走 L2 → 检查已安装 skills 看是否有"docx-skill"等近似的，推荐尝试 → 仍无解 → 引导用户走 modules/create.md 自己创建

## L3：全部失败的最终兜底

| 触发条件 | 处理策略 |
|---|---|
| L2 后仍无结果 / 系统资源缺失（无 git、无网络、无任何 API Key、无可用 CLI） | **透明告知用户**：列出已尝试路径 + 已识别失败原因 + 给出 3 条人工兜底 |

**3 条人工兜底**：
1. 浏览 https://skills.sh 手动浏览
2. 进入 setup 重新配置（`重新配置搜索偏好`）
3. 联系 skill-assistant 维护者

**典型 L3 场景**：
- 用户在断网环境 + 无 cache + 无任何源凭证 → L1/L2 全失败 → 透明告知"目前无可用搜索源"+ 给出离线时可做的事（cd 到 skill 目录扫已有 skill / 看本地 cache）

## 降级透明度强制约束

> 任何 L1/L2/L3 降级**必须在输出中显式提示** ——不允许静默 fallback。

### 强制要求

1. **降级路径写入诊断头**：搜索/推荐/质检的诊断头都有「环境/异常」段，所有降级动作（"gh 不可用 → 走 WebSearch"）必须在此段记录
2. **用户能定位偏差来源**：用户看到"为什么这次结果质量差？"时，能直接从诊断头读出"因为走了 L2 降级路径"
3. **违反触发 🚨 阻断级 `SILENT_FALLBACK`**：发现降级未提示时，重跑前必须修复

### 输出示例

✅ Good：

```
诊断头
└─ 环境快照
   ├─ gh CLI: ❌ 未安装 → 已降级到 WebSearch（精度↓）
   ├─ skills.sh API: ✅ 200ms
   └─ SkillsMP: ❌ Key 未配置 → 已跳过

漏斗收敛
├─ skills.sh: 30 候选 → 22 去重
├─ WebSearch: 5 候选（gh 降级路径）→ 5 保留
└─ 总计: 27 候选
```

❌ Bad：

```
找到 27 条结果。
[直接列结果，用户不知道 gh 失败了]
```

### 跨模块降级时的额外约束

L2 / L3 降级必须额外问用户："是否继续走降级路径？" —— 因为跨模块降级意味着用户的原始意图已经被改写（搜索 → 推荐），不能默认替用户决定。
