# Excel 自动化类 Skill 示例

## 示例目标

帮助 Skill 创建者快速生成一个面向 Excel/CSV 自动化处理任务的可复用 Skill 示例。该示例覆盖 YAML front matter、触发描述、适用与不适用场景、文件处理边界、数据质量处理、异常处理、交付契约和质量检查清单，可作为编写 `SKILL.md` 的参考模板。

> 使用方式：复制“推荐 Skill 结构”代码块中的内容到目标 Skill 的 `SKILL.md`，再根据具体业务场景替换名称、触发条件、输入输出规则和检查清单。

## 适用场景

- 财务、销售、运营、人事等结构化表格的批量清洗、合并、拆分与汇总。
- 多个 Excel/CSV 文件的字段标准化、去重、格式统一、编码识别和表头修复。
- 生成指标汇总表、透视表、异常明细、分组统计、趋势表和可交付工作簿。
- 对 Excel 文件进行质量检查，包括缺失值、重复值、字段一致性、数据类型、日期格式、金额精度和异常值。
- 将用户给出的业务口径落地为可复核的表格处理流程，并输出处理日志或说明。

## 不适用场景

- 非结构化文档理解为主的任务，例如合同审查、长文总结、PDF 逐页排版还原。
- 需要人工判断业务事实、审批结论或财务合规意见的任务；Skill 只能辅助整理和校验数据。
- 需要破解密码、绕过权限、恢复损坏严重且无法读取的工作簿。
- 需要实时连接企业数据库、内部系统或第三方 API，但用户未提供授权、接口文档或可访问凭据。
- 高复杂度 BI 建模、交互式仪表盘系统开发，除非任务被拆解为明确的 Excel 文件生成步骤。

以下是可直接作为 `SKILL.md` 起点的示例。YAML front matter 中的 `description` 建议始终加引号，避免冒号、括号等字符破坏 YAML 解析。

```markdown
# Excel 自动化助手

## 能力定位

本 Skill 用于将用户的 Excel/CSV 表格处理需求转化为稳健、可复核、可交付的自动化流程。它应优先保护原始文件，明确处理口径，输出可直接使用的工作簿、CSV、图表或说明文件。

## 触发条件

当用户出现以下意图时触发：

- 处理 Excel、CSV、工作簿、表格、台账、明细、报表、透视表或批量文件。
- 要求执行合并、拆分、去重、清洗、筛选、排序、分类、汇总、计算、匹配、格式化或图表生成。
- 要求检查缺失值、重复值、异常值、字段不一致、日期格式、金额格式或数据口径。
- 要求把多个来源的数据整理成标准模板、可交付工作簿或管理层汇总表。

不要仅因用户提到“数据”就触发；如果任务核心是数据库开发、自然语言问答、图片识别或文档写作，应使用更合适的 Skill。

## 输入要求

执行前应确认或自动识别以下信息：

1. 输入文件：文件路径、文件类型、工作表名称、编码、是否有表头、是否包含合并单元格。
2. 处理目标：清洗、合并、汇总、拆分、校验、图表、格式化或模板填充。
3. 统计口径：日期范围、分组字段、指标定义、去重规则、金额单位、小数位规则。
4. 输出要求：文件格式、工作表结构、命名规则、是否保留原始明细、是否输出异常明细。
5. 风险边界：是否允许删除行列、是否允许覆盖旧文件、是否需要保留中间处理记录。

如果关键口径缺失且无法从上下文可靠推断，应先向用户确认；如果可以采用安全默认值，应在汇报中说明。

## 文件处理边界

- 只读取用户提供或当前任务明确授权访问的文件。
- 不修改原始文件；所有处理结果写入约定的输出目录。
- 同名输出文件如需覆盖，应遵循当前任务的交付约定；没有约定时优先生成清晰命名的新交付物。
- 读取 Excel 时应识别工作表、表头行、隐藏列、空行、合并单元格和公式列的影响。
- 对 `.xlsx`、`.xls`、`.csv` 应分别处理：CSV 注意编码、分隔符和换行；XLSX 注意多 sheet、公式、格式和单元格类型。
- 不尝试破解受密码保护的文件，不绕过权限，不把敏感数据上传到未授权的外部服务。
- 对大型文件应优先采用分块读取、字段裁剪、抽样检查和汇总验证，避免因内存或超时导致失败。

## 执行流程

1. 明确任务目标、输入文件、输出格式和统计口径。
2. 探查文件结构：文件数量、工作表、列名、行数、数据类型、空行、重复列、示例记录。
3. 执行数据质量检查：缺失值、重复值、异常值、日期/金额/编码格式、字段一致性。
4. 根据口径进行清洗、转换、合并、匹配、计算、汇总、拆分、图表生成或格式化。
5. 生成交付物：结果工作簿、异常明细、汇总表、图表或处理说明。
6. 复核结果：行数守恒、关键指标对账、抽样核验、sheet 完整性、文件可打开性。
7. 汇报结果：输入范围、处理动作、关键指标、输出路径、异常数据和已知限制。

## 数据质量处理原则

- 不静默删除数据；删除、填充、替换、合并前应有明确规则或在说明中记录。
- 对缺失值应区分“空字符串、空单元格、0、NULL、N/A、-”等不同含义。
- 对重复值应明确去重主键、保留规则和重复明细输出方式。
- 对日期应统一格式并处理 Excel 序列日期、文本日期、跨地区格式和无效日期。
- 对金额和数值应处理千分位、货币符号、百分号、负数括号、小数位和文本型数字。
- 对分类字段应处理全半角、大小写、前后空格、别名映射和异常枚举值。
- 对跨表匹配应输出未匹配、重复匹配和多对一/一对多风险。
- 对公式列应说明是保留公式、读取计算值，还是重写为静态值。

## 异常处理

遇到以下情况时，应停止、降级或显式汇报，不得假装完成：

- 文件不存在、无法访问、格式损坏、密码保护、扩展名与实际格式不一致。
- 必需字段缺失、列名重复、表头无法识别、工作表为空或结构与预期不符。
- 关键统计口径缺失，且无法安全推断。
- 数据量过大导致读取、计算或写入失败。
- 输出文件写入失败、文件被占用、权限不足或生成后无法打开。
- 校验结果不一致，例如汇总金额与明细不平、处理前后行数异常变化。

异常汇报应包含：问题位置、影响范围、已完成动作、未完成动作、建议用户补充的信息或可选降级方案。

## 交付契约

- 所有交付物写入任务约定的 `output/` 目录或用户明确指定的安全输出目录。
- 不覆盖原始文件，不在用户源文件旁生成临时文件，除非用户明确要求。
- 输出文件命名应清晰、可读、可追溯，例如 `sales-summary.xlsx`、`cleaned-orders.xlsx`、`data-quality-report.xlsx`。
- 单文件任务直接交付结果文件；多文件任务可打包为 ZIP，并剔除日志、缓存、临时文件等运行态产物。
- 交付说明至少包含：输入文件范围、处理动作、输出文件、核心指标、质量检查结果、异常/限制。
- 若输出包含多个 sheet，建议包含：`readme` 或 `summary`、`cleaned_data`、`pivot_summary`、`exceptions`、`quality_checks`。
- 若存在未处理或需人工确认的数据，应单独输出异常明细，不得混入已确认结果。

## 质量检查清单

交付前必须检查：

- [ ] YAML front matter 可解析，字段名称、缩进和引号正确。
- [ ] 触发描述具体，覆盖 Excel/CSV/表格处理意图，但不过度泛化。
- [ ] 输入文件、工作表、列名、行数和关键字段已被识别或说明。
- [ ] 数据质量检查覆盖缺失、重复、异常、类型、日期、金额和字段一致性。
- [ ] 清洗/汇总/匹配规则明确，可被用户复核。
- [ ] 处理前后行数、金额、数量等关键指标已对账。
- [ ] 异常明细已保留，未静默丢弃数据。
- [ ] 输出文件可打开，sheet 命名清晰，格式满足用户需求。
- [ ] 原始文件未被覆盖或污染。
- [ ] 最终汇报包含输出路径、处理摘要、关键指标和已知风险。
```

## 示例触发描述写法

推荐写法：

- “当用户上传 Excel/CSV 并要求合并、清洗、统计、生成报表或检查数据质量时触发。”
- “当用户要求把多张表整理为标准模板、异常明细或管理层汇总表时触发。”
- “当用户提到去重、透视、匹配、VLOOKUP、字段标准化、金额汇总、日期格式修复等表格处理动作时触发。”

不推荐写法：

- “当用户处理数据时触发。”——过于宽泛，容易误触发。
- “当用户上传任何文件时触发。”——范围失控。
- “自动解决所有 Excel 问题。”——承诺过度，不利于异常边界管理。

## 文件处理边界补充

编写 Excel 自动化 Skill 时，应明确以下边界：

| 边界项 | 推荐约定 |
| --- | --- |
| 原始文件 | 只读，不覆盖，不移动，不重命名 |
| 输出位置 | 写入约定 `output/` 目录或用户指定安全目录 |
| 临时文件 | 可使用但不作为最终交付；打包时剔除 |
| 密码文件 | 不破解，要求用户提供可访问版本 |
| 公式 | 明确保留公式、读取计算值或转为静态值 |
| 多 sheet | 先列出 sheet，再按用户目标选择处理范围 |
| 大文件 | 先抽样和统计结构，必要时分块处理 |
| 敏感数据 | 不上传到未授权外部服务，汇报中避免暴露不必要明细 |

## 数据质量处理示例

| 问题类型 | 检查方式 | 推荐处理 |
| --- | --- | --- |
| 缺失值 | 统计每列空值数量和比例 | 按规则填充、保留、剔除或输出异常明细 |
| 重复值 | 按业务主键检查重复 | 输出重复清单，按保留规则去重 |
| 日期异常 | 识别文本日期、序列日期、无效日期 | 统一格式，无法解析的进入异常表 |
| 金额异常 | 检查文本型数字、货币符号、千分位 | 标准化为数值，保留原值列便于追溯 |
| 分类不一致 | 检查空格、大小写、别名、非法枚举 | 标准化映射，未识别值进入异常表 |
| 跨表匹配失败 | 检查未匹配和重复匹配 | 输出未匹配/多匹配明细，不直接丢弃 |

## 异常处理模板

当任务无法完全完成时，可按以下格式汇报：

```text
处理状态：未完全完成
卡点：文件中缺少必需字段“客户编号”，无法执行跨表匹配。
已完成：已读取 3 个文件，识别 5 个工作表，完成字段结构检查。
影响范围：订单明细表无法与客户主数据匹配，汇总结果不可确认。
建议方案：
1. 请补充包含“客户编号”的原始文件；或
2. 确认是否改用“客户名称”作为匹配字段，但存在重名风险。
```

## 交付汇报模板

```text
任务已完成。
输入范围：处理 2 个 Excel 文件，共 4 个工作表、36,248 行明细。
处理动作：统一字段名、修复日期格式、按订单号去重、按月份和区域汇总销售额。
质量检查：发现 128 行缺失客户编号，已写入 exceptions 工作表；汇总金额与明细金额已对账一致。
交付文件：cleaned-sales-report.xlsx
已知限制：客户编号缺失的数据未参与客户维度分析，需业务侧确认后补充。
```

## 最小可安装包建议

如需把该示例扩展为最小可安装包，建议目录结构如下：

```text
excel-automation/
├── SKILL.md
├── package.json
└── README.md
```

最小包要求：

- `SKILL.md` 必须包含可解析的 YAML front matter。
- `package.json` 至少包含 `name`、`version`、`description`、`main` 或约定入口字段。
- `README.md` 说明安装方式、触发场景、示例输入和示例输出。
- 示例包中不得包含真实客户数据、敏感表格或不可复现的本地绝对路径。
