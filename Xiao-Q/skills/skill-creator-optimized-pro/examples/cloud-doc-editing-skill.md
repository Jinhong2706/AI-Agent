# 云文档编辑类 Skill 示例

本示例用于指导创建“云文档编辑类”技能包，覆盖 WPS、飞书、企业知识库、在线表格等云端协作文档在读取、编辑、权限、版本、冲突、回写确认和交付验证方面的边界。它适合作为 Skill Creator 的参考样例，不直接绑定某一家平台的私有接口；实际落地时，应根据目标平台的官方 API、企业权限策略和当前运行环境可用工具进行适配。

> 核心原则：先读取和生成草稿，再预览差异，最后在用户明确确认且权限、版本、风险均可控时回写。无法安全回写时，应降级交付本地草稿、补丁或操作指引。

## YAML front matter 示例

以下 front matter 可作为云文档编辑类 Skill 的注册元数据模板。实际技能包应放在技能入口文档顶部，并保证 YAML 可解析。

```yaml
---
name: cloud-doc-editing
description: "读取、整理并在用户确认后有限编辑云端协作文档，适用于飞书、WPS、企业知识库和在线表格等场景。"
version: "0.1.0"
category: productivity
tags:
  - cloud-document
  - document-editing
  - lark
  - wps
  - collaboration
triggers:
  - "读取飞书文档并整理内容"
  - "编辑 WPS 云文档"
  - "把在线表格内容整理后写回"
  - "根据云文档生成修订建议"
  - "在确认后回写云端文档"
inputs:
  - name: document_url
    type: string
    required: true
    description: "用户提供的云文档、在线表格、知识库页面或共享文件链接。"
  - name: edit_instruction
    type: string
    required: true
    description: "用户希望读取、整理、改写、追加或回写的具体要求。"
  - name: write_back_allowed
    type: boolean
    required: false
    default: false
    description: "是否允许在用户二次确认后执行回写。默认不回写。"
permissions:
  network: required
  filesystem: optional
  browser: optional
  user_confirmation: required_for_write_back
risk_level: medium
---
```

## 适用场景

- 读取用户已授权访问的 WPS 云文档、飞书文档、在线表格、在线知识库页面的正文、标题、表格、评论或基础元数据。
- 将云文档内容整理为会议纪要、周报、方案、摘要、待办清单、客户跟进表或结构化分析结果。
- 对云端文档进行有限范围编辑，例如补充段落、修订表格、整理标题层级、追加摘要、更新会议纪要。
- 在用户明确授权后，将本地生成内容回写到云文档。
- 在回写前做变更预览、版本确认、权限确认、风险提示和二次确认。
- 当平台 API 或权限不足时，生成可复制粘贴的本地草稿、差异补丁或手工操作步骤。

## 不适用场景

- 绕过平台登录、权限、企业风控、访问控制或审计机制。
- 在没有用户明确授权的情况下批量修改、删除、迁移、发布云端内容。
- 伪造他人身份进行编辑、评论、审批、签署或发布。
- 对包含敏感隐私、合同、财务、人事、客户身份信息、商业机密的文档进行未经确认的自动回写。
- 修改超出用户当前权限或组织政策允许范围的共享空间、知识库、公开文档。
- 在无法判断当前文档版本、协同编辑状态或回滚能力时直接覆盖原文。
- 用非官方、未授权或不稳定接口写入重要业务文档。

## 触发描述建议

Skill 的触发描述应同时覆盖“读取整理”和“确认后回写”两类意图，并避免承诺无条件编辑所有云文档。

推荐写法：

- 当用户提供飞书、WPS、在线表格、企业知识库等云文档链接，并要求读取、整理、摘要、改写、生成修订建议或在确认后回写时触发。
- 当用户要求“先不要回写”“生成草稿”“整理后让我确认”“写回原表”“更新原文档”“追加到文末”时触发。
- 当用户需要检查云文档权限、版本、冲突、变更范围或生成回写前预览时触发。

不推荐写法：

- “可以自动修改任意云文档。”
- “可以绕过权限读取内部文档。”
- “无需确认即可批量替换云端内容。”

## 权限与 API 版本边界

### 权限边界

技能必须尊重平台权限和企业策略：

- 只有查看权限时，只能读取，不能回写。
- 只有评论权限时，只能生成评论草稿或建议评论，不能直接改正文。
- 有编辑权限时，也应在用户确认变更内容后再回写。
- 企业文档、共享空间、知识库页面可能有继承权限、审计规则或审批流程，需要提示用户确认影响范围。
- 不得尝试绕过登录、验证码、单点登录、企业 VPN、二次验证或平台风控。
- 不得请求或保存用户账号密码、验证码、私钥、长期访问令牌等高敏凭证。

### API 与版本边界

不同平台的 API 能力、鉴权方式和字段结构可能随版本变化。技能应显式声明：

- 优先使用平台官方 API、官方 SDK、企业已授权连接器或当前环境提供的 CLI/浏览器能力。
- 不假设所有平台均支持评论读取、历史版本读取、批量替换、富文本保真、表格公式回写或附件处理。
- 调用 API 前应确认目标接口版本、文档类型、授权范围、速率限制和返回字段。
- 对无法确认 API 版本或写入语义的接口，不执行破坏性写入；改为输出本地补丁或操作指引。
- 写入富文本、表格、嵌入对象、图片、公式、引用块时，应说明保真风险。
- 如果平台 API 返回结构与预期不一致，应停止回写，保留错误摘要并降级交付。

建议在技能实现中维护平台能力矩阵：

| 能力 | 飞书文档 | WPS 云文档 | 在线表格 | 企业知识库 |
| --- | --- | --- | --- | --- |
| 读取正文 | 依官方 API/授权而定 | 依官方 API/授权而定 | 依工作表权限而定 | 依页面权限而定 |
| 读取评论 | 可能受接口限制 | 可能受接口限制 | 通常受限制 | 可能受限制 |
| 回写正文 | 需编辑权限与确认 | 需编辑权限与确认 | 需编辑权限与确认 | 需编辑权限与确认 |
| 批量替换 | 高风险，需二次确认 | 高风险，需二次确认 | 高风险，需二次确认 | 高风险，需二次确认 |
| 版本恢复 | 依平台能力 | 依平台能力 | 依平台能力 | 依平台能力 |

## 读取流程

### 1. 识别云文档类型

技能应先识别用户提供的是哪类资源：

- WPS 云文档链接、在线表格链接或共享文件链接。
- 飞书文档、飞书多维表格、飞书知识库、飞书云文档链接。
- 其他在线协作文档链接。
- 用户导出的本地副本，例如 `.docx`、`.xlsx`、`.pdf`、`.md`。

若链接无法访问，应说明可能原因：未登录、无权限、链接过期、企业网络限制、平台接口不可用、文档已删除或当前工具不支持该平台。

### 2. 明确读取范围

读取前需要确认或自动推断：

- 是否只读取用户当前可见内容。
- 是否包含评论、历史版本、附件、表格隐藏列、筛选视图、嵌入对象等扩展内容。
- 是否需要读取整个文档，还是只读取指定章节、表格、页面。
- 是否允许将内容临时保存到本地工作区用于处理。
- 是否涉及敏感信息，是否需要脱敏后再进入分析或交付。

默认原则：只读取完成任务所需的最小范围。

### 3. 读取后标准化

读取成功后，建议统一整理为中间结构，便于差异比较和回写验证：

- 文档标识：平台、链接、文档标题、文档类型。
- 权限状态：可读、可评论、可编辑、未知。
- 版本信息：更新时间、版本号、修订 ID、读取时间。
- 内容结构：标题层级、段落、表格、列表、图片/附件占位、评论摘要。
- 任务范围：用户要求处理的章节、表格、行列或全文。

## 编辑流程

### 1. 明确编辑边界

编辑前需要明确变更范围：

- 只追加，不覆盖。
- 只修改指定段落、指定表格、指定标题下内容。
- 允许改写全文，但必须保留结构。
- 允许批量替换，但需列出替换规则、影响范围和可回滚方案。
- 禁止删除原内容，除非用户明确要求。
- 对表格编辑，应明确工作表名称、行列范围、主键字段、排序或筛选规则。

对云端文档的自动编辑应遵循“先生成草稿，再确认回写”的原则。

### 2. 生成草稿与差异摘要

回写前应输出可审阅内容：

- 新增内容草稿。
- 拟替换段落或表格区域。
- 变更摘要，例如新增、删除、修改、移动的数量。
- 高风险变更列表，例如删除、覆盖、批量替换、公式改写。
- 用户可选择的回写方式：覆盖指定区域、追加到文末、插入指定标题下、生成新文档、仅导出本地文件。

### 3. 回写确认

以下情况必须二次确认：

- 覆盖、删除、批量替换、移动章节。
- 涉及合同、财务、人事、客户隐私、商业机密。
- 修改共享给多人或对外发布的文档。
- 文档较长且改动范围无法清晰列举。
- 平台没有可靠的撤销或版本恢复能力。
- 版本信息缺失、协同编辑状态未知或疑似存在冲突。

确认信息应包含：

- 将要修改的文档名称或链接。
- 修改范围。
- 主要变更摘要。
- 是否创建备份、导出副本或保留本地补丁。
- 回写后的验证方式。
- 如果失败，将如何降级交付。

### 4. 执行回写

执行回写时应遵循：

- 写入前再次获取当前版本或更新时间，与读取时版本进行比较。
- 如版本变化，停止回写并提示可能存在协同编辑冲突。
- 优先执行最小写入，不重写无关内容。
- 对表格写入使用稳定主键或明确行列范围，避免因排序/筛选导致写错行。
- 对富文本写入保留原结构，无法保真时先说明风险。
- 回写完成后立即读取关键区域验证。

## 版本与冲突处理

回写前应尽量确认：

- 当前文档版本、更新时间或修订 ID。
- 处理期间是否有人同时编辑。
- 是否需要先创建副本、导出备份或保存补丁。
- 是否支持平台版本恢复。
- 是否存在未保存草稿、锁定状态、审批状态或只读状态。

如果发现版本冲突或无法判断当前版本，应停止自动回写，改为输出：

- 本地草稿文件。
- 可复制粘贴的修改块。
- 结构化差异补丁。
- 手工回写步骤。
- 需要用户重新确认的冲突说明。

## 异常处理

| 异常类型 | 表现 | 处理策略 |
| --- | --- | --- |
| 链接无法访问 | 404、无权限、登录失效、链接过期 | 停止读取，说明原因，要求用户检查链接或授权 |
| 权限不足 | 可读不可写、可评论不可编辑 | 降级为草稿、评论建议或手工操作说明 |
| API 版本不匹配 | 字段缺失、返回结构变化、接口弃用 | 停止写入，记录接口与错误摘要，建议更新适配 |
| 速率限制 | 429、调用频率过高 | 暂停重试，避免连续轰炸；必要时让用户稍后再试 |
| 文档过大 | 超出读取/处理上限 | 分章节、分工作表、分批处理，并说明未处理范围 |
| 富文本不保真 | 样式、图片、公式、嵌入对象无法完整还原 | 保留原文，输出补丁或局部可复制内容 |
| 协同冲突 | 回写前版本变化、有人同时编辑 | 停止回写，重新读取并生成冲突说明 |
| 写入失败 | 接口错误、权限变化、网络中断 | 不重复盲写；读取当前状态，说明成功/失败范围 |
| 敏感内容 | 涉及合同、财务、人事、客户隐私 | 默认不自动回写，要求明确确认或脱敏处理 |

## 交付契约

云文档编辑类 Skill 的交付应清楚说明“读了什么、改了什么、是否写回、如何验证”。

### 未回写时

应交付：

- 读取范围说明。
- 整理后的草稿或修订建议。
- 差异摘要或拟修改清单。
- 风险提示与待确认项。
- 下一步选项：回写原文档、追加到文末、生成新文档、导出本地文件、停止。

示例：

```text
已读取《XX 会议纪要》的正文和 2 个表格，尚未回写。
我已生成正式版会议纪要草稿，并列出 5 处拟调整内容。
请确认：覆盖原正文、追加到文末，还是生成新文档？
```

### 已回写时

应交付：

- 回写的文档名称或链接。
- 实际修改范围。
- 关键变更摘要。
- 回写时间和版本/更新时间信息（如果可获得）。
- 验证结果，例如已复读关键段落、关键单元格、标题结构或记录数量。
- 备份、补丁或恢复建议。

示例：

```text
已在用户确认后回写《客户跟进表》的“本周待办”工作表。
本次修改 12 行，新增“下一步动作”列内容 12 条，未删除原始记录。
回写后已复读客户名称、负责人、下一步动作三列，抽检结果一致。
```

### 无法回写时

应交付：

- 无法回写的原因。
- 当前已完成的读取/整理结果。
- 可复制粘贴的内容或本地补丁。
- 用户可以手动执行的步骤。
- 需要补充的权限、链接或确认信息。

## 示例指令与响应策略

### 示例一：只整理，不回写

用户：

> 读取这个飞书文档，把会议纪要整理成正式版，但先不要回写。

技能响应策略：

1. 检查链接是否可访问。
2. 读取正文和相关表格。
3. 生成正式版会议纪要草稿。
4. 明确说明尚未回写。
5. 询问用户是否需要覆盖原文、追加到文末，或生成新文档。

### 示例二：整理在线表格并写回

用户：

> 把这个 WPS 在线表格里的客户跟进记录按“本周待办”整理一下，并写回原表。

技能响应策略：

1. 确认是否有编辑权限。
2. 读取必要工作表与列名。
3. 明确主键字段、行列范围和排序/筛选状态。
4. 生成拟修改清单。
5. 提示会改动哪些行、哪些列。
6. 获取用户明确确认后再回写。
7. 回写后复读关键单元格或导出变更摘要进行验证。

### 示例三：权限不足时降级

用户：

> 帮我改这个知识库页面，把产品介绍更新成最新版本。

技能响应策略：

1. 检查页面可访问性和权限。
2. 如只有查看权限，读取必要内容并生成新版产品介绍草稿。
3. 明确说明无法直接回写，因为当前没有编辑权限。
4. 提供可复制粘贴的更新内容和建议插入位置。
5. 提醒用户授权编辑权限后可再次确认回写。

## 推荐 Skill 能力描述

- 能读取用户授权访问的云文档内容。
- 能识别云文档类型、权限状态和基础版本信息。
- 能生成结构化编辑建议、草稿和差异摘要。
- 能在用户确认后执行有限范围回写。
- 能识别权限不足、版本冲突、API 版本不匹配和高风险变更。
- 能在无法安全回写时降级为本地草稿、补丁或操作指引。
- 能在回写后读取关键区域进行验证，并向用户报告验证结果。

## 安全原则

1. 最小读取：只读取任务必要内容。
2. 最小编辑：只改用户明确要求的范围。
3. 明确确认：回写前展示变更摘要并获得确认。
4. 可回滚：优先创建备份、保留版本或输出补丁。
5. 尊重权限：不绕过平台访问控制。
6. 风险降级：不确定时停止回写，交付可审阅草稿。
7. 凭证安全：不要求用户提供账号密码、验证码或长期令牌。
8. 审计友好：清楚记录读取范围、修改范围、确认状态和验证结果。

## 质量检查清单

在提交云文档编辑类 Skill 前，应检查：

### 元数据与触发

- [ ] YAML front matter 存在且可被 YAML 解析。
- [ ] `description` 没有使用会破坏 YAML 的未转义冒号、引号或换行。
- [ ] 触发描述覆盖读取、整理、差异预览、确认后回写、无法回写降级等典型意图。
- [ ] 未承诺绕过权限、无确认回写或支持所有平台私有能力。

### 场景与边界

- [ ] 明确适用场景和不适用场景。
- [ ] 明确查看、评论、编辑权限下分别允许的动作。
- [ ] 明确 API 版本、平台能力、速率限制和富文本保真风险。
- [ ] 对敏感文档和共享文档设置二次确认要求。

### 流程完整性

- [ ] 读取流程包含文档类型识别、读取范围确认、内容结构化。
- [ ] 编辑流程包含变更范围、草稿生成、差异摘要、用户确认、最小回写。
- [ ] 回写前检查版本或更新时间。
- [ ] 回写后读取关键区域进行验证。
- [ ] 版本冲突时停止自动回写并降级交付。

### 异常处理

- [ ] 覆盖链接失效、权限不足、API 不匹配、速率限制、文档过大、富文本不保真、协同冲突、写入失败等异常。
- [ ] 对失败写入有明确的状态核验，避免重复盲写。
- [ ] 对无法安全处理的情况有本地草稿、补丁或手工步骤作为降级方案。

### 交付契约

- [ ] 未回写、已回写、无法回写三种状态均有明确汇报模板。
- [ ] 汇报内容包含读取范围、修改范围、确认状态、验证结果和下一步。
- [ ] 如生成本地文件，路径应符合项目输出目录规则。
- [ ] 如修改云端文档，应能说明修改对象、时间、范围和验证结果。
