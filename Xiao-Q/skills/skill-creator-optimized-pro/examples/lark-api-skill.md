# 飞书 API 类 Skill 示例

> 本示例用于指导创建“通过飞书开放平台 API 读取、编辑云文档”的 Skill。它不是可直接运行的业务脚本，而是一个技能包示例模板，重点展示元数据、触发条件、API 边界、权限边界、交付契约与质量检查方式。

## 1. YAML front matter 示例

```yaml
---
name: lark-api-doc-assistant
description: "通过飞书开放平台 API 读取、分析与有限编辑飞书云文档的技能示例"
version: "0.1.0"
language: zh-CN
category: api-integration
maintainers:
  - office-raccoon
entrypoint: skill.md
capabilities:
  - 读取飞书云文档正文、标题、块结构与表格内容
  - 根据用户授权对文档进行有限编辑、追加、替换或生成摘要
  - 输出结构化处理报告与异常说明
triggers:
  - 用户提供飞书文档链接并要求读取、总结、改写或结构化提取
  - 用户明确要求在飞书文档中追加会议纪要、周报、任务清单或数据分析结果
  - 用户要求检查飞书文档格式、权限、可编辑性或 API 调用失败原因
permissions:
  required:
    - docs:document:readonly
    - drive:drive:readonly
  optional:
    - docs:document
    - drive:drive
    - wiki:wiki:readonly
risk_level: medium
---
```

## 2. 适用场景

本类 Skill 适合处理需要调用飞书开放平台 API 的文档任务，包括但不限于：

1. **读取类任务**：读取飞书文档、飞书 Wiki 文档、云空间文件的标题、正文、表格、块结构或元信息。
2. **分析类任务**：对文档内容进行摘要、提纲整理、任务拆解、会议纪要生成、待办提取、数据字段抽取。
3. **编辑类任务**：在用户明确授权后，对文档进行追加内容、替换指定片段、插入标题/段落/表格、修正格式。
4. **诊断类任务**：排查文档链接无权限、token 过期、tenant 不匹配、API 版本不兼容、块 ID 不存在等问题。
5. **交付类任务**：将处理结果回写到飞书文档，并同步输出本地报告、变更摘要或失败原因。

不适合场景：

- 用户未提供文档链接、文档 token 或可定位资源的信息。
- 用户未授权编辑，但要求直接改写线上文档。
- 需要绕过飞书权限、读取他人私密文档、破解 token 或规避审计。
- 涉及批量删除、覆盖整篇文档、迁移大量敏感文件等高风险操作，除非另有明确审批和回滚方案。

## 3. 触发描述

当用户出现以下意图时，可触发飞书 API 类 Skill：

- “读取这个飞书文档并总结重点。”
- “把会议纪要追加到这个飞书文档末尾。”
- “根据这个飞书链接提取任务负责人和截止时间。”
- “检查为什么这个飞书文档 API 读取失败。”
- “把分析结果写回飞书多维表格/云文档。”

触发前应确认：

1. 是否存在可解析的飞书文档链接、Wiki 链接、文件 token、document_id 或 block_id。
2. 用户请求是只读、有限编辑，还是高风险变更。
3. 当前 token 类型、权限范围和目标文档归属是否匹配。
4. 是否需要先生成本地交付物，再由用户确认后回写。

## 4. 读取/编辑文档 API 版本要求

### 4.1 推荐 API 版本

应优先使用飞书开放平台新版文档 API，并在 Skill 文档中明确版本边界：

- **读取文档内容**：优先使用 Docs v2 / Docs v3 中稳定的文档、块、内容读取接口，以飞书开放平台当前文档为准。
- **编辑文档内容**：优先使用支持块级编辑的新版接口，例如按 block_id 插入、更新、删除或追加内容的接口。
- **云空间与 Wiki 解析**：当用户提供 Wiki 链接或云空间链接时，应先通过 Drive / Wiki API 解析真实文档 token 或 node token，再进入文档读取流程。

### 4.2 版本兼容原则

1. 不要假设所有飞书链接都能直接映射为 document_id；应区分 Doc、Docx、Sheet、Bitable、Wiki、Drive 文件。
2. 老版 Doc 与新版 Docx 的 token、块结构、接口路径可能不同，应先识别资源类型。
3. 编辑前必须确认文档支持目标 API；不支持块级编辑时，应降级为“生成可复制内容”或“输出编辑建议”。
4. 对涉及表格、电子表格、多维表格的任务，不应复用普通文档 API，应切换到对应 Sheet / Bitable API。

## 5. 权限与 token 边界

### 5.1 token 类型边界

常见 token 包括：

- **tenant_access_token**：应用维度 token，适合访问已授权给应用的租户资源。
- **user_access_token**：用户授权 token，适合代表用户访问其有权限的资源。
- **app_access_token**：应用身份 token，通常用于获取其他 token 或访问应用级资源。

Skill 应明确：

1. 不在示例、日志或交付文件中明文保存 token。
2. 不将 token 写入文档正文、截图、报错摘要或可共享文件。
3. 不跨租户复用 token；遇到 tenant 不匹配时应停止并说明原因。
4. 不尝试扩大权限范围；缺权限时提示用户补充授权，而不是绕过权限。

### 5.2 权限范围建议

只读任务建议最小权限：

- `docs:document:readonly`
- `drive:drive:readonly`
- 如涉及 Wiki：`wiki:wiki:readonly`

编辑任务可能需要：

- `docs:document`
- `drive:drive`
- 对应 Sheet / Bitable 的读写权限

权限原则：

- 默认只读。
- 编辑必须有用户明确授权。
- 删除、覆盖、批量修改必须二次确认。
- 如果用户只要求“给我一版可复制内容”，不得擅自回写线上文档。

## 6. 异常处理

Skill 应对以下异常进行分类处理并给出可操作说明：

| 异常类型 | 常见原因 | 处理方式 |
| --- | --- | --- |
| 链接解析失败 | 链接格式不完整、非飞书资源、短链无法解析 | 要求用户提供完整链接或资源 token |
| 401 / token 无效 | token 过期、签发失败、环境变量缺失 | 重新获取 token，不输出 token 明文 |
| 403 / 无权限 | 应用未授权、用户无文档权限、租户不匹配 | 停止读取/编辑，提示补充授权或邀请应用进文档 |
| 404 / 资源不存在 | document_id、file_token、block_id 错误或文档已删除 | 重新解析链接并核验资源类型 |
| 429 / 频率限制 | 调用过快、批量读取过大 | 指数退避、分批重试，记录已完成范围 |
| 5xx / 服务异常 | 飞书服务端临时异常 | 有限重试后降级输出失败报告 |
| 结构不兼容 | Doc/Docx/Sheet/Bitable API 混用 | 识别资源类型后切换对应 API |
| 编辑冲突 | 文档被他人更新、块位置变化 | 重新拉取块结构，必要时请求用户确认 |

异常处理要求：

1. 错误信息应面向用户可理解，不暴露密钥、完整 token 或内部堆栈。
2. 所有写操作失败后，应说明是否已产生部分变更。
3. 批量操作应记录成功、失败和跳过的对象清单。
4. 对不可恢复错误，应停止继续写入，避免扩大影响。

## 7. 交付契约

Skill 执行完成后，应按任务类型交付以下内容。

### 7.1 只读/分析类任务

交付内容：

- 文档标题或资源标识摘要。
- 读取范围说明，例如全文、指定章节、指定表格或前 N 个块。
- 分析结果，例如摘要、待办、结构化字段、问题清单。
- 未读取内容或权限受限内容说明。

### 7.2 编辑类任务

交付内容：

- 是否已完成线上文档编辑。
- 变更位置，例如文档末尾、指定标题下、指定 block_id 后。
- 变更摘要，例如新增段落数、表格行数、替换片段数。
- 若部分失败，列出成功项、失败项、失败原因和建议补救方式。
- 如有高风险变更，应提供回滚建议或原内容备份说明。

### 7.3 本地文件交付

如任务需要生成本地文件，应遵循统一交付规则：

- 文件写入 `output/` 目录下的指定路径。
- 文件名使用英文小写、短横线分隔。
- 引用时使用 `local-file://` 加绝对路径。
- 多文件交付应打包为 ZIP。

## 8. 质量检查清单

在提交飞书 API 类 Skill 前，应完成以下检查：

### 8.1 元数据检查

- [ ] YAML front matter 存在，且位于文件顶部。
- [ ] YAML 可被解析，含英文冒号的字段已加引号。
- [ ] `name`、`description`、`version`、`language`、`entrypoint` 字段完整。
- [ ] 权限声明与能力范围一致，没有声明过宽权限。

### 8.2 触发与范围检查

- [ ] 触发描述清晰，能区分只读、编辑和诊断任务。
- [ ] 明确不适用场景和高风险场景。
- [ ] 编辑任务要求用户明确授权。
- [ ] 删除、覆盖、批量修改要求二次确认。

### 8.3 API 与权限检查

- [ ] 区分 Doc、Docx、Sheet、Bitable、Wiki、Drive 文件。
- [ ] 说明读取和编辑 API 的版本兼容边界。
- [ ] token 类型与权限范围说明准确。
- [ ] 不记录、不输出、不持久化 token 明文。

### 8.4 异常与降级检查

- [ ] 覆盖 401、403、404、429、5xx 等常见错误。
- [ ] 对部分成功和部分失败有明确交付说明。
- [ ] 对 API 不兼容场景提供降级方案。
- [ ] 对不可恢复错误会停止写入，避免继续扩大影响。

### 8.5 交付检查

- [ ] 输出结果说明读取范围或编辑范围。
- [ ] 编辑类任务说明具体变更位置和变更摘要。
- [ ] 本地文件路径遵循 `output/` 目录与 `local-file://` 引用规则。
- [ ] 交付内容不包含 token、密钥、个人隐私或不必要的内部日志。

## 9. 示例执行口径

当用户要求“读取并总结飞书文档”时：

1. 解析链接，判断资源类型。
2. 校验 token 与只读权限。
3. 拉取文档标题、块结构和正文内容。
4. 生成摘要、待办和风险点。
5. 汇报读取范围、未覆盖范围和失败项。

当用户要求“把结果写回飞书文档”时：

1. 确认用户明确授权编辑。
2. 获取最新文档结构和目标插入位置。
3. 先生成待写入内容摘要供用户确认，或在授权充分时执行有限写入。
4. 写入后重新读取目标区域进行校验。
5. 汇报变更位置、变更摘要、失败项和回滚建议。
