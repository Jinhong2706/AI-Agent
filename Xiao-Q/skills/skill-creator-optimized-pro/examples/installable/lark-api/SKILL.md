# lark-api

这是一个最小可安装的飞书 API 类 Skill 示例包，用于约束飞书开放平台 API 调用、飞书云文档 Docs v2 版本选择、权限/token 使用边界、异常处理与交付契约。

## 触发条件

当用户提出以下需求时，应启用本技能：

- 需要调用、封装、审查或排障飞书开放平台 API。
- 需要读取、创建、更新、导出或解析飞书云文档内容。
- 需要基于飞书文档 token、应用凭证或访问 token 设计安全调用流程。
- 需要生成飞书 API 示例、Skill 包、接口说明、请求/响应契约或错误处理模板。

## Docs v2 API 版本约束

- 飞书云文档相关能力默认限定为 **Docs v2 API**。
- 不得在同一实现中混用 Docs v1、历史接口、未确认版本接口或非官方路径。
- 在编写示例前，必须明确接口所属资源类型，例如文档、电子表格、多维表格、知识空间或文件。
- 若用户只说“飞书文档 API”，但未说明资源类型，应先按 Docs v2 的文档类接口设计最小方案，并在交付中标注需按实际资源 token 核验。
- 若接口版本、资源类型或 token 类型不确定，不得假定可写入，应降级为只读说明或请求用户补充授权信息。

## 权限与 token 边界

- 不生成、不伪造、不泄露、不长期保存任何 token、secret、cookie 或会话凭据。
- 只使用用户明确提供、当前任务必需、权限范围足够的 token。
- 区分以下边界：
  - `tenant_access_token`：应用级租户访问能力，不能代表具体用户越权访问私有文档。
  - `user_access_token`：用户授权能力，只能在授权范围与有效期内使用。
  - 文档 token / file token：只表示资源标识，不等同于访问权限。
  - app secret：只能用于换取 token，不得写入交付文件、日志或示例明文。
- 权限不足、token 过期、接口返回无权限或资源不存在时，必须停止写操作，并说明需要用户补充的授权范围。
- 不绕过飞书租户、应用、用户、知识库、空间、文档、文件夹或协作者权限。

## 异常处理要求

调用飞书 API 或编写调用示例时，必须覆盖以下异常：

- 网络异常：超时、DNS、TLS、代理、连接重置，应支持重试但避免无限循环。
- 鉴权异常：token 缺失、过期、无效、scope 不足，应提示重新授权或补充权限。
- 权限异常：无文档、空间或字段权限时，不继续尝试绕过。
- 版本异常：接口路径、参数或资源类型不匹配时，回到 Docs v2 官方版本约束重新核对。
- 频控异常：遇到限流或 429，应遵循返回的重试建议或采用指数退避。
- 数据异常：空文档、结构变化、字段缺失、分页未取完、响应码非 0，都要显式处理。
- 写入异常：创建、更新、删除、导出失败时，不声明成功；应交付当前状态、失败原因和人工核验建议。

## 交付契约

使用本技能形成的交付物必须至少包含：

1. **目标说明**：要调用的飞书能力、资源类型与业务目的。
2. **输入边界**：需要用户提供哪些 token、文档 token、app 信息、权限 scope 或资源链接。
3. **版本约束**：明确使用 Docs v2 API；若不是文档类资源，要说明对应开放平台资源版本。
4. **权限边界**：说明不会绕权、不会持久化凭据、不会把 secret 写入产物。
5. **请求/响应契约**：列出核心请求参数、分页策略、成功判定、输出字段。
6. **异常处理**：列出鉴权、权限、频控、网络、数据、写入失败的处理方式。
7. **交付路径**：若生成文件，只写入用户允许的路径；若修改云端内容，必须说明回写目标和确认状态。
8. **人工核验点**：对写入、删除、批量更新、权限变更等高风险操作，要求用户在飞书端复核。

## 最小执行流程

1. 识别用户需求是否属于飞书 API 或 Docs v2 场景。
2. 明确资源类型、token 类型、权限 scope 与是否存在写操作。
3. 若涉及真实调用，先确认不会暴露凭据，并限定最小必要权限。
4. 编写或审查请求逻辑时，优先给出可替换占位符，不在示例中写入真实 token。
5. 对响应结果进行成功码、分页、空值、权限与版本检查。
6. 交付时说明完成状态、风险、限制与人工核验建议。

## 禁止事项

- 禁止输出真实 token、secret、cookie 或会话信息。
- 禁止引导用户绕过飞书权限、租户限制或审计机制。
- 禁止把用户凭据写入 README、SKILL.md、日志、示例代码或提交记录。
- 禁止在未确认 Docs v2 资源类型前直接执行写入、删除、分享或权限变更。
- 禁止在接口失败时伪造成功结果。
