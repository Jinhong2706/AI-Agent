# 数据分析助手

## 触发条件

当用户提出以下需求时启用本技能：

- 读取 CSV、Excel、JSON、文本表格或其他结构化数据文件。
- 对数据进行缺失值、重复值、异常值、字段类型或口径一致性检查。
- 生成描述性统计、分组汇总、趋势分析、相关性分析或业务指标拆解。
- 输出图表、分析结论、数据明细、清洗后数据集或可复核报告。
- 需要在交付前说明数据口径、处理步骤、关键发现和已知风险。

## 数据读取清洗

1. 明确输入文件路径、数据格式、编码、工作表名称、字段含义和分析口径。
2. 优先使用稳定库读取数据，例如 `pandas`、`openpyxl`、`xlrd`、`json` 或标准库。
3. 读取后先记录原始行列数、字段列表、数据类型、空值比例和重复记录数量。
4. 对字段名进行必要规范化，例如去除首尾空格、统一大小写或映射为用户指定名称。
5. 对日期、数值、金额、比例、分类字段做显式类型转换，并保留无法转换的异常明细。
6. 清洗缺失值、重复值和明显异常值时，必须说明处理规则；不确定时保留原值并标记。
7. 不直接覆盖用户原始文件；清洗后数据应写入新的交付文件或中间结果。

## 分析流程

1. 对齐用户问题，确认核心指标、维度、时间范围、筛选条件和输出格式。
2. 建立基础数据质量检查表，确保样本量、字段完整性和关键口径可解释。
3. 先做总体概览，再按时间、区域、产品、客户、渠道等维度逐层拆解。
4. 对关键指标进行同比、环比、占比、排名、分布、离群点或贡献度分析。
5. 对发现的异常或变化给出数据证据，避免无依据推断。
6. 在结论中区分事实、推断和建议，必要时列出仍需补充的数据。
7. 交付前复核计算口径、样本数量、图表数据源和输出文件可打开性。

## 可视化

- 根据问题选择图表类型：趋势用折线图，结构用柱状图或饼图，分布用直方图或箱线图，关系用散点图。
- 单图最多两个子图，优先保证标题、坐标轴、单位、图例和数据标签清晰。
- 图表应服务于结论，不添加无关装饰；颜色保持克制并兼顾可读性。
- 图表文件应输出到工作区 `output/` 下，文件名使用小写英文和连字符。
- 若图表基于清洗后数据，应能追溯到对应数据文件或计算步骤。

## 异常处理

- 输入缺失或不可访问：停止分析，说明缺少的文件或权限，并请求用户补充。
- 格式不支持或文件损坏：说明失败原因，建议用户提供 CSV、XLSX、JSON 等可读版本。
- 字段缺失：列出缺失字段、影响范围和可替代字段；不可替代时降级交付。
- 编码或解析失败：尝试常见编码和分隔符；仍失败时保留错误信息并请求确认。
- 数据类型异常：先显式转换；无法转换的记录输出异常明细，不静默丢弃。
- 样本量过小或口径不一致：在结论和风险中明确提示，不做过度推断。
- 计算或校验不通过：不得作为最终结果交付，应先返回问题清单和修复建议。

## 交付契约

- 所有交付物必须写入工作区 `output/` 下的明确路径，不覆盖用户原始文件。
- 汇报必须包含：输入范围、清洗规则、分析口径、关键结论、输出文件和风险提示。
- 若生成多个文件，应打包为 ZIP；单文件可直接给出符合环境规范的本地文件链接。
- 交付文件命名使用描述性小写英文、连字符分隔，不使用时间戳、UUID、空格或中文。
- 结论必须可被数据追溯；不确定内容应标记为假设、限制或待确认事项。
- 交付前至少检查：行列数量、关键字段、空值、重复值、指标总量、图表引用数据和文件可打开性。

## 最小能力边界

本示例是可安装 Skill 包骨架，重点定义数据分析类任务的触发、执行和交付规范；具体代码由运行环境根据用户数据与目标动态生成。
