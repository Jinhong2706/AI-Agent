# multi-agent-collaboration

这是一个最小可安装的多 Agent 协作类 Skill 示例包，用于演示如何在复杂任务中设计“主控 Agent + 子 Agent”的协作流程。

## 包含文件

- `SKILL.md`：Skill 主说明，包含可解析 YAML front matter。
- `package.json`：最小包元数据。
- `README.md`：安装包说明。

## 能力范围

本 Skill 关注流程设计，不直接执行具体业务任务。它提供以下约束和模板：

1. 任务拆分：按文件、章节、页码、数据切片或检查维度拆分。
2. 并行边界：明确哪些任务可以并行，哪些任务必须串行。
3. 冲突合并：由主控 Agent 统一处理事实、口径、格式和越界冲突。
4. 主控验收：确保最终交付覆盖完整、路径正确、风险透明。
5. 异常处理：面对失败、缺失、权限或工具问题时提供降级方案。
6. 交付契约：限定输出位置、命名、引用、覆盖和可复核要求。

## 使用方式

在创建或优化协作型 Skill 时，先读取 `SKILL.md`，并按以下顺序使用：

1. 明确用户的最终目标和交付契约。
2. 判断任务是否适合多 Agent 拆分。
3. 为每个子 Agent 设置独立输入范围和输出路径。
4. 禁止子 Agent 覆盖最终交付物或共享文件。
5. 主控 Agent 收集结果后统一复核、合并和验收。
6. 最终向用户汇报产物路径、关键结论和残余风险。

## 最小派发示例

```text
你是子 Agent，只负责 unit-01，不要修改最终交付物。
输入：source-a.md。
输出：drafts/unit-01.md。
要求：提取关键结论、证据、风险和建议合并方式。
禁止：处理其他文件；覆盖 final.md；编造缺失信息。
```

## 风险提示

- 多 Agent 并行不等于无条件提速；拆分不清会带来合并成本。
- 子 Agent 不应共享写入同一文件。
- 主控 Agent 必须承担最终验收责任。
- 缺失信息和冲突结论必须显式说明。
