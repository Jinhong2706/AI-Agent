# 浏览器自动化最小 Skill

## 目标

当用户要求访问网页、填写后台表单、检查页面状态、截图留痕或排查网页请求时，使用本 Skill 形成稳定、合规、可交付的浏览器自动化流程。

## 工作原则

1. 先确认目标页面、目标动作和最终交付物。
2. 优先使用页面语义定位元素，例如按钮名称、输入框标签、可见文本、`data-testid`。
3. 不绕过登录、验证码、权限、风控或付费限制。
4. 涉及删除、付款、发布、发送通知、提交不可逆内容时，必须请求用户确认。
5. 截图、报告、导出文件统一写入 `output/`。
6. 交付时说明完成结果、证据路径、失败原因和风险。

## 浏览器自动化流程

### 1. 打开与识别页面

- 导航到用户提供的 URL；
- 读取页面标题、URL、关键文本和可访问性快照；
- 判断是否进入登录页、权限页、错误页或目标业务页；
- 如页面未加载完成，等待关键文本或元素出现。

### 2. 网页定位

优先级：

1. 角色 + 名称，例如“按钮：查询”；
2. label、placeholder、aria-label；
3. 稳定属性，例如 `data-testid`；
4. CSS 选择器；
5. 坐标或复杂 XPath 仅作为临时兜底，并在交付中提示脆弱性。

### 3. 表单处理

- 识别字段类型：文本框、复选框、单选框、下拉框、日期、上传控件；
- 填写前校验数据格式；
- 填写后读取页面回显，确认内容正确；
- 提交后验证成功提示、列表结果、详情页或网络响应。

### 4. 截图留痕

需要截图时：

- 保存到 `output/`；
- 使用小写英文和短横线命名；
- 必要时截取全页；
- 页面含敏感信息时，交付中提示隐私风险，必要时建议打码。

### 5. 网络请求排查

当页面报错或结果不一致时：

- 查看相关请求的状态码、请求路径、响应错误信息；
- 只提取必要字段；
- 对 token、Cookie、手机号、邮箱、证件号等敏感信息脱敏；
- 不伪造鉴权，不绕过权限。

### 6. 登录与权限

- 若出现扫码、验证码、二次认证，暂停并提示用户在浏览器中完成；
- 不保存、复述或导出密码、验证码、Cookie、Token；
- 若返回 401/403 或页面提示无权限，说明证据和建议开通的权限范围。

### 7. 失败降级

失败时按以下顺序处理：

1. 等待页面稳定或刷新页面；
2. 重新读取快照并更换定位方式；
3. 检查弹窗、遮罩、登录态和权限；
4. 检查控制台和网络请求；
5. 截图记录现场；
6. 停止盲目重试，交付当前状态和可选下一步。

## 交付格式

```markdown
已完成/部分完成：<一句话结果>

- 页面：<页面名称或 URL>
- 动作：<查询/填写/上传/截图/排查>
- 结果：<成功提示、数据结果或失败原因>
- 证据：<local-file:// 绝对路径>
- 风险：<登录、权限、隐私、不可逆操作、定位脆弱性等>
- 下一步：<如需用户确认或授权，写明动作>
```

## 禁止事项

- 绕过验证码、风控、登录或权限；
- 泄露账号、密码、Cookie、Token、密钥；
- 未确认就执行付款、删除、发布、群发、签署等高风险动作；
- 将截图或导出文件写到 `output/` 以外的交付目录。
