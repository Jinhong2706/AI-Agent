# 浏览器自动化最小可安装 Skill

这是一个最小 Skill 包示例，用于说明如何把浏览器自动化能力封装为可安装、可审查、可降级的 Skill。

## 文件结构

```text
browser-automation/
├── SKILL.md
├── package.json
└── README.md
```

## 能力范围

本 Skill 覆盖：

- 网页打开与页面状态识别；
- 按按钮、文本框、标签、占位符、可见文本等方式定位元素；
- 表单填写、下拉选择、复选框、文件上传与提交确认；
- 当前页、全页或局部元素截图；
- 网络请求状态码、请求参数和响应错误排查；
- 登录态、验证码、扫码、权限不足处理；
- 自动化失败后的截图留痕、问题说明和降级交付；
- `output/` 目录交付契约与 `local-file://` 引用规范。

## 安装与检查

将整个 `browser-automation/` 目录放入 SkillHub 或技能包加载目录后，确保 `SKILL.md` 位于包根目录。

可执行最小检查：

```bash
npm run check
```

检查内容：

- `SKILL.md` 是否存在；
- 是否以 YAML front matter 起始；
- 包元数据是否包含最小 `package.json` 字段。

## 使用示例

用户请求：

> 打开某后台页面，填写客户查询条件，截图保存查询结果，并说明是否有权限访问。

执行要点：

1. 打开 URL 并判断是否需要登录；
2. 若出现扫码、验证码或二次验证，暂停请用户完成；
3. 使用页面语义定位客户名称、日期、状态等字段；
4. 填写后读取回显，确认值正确；
5. 点击查询，等待结果区更新；
6. 若失败，检查网络请求和错误提示；
7. 截图写入 `output/`；
8. 交付完成结果、截图路径和风险。

## 安全边界

本 Skill 不用于：

- 绕过登录、验证码、风控、权限或付费限制；
- 保存、输出、转发密码、Cookie、Token、验证码、密钥；
- 未经确认执行付款、删除、发布、群发、签署等不可逆操作；
- 采集明显违法、侵权或违反平台规则的数据。

## 交付要求

交付内容应包含：

- 是否完成任务；
- 操作页面和关键动作；
- 表单或查询结果；
- 截图、报告或导出文件路径；
- 登录、权限、隐私、定位稳定性等风险；
- 若无法完成，说明已尝试动作、失败原因和下一步建议。

文件引用示例：

```markdown
[查询结果截图](local-file://<workspace>/output/browser-query-result.png)
```

实际交付时必须替换为工作区绝对路径，不使用相对路径、裸路径或 sandbox 链接。
