# 云文档编辑 Skill

当用户要求处理 WPS、飞书等云文档时，使用本技能。目标是在不越权、不静默覆盖、不制造版本风险的前提下，完成读取、编辑建议、变更预览与经确认后的回写。

## 适用范围

- 读取云文档标题、正文、表格、评论、修订记录、元数据或导出内容。
- 对文档进行有限编辑：追加摘要、修正文案、整理结构、更新表格、补充会议纪要。
- 在用户明确授权后，将确认过的内容回写到原云文档或新副本。
- 处理 WPS、飞书等平台时，以平台 API、官方导出、用户授权链接或用户提供文件为准。

## 必须遵守的边界

1. 权限边界：只处理用户有权访问并明确授权的文档；不得绕过登录、分享限制、企业权限或风控限制。
2. 读取边界：读取前确认来源、文档类型、目标范围；敏感内容仅用于当前任务。
3. 编辑边界：默认先生成变更建议或补丁，不直接覆盖云端原文。
4. 版本边界：回写前检查最近修改时间、版本号、协作者状态；存在并发编辑时暂停并提示用户。
5. 回写确认：必须向用户展示目标文档、编辑范围、关键变更和潜在风险，获得明确确认后才能回写。
6. 审计边界：记录回写时间、目标、摘要和是否创建备份或副本。

## 推荐流程

1. 识别平台：判断是 WPS、飞书、在线表格、知识库页面还是导出文件。
2. 确认授权：确认用户提供了可访问链接、令牌、导出文件或其他合法凭据。
3. 安全读取：优先只读取任务所需范围，避免无关内容扩散。
4. 生成预览：输出差异、改动清单或新版本草稿。
5. 版本检查：确认文档未被他人更新；如有冲突，改为生成本地补丁。
6. 用户确认：取得明确“允许回写”的指令。
7. 执行回写：写入原文档或副本，并返回结果、链接、版本信息与风险说明。

## 回写前确认话术

在执行回写前，应使用类似表述：

“我将把以下变更写回到【文档名称/链接】的【具体范围】。本次会影响【段落/表格/章节】；已检查权限和版本状态。是否确认回写？”

## 风险处理

- 无法确认权限：停止回写，仅提供本地草稿。
- 无法确认版本：停止覆盖，建议创建副本或导出补丁。
- 平台接口失败：汇报失败原因、已完成操作和可选替代路径。
- 内容含敏感信息：最小化处理范围，避免在日志或报告中复述敏感原文。
