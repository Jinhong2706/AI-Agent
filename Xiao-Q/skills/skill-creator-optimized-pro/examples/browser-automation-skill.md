# 浏览器自动化类 Skill 示例

本示例用于指导 Skill 作者把“需要真实网页交互”的任务拆解为可执行、可验证、可降级、可复用的浏览器自动化流程。适用任务包括网页信息采集、后台表单填写、上传/下载、登录态检查、截图留痕、网络请求排查、页面回归验证等。

> 核心原则：先定位真实页面状态，再执行操作；先验证结果，再交付结论；遇到权限、验证码、不可逆操作或敏感信息时，立即降级为用户确认或证据化汇报。

## 1. YAML front matter 示例

浏览器自动化类 Skill 的 `SKILL.md` 必须包含可解析的 YAML front matter。`description` 要明确触发条件、适用场景和边界，避免只写“浏览器自动化”这类泛化描述。

```markdown
---
name: browser-automation
version: "0.1.0"
description: "当任务需要打开真实网页、读取页面元素、填写表单、上传/下载文件、截图留痕、检查网络请求或执行页面回归验证时使用。适用于有权限访问的网页后台和业务系统；不用于绕过登录、破解验证码、规避风控或执行未经确认的不可逆操作。"
author: "your-team"
tags:
  - browser
  - automation
  - web
  - form
  - screenshot
  - regression
capabilities:
  - page-navigation
  - semantic-element-location
  - form-filling
  - file-upload-download
  - screenshot-evidence
  - network-debugging
  - regression-check
triggers:
  - "打开网页并操作"
  - "填写网页表单"
  - "上传文件到后台"
  - "从网页下载导出文件"
  - "网页截图留痕"
  - "检查页面是否正常"
  - "复现并定位网页问题"
---
```

### 1.1 front matter 编写要求

- `description` 必须说明“何时触发”和“何时不触发”。
- 含英文冒号、引号、括号等易影响 YAML 解析的内容，应使用引号包裹。
- `tags` 与 `capabilities` 应覆盖真实能力，不夸大“绕过验证”“自动登录”等能力。
- 如 Skill 需要外部依赖、环境变量或账号权限，应在正文中另行说明，不要把密钥写入 front matter。

## 2. 适用 / 不适用场景

### 2.1 适用场景

浏览器自动化 Skill 应优先处理以下任务：

- 需要打开网页，并根据页面真实状态进行操作；
- 需要读取可访问性树、按钮、输入框、表格、弹窗、提示语等页面元素；
- 需要填写表单、提交查询、筛选数据、翻页、复制页面结果；
- 需要上传本地文件到网页后台，或从网页导出/下载文件；
- 需要截图作为交付证据、错误复现证据或视觉验收依据；
- 需要检查接口请求、响应状态、控制台错误或页面加载异常；
- 需要验证一个修复是否在真实页面上生效；
- 需要在登录、权限、验证码、二次确认等环节做稳健处理。

### 2.2 不适用场景

不应用于以下任务：

- 绕过权限、破解验证码、伪造身份、规避风控；
- 未授权采集、批量爬取、攻击、撞库、压力测试；
- 自动执行付款、删除、发布、发送通知、变更权限等不可逆操作，除非用户明确确认；
- 保存、复述或导出用户密码、验证码、Cookie、Token、密钥等敏感凭证；
- 对受隐私、合规或平台规则限制的内容进行非必要访问或传播。

## 3. 触发描述与任务拆解

### 3.1 推荐触发描述

Skill 的触发描述应使用用户会自然说出的任务语言，例如：

- “帮我打开这个后台，把客户资料填进去并截图确认。”
- “去这个页面下载最新报表。”
- “复现一下这个按钮点了没反应的问题，看看控制台和接口报错。”
- “检查登录后首页、订单页、导出功能是否正常。”
- “上传这份 Excel 到系统，确认页面显示导入成功。”

### 3.2 标准任务拆解

建议把浏览器任务拆成以下阶段：

1. **目标确认**：明确目标 URL、账号状态、要操作的数据、期望结果、是否涉及不可逆操作。
2. **页面进入**：打开 URL，判断登录态、权限、加载状态和当前页面是否符合预期。
3. **真实定位**：通过可访问性快照、可见文本、label、placeholder、稳定属性定位元素。
4. **执行操作**：填写、点击、上传、下载、筛选、翻页、提交或截图。
5. **结果验证**：从页面提示、URL、列表记录、详情页、下载文件或网络请求中确认结果。
6. **异常处理**：遇到失败时记录现场、检查网络/控制台、脱敏汇报、给出下一步。
7. **交付归档**：输出结论、证据文件、未完成项、风险和建议。

## 4. 真实页面定位原则

### 4.1 首选稳定语义定位

定位元素时，优先使用以下顺序：

1. 页面可访问性快照中的角色和名称，例如“按钮：登录”“文本框：手机号”；
2. 表单 label、placeholder、aria-label、可见文本；
3. 稳定的业务属性，例如 `data-testid`、`data-qa`、`data-cy`；
4. 结构清晰且稳定的 CSS 选择器；
5. XPath、相对位置或坐标点击仅作为最后兜底。

### 4.2 避免脆弱定位

应避免依赖：

- 动态 class 名、哈希 class 名；
- 页面中的绝对序号，例如“第 7 个 div”；
- 截图坐标或固定屏幕分辨率；
- 仅在当前浏览器缩放比例有效的布局位置；
- 未确认页面状态就直接点击。

### 4.3 定位前检查页面状态

每次关键操作前应确认：

- 当前 URL 是否符合预期；
- 页面是否已加载完成，是否还在转圈或骨架屏；
- 目标元素是否可见、可点击、未禁用；
- 是否出现登录页、权限页、弹窗、遮罩层、错误页或新标签页；
- 页面语言、租户、组织、项目空间是否正确。

### 4.4 定位证据记录

对复杂任务，建议在执行记录中保留关键定位依据：

```text
页面：/customers/new
目标元素：文本框“客户名称”
定位依据：label=客户名称，placeholder=请输入客户名称
操作：填入“示例客户 A”
验证：提交后详情页字段显示“示例客户 A”
```

## 5. 表单自动化规范

表单类任务应按“识别字段 → 填写 → 校验 → 提交 → 确认结果”的顺序执行。

### 5.1 字段识别

- 明确字段名称、类型、必填规则和默认值；
- 对下拉框、单选、多选、日期选择器、级联选择器单独处理；
- 对上传控件检查文件格式、大小、路径和页面限制；
- 对金额、手机号、邮箱、日期、身份证号等字段先做格式校验；
- 如果字段与用户给出的数据无法对应，应暂停说明缺口，不要猜填。

### 5.2 填写策略

- 普通输入框优先整体填入；
- 对带前端监听、自动搜索、格式化校验的输入框，可采用逐字输入触发事件；
- 修改已有内容前先确认是否允许覆盖，再清空或全选覆盖；
- 下拉框应先打开选项列表，再选择与用户数据完全匹配或最接近的选项；
- 日期控件优先使用页面支持的输入格式，避免误选月份或时区；
- 提交前读取页面回显，确认字段值与任务要求一致。

### 5.3 提交与结果确认

提交后至少确认一种结果，关键任务建议确认两种以上：

- 页面出现成功提示；
- URL 或页面状态发生预期变化；
- 列表中出现新增或更新记录；
- 详情页可读到最终值；
- 网络请求返回成功状态；
- 系统生成了记录 ID、订单号、任务号或导入批次号。

若提交涉及不可逆操作、付款、删除、发布、发送通知、权限变更，应暂停并请求用户确认。

## 6. 上传、下载与文件处理

### 6.1 文件上传

上传前应检查：

- 本地文件路径存在，且位于允许访问范围内；
- 文件类型、大小、命名符合页面限制；
- 页面是否要求模板格式、列名、编码或压缩包结构；
- 上传后是否出现文件名、进度条、预览、校验结果或错误提示。

上传后应验证：

- 页面显示“上传成功”“导入成功”等明确提示；
- 文件名、条数、批次号、校验结果与预期一致；
- 如有失败明细，应下载或截图保存，并在交付中说明。

### 6.2 文件下载 / 导出

下载或导出前应确认：

- 下载范围、筛选条件、时间区间、文件格式是否符合用户要求；
- 导出按钮是否触发异步任务，是否需要等待任务完成；
- 下载文件是否落在 `output/` 下的明确路径；
- 文件名使用小写英文、短横线分隔，避免时间戳、UUID、中文和空格。

下载后应验证：

- 文件确实存在且大小不为 0；
- 文件扩展名、MIME 类型或可打开性符合预期；
- Excel/CSV/PDF 等文件可读取关键字段或页数；
- 如下载的是压缩包，应检查压缩包内文件列表是否符合预期。

### 6.3 敏感文件处理

- 不上传用户未授权的隐私文件；
- 不在报告中暴露身份证号、手机号、邮箱、Token、Cookie、合同敏感条款等；
- 含敏感信息的截图或导出文件，应在交付中标注风险，必要时打码或仅交付摘要。

## 7. 截图与视觉留痕

截图适用于证明页面状态、错误现象、提交结果或视觉回归。

### 7.1 截图要求

- 文件必须写入 `output/` 目录下的明确路径；
- 文件名使用小写英文、短横线分隔；
- 避免包含时间戳、UUID、中文和空格；
- 截图前关闭无关弹窗，必要时滚动到目标区域；
- 对包含隐私信息的截图，应在报告中提示风险，必要时打码后交付；
- 截图不是验证的唯一依据，关键提交应结合页面文本、记录或请求结果验证。

### 7.2 截图类型

- 当前视口截图：适合确认按钮、弹窗、表单状态；
- 全页截图：适合长页面审查；
- 元素截图：适合局部组件回归；
- 前后对比截图：适合修复验证、视觉变更和异常复现。

### 7.3 视觉回归注意事项

- 固定浏览器窗口大小或记录当前窗口大小；
- 等待图片、字体、异步数据加载完成后再截图；
- 排除动态内容，如时间、随机推荐、广告、闪烁动效；
- 对关键区域进行人工或视觉模型复核，不把“截图成功”等同于“页面正确”。

## 8. 网络请求与控制台排查

当页面操作失败、数据未更新或接口异常时，应检查网络请求和控制台。

### 8.1 关注内容

- 请求 URL、方法、状态码；
- 请求参数是否与表单内容一致；
- 响应体中的错误码、错误信息；
- 鉴权、Cookie、CSRF、组织空间、租户、项目 ID 等参数是否缺失；
- 是否存在跨域、超时、重定向、限流、网关错误或权限拒绝；
- 控制台是否有 JavaScript 错误、资源加载失败或前端异常堆栈。

### 8.2 使用边界

- 不输出敏感 token、Cookie、手机号、身份证号、密钥等；
- 如需引用接口信息，只保留必要字段并脱敏；
- 不尝试伪造身份、绕过权限或修改请求绕开前端限制；
- 不把网络接口当作隐藏 API 进行未授权调用。

## 9. 登录、权限与确认点处理

浏览器自动化 Skill 必须显式处理登录态、权限和高风险确认点。

### 9.1 登录态判断

可通过以下方式判断是否已登录：

- 页面是否跳转到登录页；
- 是否存在“登录”“扫码登录”“获取验证码”等元素；
- 当前 URL 是否包含 `login`、`auth`、`sso` 等路径；
- 关键业务接口是否返回 401/403；
- 页面是否显示当前用户头像、组织名或业务导航。

### 9.2 处理规则

- 如果需要用户扫码、短信验证码、二次验证，应暂停并提示用户完成；
- 不记录、不保存、不复述用户密码、验证码、Cookie、Token；
- 权限不足时，应说明缺少的权限、当前页面证据和建议找谁开通；
- 对企业系统、客户后台、财务系统等高敏页面，默认采用最小操作原则；
- 若页面弹出“确定删除 / 确定发布 / 确定付款 / 确定发送”等提示，应停止在确认前一步，等待用户授权。

## 10. 回归验证规范

回归验证类任务应明确“验证对象、验证步骤、预期结果、实际结果、证据”。

### 10.1 验证范围

常见回归范围包括：

- 页面能否正常打开，是否有白屏、报错、死循环加载；
- 登录后导航、菜单、权限入口是否正常；
- 关键按钮、表单、筛选、分页、上传、下载是否可用；
- 修复过的问题是否不再复现；
- 控制台和网络请求是否无关键错误；
- 不同角色、组织、项目空间或浏览器窗口尺寸下是否表现一致。

### 10.2 验证记录模板

```markdown
| 用例 | 步骤 | 预期结果 | 实际结果 | 证据 | 结论 |
| --- | --- | --- | --- | --- | --- |
| 登录后进入客户列表 | 打开 /customers，等待加载 | 显示客户表格 | 表格正常显示 20 条数据 | customer-list.png | PASS |
| 新增客户表单校验 | 必填项为空点击保存 | 显示必填提示 | 客户名称字段下方显示“必填” | form-validation.png | PASS |
| 导出客户列表 | 设置时间区间后点击导出 | 下载 xlsx 文件 | 已下载 customer-export.xlsx，可打开 | customer-export.xlsx | PASS |
```

### 10.3 通过 / 失败标准

- **PASS**：页面行为与预期一致，且有文本、截图、文件或请求证据支撑。
- **ISSUE**：存在可复现异常，但不阻断主流程；需记录影响范围和复现步骤。
- **BLOCKED**：因登录、权限、验证码、环境故障、数据缺失等无法验证；需说明阻塞原因和需要用户/系统提供什么。
- **FAIL**：关键路径不符合预期，且已排除页面未加载、定位错误、操作误差等因素。

## 11. 异常处理与降级策略

自动化失败时，不应反复盲目重试。建议按以下路径降级：

1. 刷新页面并重新读取页面状态；
2. 改用更稳定的语义定位；
3. 检查是否存在弹窗、遮罩、登录失效或权限变化；
4. 查看网络请求和控制台错误；
5. 用截图记录失败现场；
6. 如涉及用户确认、验证码、权限或不可逆操作，暂停等待授权；
7. 如果无法继续，交付“已完成事项 + 卡点 + 证据 + 可选下一步”。

常见失败与处理：

| 失败类型 | 可能原因 | 降级动作 |
| --- | --- | --- |
| 找不到元素 | 页面未加载、文案变化、权限不同 | 等待、读取快照、换语义定位 |
| 点击无效 | 遮罩层、按钮禁用、表单校验未过 | 检查禁用原因和必填项 |
| 输入后无响应 | 前端监听未触发、格式错误 | 改用逐字输入、失焦、检查校验提示 |
| 上传失败 | 格式/大小/模板不符、路径不可访问 | 检查文件限制、截图错误、输出失败明细 |
| 下载失败 | 异步任务未完成、浏览器拦截、权限不足 | 等待任务、检查请求、确认文件存在 |
| 提交失败 | 参数错误、接口报错、登录失效 | 检查网络请求并脱敏汇报 |
| 截图异常 | 页面未稳定、滚动位置不对 | 等待稳定后重截 |
| 登录受阻 | 验证码、扫码、二次认证 | 暂停请用户完成验证 |
| 权限不足 | 角色无入口、接口 403 | 记录页面和接口证据，建议开通权限 |

## 12. 交付契约

浏览器自动化 Skill 的最终交付应包含：

- 是否完成用户目标；
- 已访问或操作的关键页面；
- 已填写、查询、上传、下载、导出或提交的关键结果；
- 验证依据，例如页面提示、详情页字段、下载文件、截图、网络请求状态；
- 生成的截图、报告、表格或压缩包路径；
- 未完成项、失败原因、阻塞点和风险；
- 如使用联网搜索或外部资料，应给出引用；
- 文件引用必须使用 `local-file://` 加绝对路径，不使用相对路径、裸路径或 sandbox 链接；
- 多文件交付应打包为 ZIP 后引用 ZIP，避免散落多个链接。

示例交付格式：

```markdown
已完成网页表单填写与提交验证。

- 操作页面：客户管理 / 新增客户
- 提交结果：页面提示“保存成功”，列表出现客户名称“示例客户 A”
- 验证依据：详情页客户名称、手机号、客户等级均与输入一致
- 留痕截图：[提交结果截图](local-file://<workspace>/output/customer-submit-result.png)
- 导出文件：[客户导出表](local-file://<workspace>/output/customer-export.xlsx)
- 风险：本次未执行付款、删除、发布等不可逆操作；登录验证码由用户自行完成，未保存敏感凭证。
```

阻塞交付格式：

```markdown
本次未能完成最终提交，当前状态为 BLOCKED。

- 已完成：已打开目标页面并填写客户名称、联系人、手机号字段
- 卡点：点击保存后页面要求短信验证码
- 证据：[验证码弹窗截图](local-file://<workspace>/output/verification-blocked.png)
- 风险：验证码属于用户敏感验证信息，我未记录或代填
- 建议下一步：请用户完成验证码后继续，或提供测试环境免验证账号
```

## 13. 最小 Skill 结构建议

```text
browser-automation/
├── SKILL.md
├── package.json
└── README.md
```

其中：

- `SKILL.md` 必须包含可解析 YAML front matter，并写明触发条件、适用边界、操作流程和交付契约；
- `package.json` 用于描述包名、版本、入口和关键词；
- `README.md` 面向安装者说明用途、边界、风险、权限要求和示例；
- 如提供脚本或模板，应明确输入输出路径，并默认写入 `output/`。

## 14. 质量检查清单

发布或复用浏览器自动化类 Skill 前，至少检查以下事项：

### 14.1 元数据检查

- [ ] `SKILL.md` 存在 YAML front matter；
- [ ] YAML front matter 可被解析；
- [ ] `name`、`version`、`description`、`tags` 字段完整；
- [ ] `description` 明确说明触发场景和禁止场景；
- [ ] 没有把密钥、账号、Cookie、Token 写入文件。

### 14.2 能力边界检查

- [ ] 明确适用与不适用场景；
- [ ] 明确登录、验证码、权限不足时的暂停规则；
- [ ] 明确不可逆操作必须用户确认；
- [ ] 明确敏感信息脱敏和最小化原则。

### 14.3 操作流程检查

- [ ] 包含真实页面状态判断；
- [ ] 包含稳定元素定位原则；
- [ ] 包含表单填写、校验、提交和结果确认流程；
- [ ] 包含上传和下载文件处理流程；
- [ ] 包含截图留痕规范；
- [ ] 包含网络请求和控制台排查方法；
- [ ] 包含回归验证的记录模板和通过标准。

### 14.4 交付检查

- [ ] 所有交付物写入 `output/`；
- [ ] 文件命名小写英文、短横线分隔，无中文、空格、时间戳和 UUID；
- [ ] 单文件使用 `local-file://` 绝对路径引用；
- [ ] 多文件打包为 ZIP 后引用；
- [ ] 结论包含完成状态、证据、风险和未完成项；
- [ ] BLOCKED / ISSUE / FAIL 情况有复现步骤、截图或请求证据。

### 14.5 回归检查

- [ ] 至少覆盖关键路径，而不是只验证页面能打开；
- [ ] 截图前等待页面稳定；
- [ ] 下载文件已确认存在且可读；
- [ ] 上传结果已确认成功或失败原因明确；
- [ ] 控制台和网络请求中的关键错误已检查并脱敏记录；
- [ ] 未因自动化失败而反复无效重试。
