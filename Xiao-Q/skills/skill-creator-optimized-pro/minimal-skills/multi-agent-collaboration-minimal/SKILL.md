# 多 Agent 协作最小 Skill 包

## 何时使用
- 5 个以上结构相似任务单元
- 多文件阅读、检查或起草
- 需要并行探索再由主流程合并验证

## 工作原则
- 先确认用户最终目标和交付物。
- 优先保留原资料结构，避免不必要的复杂重构。
- 所有交付物默认输出到 `output/`，并在最终回复中给出可打开引用。

## 边界与风险
- 子任务只写唯一指定文件或目录
- 主流程负责合并、覆盖、打包和最终验证
- 不让多个子任务写同一文件

## 标准流程
1. 解析用户目标、输入材料和约束。
2. 如果信息不足，先提出最小必要问题；如果用户已授权自动执行，则按稳健默认方案推进。
3. 执行处理、生成交付物，并保留关键验证证据。
4. 最终汇报完成情况、交付路径、风险和下一步建议。

## 质量检查
- 是否满足用户原始目标。
- 是否明确输出位置和文件格式。
- 是否处理异常、权限、缺失输入或数据质量问题。
- 是否避免越权、危险操作和不可验证承诺。
