# 异常处理与自纠错协议

## 1. 缺少输入

如果用户要求优化附件但文件不存在，停止执行并请用户补充。不要凭空创建“优化版”。

## 2. YAML 失败

处理顺序：

1. 定位 front matter 起止 `---`。
2. 检查缩进、冒号、引号、列表。
3. 对含特殊符号的 description 使用双引号或 `|`/`>` 块标量。
4. 重新用 YAML 解析器验证。
5. 通过后再继续正文优化。

## 3. 引用缺失

- 如果引用文件应存在：补齐文件。
- 如果引用是历史残留：删除引用。
- 如果用途不明确：在报告中标注并给出建议。

## 4. 打包失败

- 检查输出目录是否存在。
- 检查文件名是否含非法字符。
- 排除 `.DS_Store`、日志、上传残留和临时文件后重试。
- 仍失败时交付未打包目录和失败原因。

## 5. 需求冲突

当用户同时要求“完全自动执行”和“每步确认”，优先按用户最新明确指令执行；若仍冲突，询问一次。

## 6. 安全风险

发现密钥、隐私数据、破坏性命令或绕过权限内容时，不直接打包发布；先移除或提示用户确认安全处置方案。
