# 优秀 Skill 方法论

## 1. 一句话定义

优秀 Skill 不是“长提示词”，而是一套面向 AI Agent 的**可复用任务执行系统**：它告诉 Agent 何时触发、负责什么、不负责什么、如何读取资源、如何执行、如何验证、如何交付。

## 2. 三层加载模型

| 层级 | 内容 | 作用 | 写作要求 |
|---|---|---|---|
| Level 1 | YAML front matter：`name` + `description` | 常驻上下文，决定是否触发 | 极短、具体、包含触发/排除边界 |
| Level 2 | `SKILL.md` 正文 | 触发后加载，定义主流程 | 精简、可执行、避免堆背景 |
| Level 3 | `references/`、`scripts/`、`assets/` | 执行中按需读取 | 长资料、模板、示例、校验脚本放这里 |

核心原则：**不是让 AI 知道更多，而是让 AI 在恰当时间知道恰当的事。**

## 3. 优秀 Skill 的 10 条标准

1. **问题真实且高频**：来自重复出现的业务/工作场景，而不是为了炫技。
2. **边界足够窄**：解决一类问题；过宽时拆分或路由。
3. **description 精准**：同时写清能力、触发词、典型表达、排除场景。
4. **目标清楚**：开头说明要交付什么，做到哪里算完成。
5. **流程可执行**：给阶段、决策分支和检查点，而不是抽象说教。
6. **输出有契约**：格式、文件、命名、验收标准都明确。
7. **资料分层**：主文件保留必要规则，长内容放 references。
8. **确定性动作工具化**：校验、解析、批处理、打包等用脚本或明确门禁兜底。
9. **错误可恢复**：缺文件、权限失败、YAML 错误、需求冲突都有处理策略。
10. **发布可审计**：元数据一致、引用存在、无临时文件、附测试 Prompt 和变更说明。

## 4. Description 写法

推荐公式：

> 执行什么任务；当用户出现哪些自然表达/文件/场景时触发；覆盖哪些关键动作；不要用于哪些相邻但不属于本 Skill 的任务。

反例：

```yaml
description: 帮助创建技能
```

正例：

```yaml
description: "创建、优化、审计和打包 AI Skill 技能包；当用户要求创建技能、生成 Skill、优化 SKILL.md、修复技能包结构、补齐 YAML front matter、打包发布 SkillHub，或把业务流程沉淀为可安装技能时使用。不要用于一次性提示词润色、普通文档写作或与 Skill 无关的软件开发任务。"
```

## 5. SKILL.md 主体骨架

```markdown
# Skill 名称

## 目标
## 何时使用 / 何时不要使用
## 输入与前置检查
## 工作流
## 输出契约
## 质量门禁
## 异常处理
## 资源索引
```

## 6. 创建 Skill 的推荐流程

1. **场景洞察**：确认问题是否真实、高频、值得复用。
2. **需求访谈**：收集目标、触发表达、排除边界、输入、输出、完成标准。
3. **流程试跑**：复杂 Skill 先跑通真实业务流程，再写成规范。
4. **结构设计**：决定是否需要 references、scripts、assets。
5. **正文编写**：主文件只放执行必需内容。
6. **静态验证**：YAML、路径、元数据、临时文件、引用一致性。
7. **触发测试**：准备应触发、不应触发、边界三类 Prompt。
8. **真实迭代**：记录 Gotchas 和成功范式，持续更新。

## 7. 发布前门禁

- YAML front matter 可解析。
- description 能通过正反触发测试。
- 正文没有大段无执行价值的背景介绍。
- 引用路径和实际文件一致。
- README 与 package metadata 一致。
- 不含密钥、日志、上传残留、`.DS_Store`。
- zip 解压结构可直接安装。

## 8. 针对 Skill Creator 类技能的特别要求

Skill Creator 是“元 Skill”，它本身会影响后续所有 Skill 的质量，因此必须额外关注：

- 自动检查并修复 YAML front matter；不仅看是否存在，还要实际解析。
- description 要包含“创建/优化/审计/打包/SkillHub”等触发词，也要排除“一次性提示词”。
- 输出必须包括完整文件包，而不是只给一段 SKILL.md。
- 默认生成测试 Prompt 和质量清单。
- 对已有包优化时，优先保留原结构，只修复影响质量和发布的关键问题。
