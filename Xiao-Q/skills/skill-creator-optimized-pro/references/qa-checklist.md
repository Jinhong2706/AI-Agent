# Skill 包质量门禁清单

## A. Front matter

- [ ] `SKILL.md` 以 `---` 开始，并有闭合的 `---`。
- [ ] YAML 可被解析，不依赖肉眼判断。
- [ ] 至少包含 `name` 和 `description`。
- [ ] `name` 使用小写字母、数字、连字符。
- [ ] `description` 具体说明触发场景和排除场景。
- [ ] 含冒号、列表、特殊符号的 description 使用引号或块标量。

## B. 主体内容

- [ ] 目标明确：说明最终交付物。
- [ ] 使用边界明确：何时用、何时不用。
- [ ] 工作流可执行：有阶段、动作、分支和检查点。
- [ ] 输出契约明确：格式、命名、文件、报告要求。
- [ ] 完成标准可打勾验证。
- [ ] 异常处理包含缺文件、权限、解析、打包失败。

## C. 渐进式加载

- [ ] 主 `SKILL.md` 不堆长模板和大段背景。
- [ ] 长示例、模板、规范放入 `references/`。
- [ ] 确定性动作优先脚本化或写入明确校验逻辑。
- [ ] `SKILL.md` 中引用的本地文件实际存在。

## D. 发布包

- [ ] README 与 SKILL.md 名称、版本、用途一致。
- [ ] package metadata 与 SKILL.md 一致。
- [ ] 不含 `.DS_Store`、上传残留、日志、临时文件、密钥。
- [ ] zip 解压后目录结构清晰，可直接安装。
- [ ] 附至少 5 条测试 Prompt：2 条应触发、2 条不应触发、1 条边界。

## E. 触发测试 Prompt 模板

### 应触发

1. “帮我创建一个用于周报生成的 Skill。”
2. “优化这个 SKILL.md，补齐 front matter 并打包。”

### 不应触发

1. “帮我润色这段普通提示词，不需要做成 Skill。”
2. “帮我分析这个 Excel 数据。”

### 边界

1. “我有一套固定 SOP，想看看是否值得沉淀成 Skill。”
