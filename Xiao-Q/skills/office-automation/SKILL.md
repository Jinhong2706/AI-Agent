# 全能办公自动化流水线助手

## 概述

本技能面向全行业职场上班族、行政人员、运营人员、管理人员及文职人员，提供一站式AI办公提效能力。技能严格按照真实职场办公流程设计，输出风格正式严谨、商务得体、简洁干练，适配各大中小型企业日常办公使用。

### 核心能力

| 模块 | 功能 | 适用场景 |
|------|------|---------|
| 工作总结自动生成 | 一键生成正式汇报版、精简快捷版、领导审阅版三种工作总结 | 周报/月报/季度汇报/年度述职 |
| 会议纪要智能整理 | 将会议记录、聊天记录、语音转文字长文本自动拆分为结构化会议纪要 | 部门会议/项目会议/跨部门沟通/远程会议 |
| PPT全套文案生成 | 自动生成封面标题、目录、正文、结尾话术及现场演讲词 | 工作汇报/项目路演/部门述职/竞聘演讲 |
| 日常公文速成 | 各类企业内部通知、通报、简报、计划、述职报告智能撰写 | 行政管理/人事通知/部门通报/季度规划 |

### 何时使用本技能

- 用户需要撰写工作总结、周报、月报、季度报告或年度述职
- 用户提供了会议聊天记录、语音转文字长文本或会议草稿，需要整理为正式会议纪要
- 用户需要生成PPT文案，包括封面、目录、正文及演讲词
- 用户需要起草企业日常公文（通知、通报、简报、计划、述职报告等）
- 用户提到"办公""汇报""会议纪要""公文""PPT文案""工作总结"等关键词

## 模块一：工作总结自动生成

### 功能描述

根据用户输入的岗位名称、本周完成工作、未完成工作、下周工作计划等基础信息，自动梳理工作逻辑，合理扩充内容，排版分段清晰，生成三种版本的工作总结。

### 输入要求

| 字段 | 说明 | 必填 |
|------|------|------|
| 岗位名称 | 用户的职务/岗位，如"市场部运营专员""研发部高级工程师""行政主管" | 是 |
| 汇报周期 | 周报/月报/季度/年度 | 是 |
| 本周完成工作 | 已完成的工作事项列表，支持短句或长文本 | 是 |
| 未完成工作 | 计划但未完成的事项，说明原因 | 否 |
| 下周工作计划 | 下周/下阶段的工作安排 | 否 |
| 需要协调事项 | 需要上级或其他部门支持的事项 | 否 |
| 补充说明 | 其他需要写入汇报的内容 | 否 |

### 输出规范

#### 版本一：正式汇报版（详版）

适用于正式邮件汇报、OA系统提交、部门存档。结构如下：

```
【20XX年X月X日 — X月X日工作总结】

一、本周工作完成情况
1. 重点工作一：标题概括
   内容描述（含量化数据、完成进度、取得成效）

2. 重点工作二：标题概括
   内容描述（含量化数据、完成进度、取得成效）

3. 常规工作：标题概括
   内容描述

二、未完成工作及原因分析
1. 事项名称 — 未完成原因 — 当前进度 — 预计完成时间

三、下周工作计划
1. 优先级一：事项名称及预期目标
2. 优先级二：事项名称及预期目标

四、需要协调与支持
1. 需协调事项及建议方案

汇报人：[姓名]
日期：20XX年X月X日
```

#### 版本二：精简快捷版

适用于即时通讯工具快速同步、站会汇报。结构如下：

```
本周完成：
1. xxx（进度xx%）
2. xxx（已上线/已完成）
3. xxx（推进中）

未完成：
- xxx → 原因：xxx → 预计X月X日完成

下周计划：
1. xxx
2. xxx

需协调：xxx
```

#### 版本三：领导审阅版

适用于向直属上级或高层领导简要汇报，突出关键结果和风险。结构如下：

```
【汇报摘要】

核心产出：
1. xxx — 达成xxx效果/数据
2. xxx — 上线xxx/完成xxx

关键风险/阻塞：
- xxx事项：当前xxx状态，需xxx支持

下阶段重点：
1. xxx（预期xxx日前完成）
2. xxx

一句话总结：[一句话概括本周最核心成果]
```

### 生成规则

1. **逻辑梳理**：将用户输入的碎片化信息按重要性排序，合并同类项，拆分过大的工作项
2. **内容扩充**：在用户原文基础上，合理补充工作背景、执行过程、量化成果、产生影响等要素，使内容丰满但不虚构
3. **语言优化**：使用商务正式语言，避免口语化表达，修正语句语病
4. **数据量化**：引导用户在输入时提供量化数据；若用户未提供，在生成内容中注明"建议补充具体数据"
5. **空字段处理**：若用户未提供"未完成工作"或"需协调事项"，对应章节标注"无"或省略

## 模块二：会议纪要智能整理

### 功能描述

支持用户直接粘贴会议聊天记录、语音转文字长文本、会议草稿笔记，智能拆分为结构化会议纪要，条理清晰，一目了然。

### 输入要求

| 字段 | 说明 | 必填 |
|------|------|------|
| 会议原始内容 | 聊天记录、语音转文字、会议笔记草稿等 | 是 |
| 会议名称 | 会议主题/名称 | 否（可从内容推断） |
| 会议时间 | 会议召开时间 | 否（可从内容推断） |

### 输出规范

```markdown
# [会议名称] 会议纪要

## 基本信息
- 会议主题：
- 会议时间：
- 会议形式：线上/线下/混合
- 主持人：
- 记录人：
- 参会人员：

## 一、会议议题
1. 议题一：xxx
2. 议题二：xxx
3. 议题三：xxx

## 二、讨论要点

### 议题一：xxx
- 讨论内容概述
- 各方主要观点
- 存在分歧（如有）

### 议题二：xxx
- 讨论内容概述
- 各方主要观点
- 存在分歧（如有）

## 三、会议决议
1. 决议一：xxx（通过/有条件通过）
2. 决议二：xxx（待进一步讨论）

## 四、待办任务清单

| 序号 | 任务内容 | 执行负责人 | 完成截止时间 | 优先级 | 备注 |
|------|---------|-----------|-------------|--------|------|
| 1 | xxx | 张三 | 20XX.XX.XX | 高 | |
| 2 | xxx | 李四 | 20XX.XX.XX | 中 | |

## 五、遗留问题
1. 问题一：xxx — 建议解决方案：xxx
2. 问题二：xxx — 待下次会议继续讨论

## 六、下次会议计划
- 预计时间：
- 预定议题：

纪要整理：[姓名]
整理日期：20XX年X月X日
```

### 提取与整理规则

1. **会议主题推断**：若用户未明确提供会议主题，从对话/文本中提取核心讨论事项作为会议主题
2. **参会人员识别**：从聊天记录中自动识别发言者姓名或昵称，整理为参会人员列表
3. **议题拆分**：识别对话中的话题切换点，将内容拆分为独立议题，每个议题概括核心讨论要点
4. **决议提取**：识别会议中达成共识的决策点、投票结果、最终结论，归纳为明确决议
5. **待办任务提取**：识别对话中的任务分配语句（如"xxx你负责""xxx就交给xxx了""下次xxx之前完成"），整理为结构化待办清单，自动标注执行负责人和截止时间
6. **优先级判定**：根据对话中的紧迫性表述（如"紧急""尽快""优先级高"）自动标注任务优先级
7. **遗留问题捕获**：识别未解决的争议、悬而未决的议题、需要外部信息才能推进的事项
8. **容错处理**：对于内容过短或质量过低的输入，提示用户补充更多信息；对于无法识别的部分，标注"待确认"

## 模块三：PPT全套文案生成

### 功能描述

输入汇报主题、使用场景、汇报对象，自动完整生成一套可直接使用的PPT文案，包括封面、目录、正文逐页内容、结尾总结话术及现场口头演讲主持词。

### 输入要求

| 字段 | 说明 | 必填 |
|------|------|------|
| 汇报主题 | PPT的核心主题，如"2026年Q1市场部工作汇报""新产品上线推广方案" | 是 |
| 使用场景 | 周会/月会/季度汇报/年度述职/项目路演/竞聘演讲/培训分享等 | 是 |
| 汇报对象 | 直属上级/部门全体/公司高层/外部客户/投资人等 | 是 |
| 汇报时长 | 预计演讲时长（分钟） | 否（默认15分钟） |
| 核心数据/亮点 | 需要在PPT中突出的关键数据或亮点成果 | 否 |
| 特殊要求 | 如品牌色调、公司模板风格提醒等 | 否 |

### 输出规范

```markdown
# PPT全套文案：[汇报主题]

## 封面页
- 主标题：
- 副标题：
- 汇报人/部门：
- 汇报日期：

## 目录页
1. 背景与概述
2. 核心成果/主要内容
3. 数据分析与洞察
4. 问题与反思
5. 下阶段规划
6. 总结与展望

## 正文逐页内容

### 第1页：背景与概述
- 页面标题：
- 核心要点（3-5条）：
- 建议配图/图表：

### 第2页：核心成果一
- 页面标题：
- 核心要点（3-5条）：
- 关键数据：
- 建议配图/图表：

### 第3页：核心成果二
...

### 第N页：数据分析
- 页面标题：
- 数据呈现方式建议（柱状图/折线图/饼图等）：
- 核心要点（3-5条）：

### 第N+1页：问题与反思
- 页面标题：
- 存在问题（2-3条）：
- 改进思路：

### 第N+2页：下阶段规划
- 页面标题：
- 规划要点（按优先级排列）：
- 时间节点：

## 结尾总结页
- 核心结论（一句话）：
- 致谢语：
- 联系方式/二维码建议：

## 现场演讲主持词
### 开场白（约x分钟）
### 正文讲解话术（逐页对应）
### 过渡串词（页与页之间）
### 收尾总结（约x分钟）
### 应对提问的预备话术
```

### 生成规则

1. **标题提炼**：封面主标题精炼有力，不超过15字；副标题补充说明场景和范围
2. **内容分层**：每页正文控制在3-5个要点，每个要点不超过2行文字，遵循"一页一个核心观点"原则
3. **数据可视化建议**：根据内容自动建议最优图表类型（趋势→折线图，占比→饼图，对比→柱状图，分布→雷达图等）
4. **演讲词适配**：根据汇报对象和场景调整话术风格（高层汇报→简洁结论先行，团队内部分享→可适度展开，对外路演→专业商务感强）
5. **时长控制**：根据PPT页数和汇报时长自动分配每页讲解时间，写入主持词中的时间提示
6. **过渡串词**：为每页之间的切换设计自然过渡语，避免生硬翻页

## 模块四：日常公文速成

### 功能描述

支持企业内部通知、工作通报、部门简报、季度计划、年度述职等各类通用办公文案智能撰写，自动修正语句语病，统一办公格式。

### 支持的公文类型

| 公文类型 | 适用场景 | 核心要素 |
|---------|---------|---------|
| 内部通知 | 制度变更、活动通知、人事任免、放假安排 | 标题-正文-落款-日期 |
| 工作通报 | 项目进度通报、业绩通报、表彰/批评通报 | 事由-事实-评价-决定-要求 |
| 部门简报 | 部门阶段性工作汇总分享 | 工作动态-数据亮点-下步计划 |
| 季度计划 | 季度/月度工作规划部署 | 目标-任务分解-时间节点-责任人 |
| 年度述职 | 年度工作述职报告 | 履职概况-主要业绩-问题反思-改进计划-未来展望 |
| 会议通知 | 会议召集通知 | 会议主题-时间地点-参会人员-议程-会议要求 |
| 申请报告 | 资源申请、项目立项申请 | 申请事由-必要性-方案-预算-预期效益 |

### 输入要求

| 字段 | 说明 | 必填 |
|------|------|------|
| 公文类型 | 通知/通报/简报/计划/述职等 | 是 |
| 公文主题 | 公文的核心内容概述 | 是 |
| 关键信息 | 涉及的时间、人物、事件、数据等 | 是 |
| 收文对象 | 面向哪些部门/人员 | 否 |
| 发文部门 | 发布公文的部门名称 | 否 |
| 特殊要求 | 字数要求、格式偏好等 | 否 |

### 输出规范

每种公文类型按照国家标准公文格式和企业通用行文规范输出：

#### 内部通知格式示例

```
关于[事项]的通知

[收文对象]：

[正文：背景 + 具体事项 + 执行要求 + 时间节点，分段书写]

特此通知。

[发文部门]
20XX年X月X日
```

#### 季度计划格式示例

```
[部门名称]202X年第X季度工作计划

一、工作目标
[本季度核心目标概述，2-3条]

二、重点任务
（一）任务一：任务名称
预期成果：
时间节点：
责任人：

（二）任务二：任务名称
预期成果：
时间节点：
责任人：

三、资源需求
[所需资源列表]

四、风险评估与应对
[主要风险及应对策略]

[发文部门]
20XX年X月X日
```

#### 年度述职格式示例

```
[姓名] 20XX年度述职报告

一、履职总体情况
[岗位职责履行总体概述，2-3段]

二、年度主要业绩
1. 业绩指标一：概述 + 数据成果
2. 业绩指标二：概述 + 数据成果
3. 重点项目：项目名称 + 角色 + 成果

三、问题与反思
1. 问题一：表现 + 原因分析
2. 问题二：表现 + 原因分析

四、改进措施与规划
1. 改进方向一：具体措施
2. 改进方向二：具体措施

五、未来工作展望
[未来工作方向和目标]

述职人：[姓名]
20XX年X月X日
```

### 生成规则

1. **格式统一化**：所有公文自动套用标准企业公文格式，包含标准标题、称谓、正文、落款、日期
2. **语言规范化**：使用正式书面语体，自动修正口语化表达、语法错误及不当用词
3. **结构完整化**：根据公文类型自动补充必要段落，确保逻辑闭环（如通知类必须有"特此通知"收尾）
4. **术语标准化**：涉及行政、人事、财务等专业领域时，使用标准行业术语
5. **排版优化**：自动分段、自动编号层级分明（一级用"一、二、三"，二级用"（一）（二）（三）"，三级用"1. 2. 3."）

## 交互使用规则

### 触发方式

用户无需记忆复杂指令，直接以自然语言输入即可。系统根据输入内容和关键词自动判断用户意图，触发对应功能模块。

**触发关键词速查：**

| 功能模块 | 触发关键词 |
|---------|-----------|
| 工作总结自动生成 | 工作总结、周报、月报、季度汇报、年终总结、述职报告、"帮我写一份工作总结" |
| 会议纪要智能整理 | 会议纪要、会议记录、整理会议、会议整理、"帮我整理会议" |
| PPT全套文案生成 | PPT、幻灯片、演示文稿、汇报PPT、演讲词、"帮我做一份PPT""生成PPT文案" |
| 日常公文速成 | 通知、通报、简报、计划、公文、"帮我写一份通知""起草一份通报" |

### 多模块自动组合

若用户输入内容同时涉及多个模块（如"帮我整理会议纪要，并生成PPT，最后写个工作总结"），则按"会议纪要→PPT文案→工作总结"的顺序依次输出，保证内容连贯性和逻辑一致性。

### 输入模式支持

1. **短句快速模式**：适用于熟悉技能的用户，直接输入关键信息即可触发
   - 示例："周报 产品经理 完成需求文档评审和原型设计"
2. **长文本批量粘贴模式**：适用于会议记录、语音转文字等大量文本的处理
   - 示例：直接粘贴大段聊天记录或会议笔记

### 容错与引导机制

1. **必填信息缺失**：当用户输入的必填信息不完整时，主动列出缺失项并以友好方式引导补充
   - 示例："已收到您的周报撰写请求。为确保内容完整，请补充以下信息：① 您的岗位名称 ② 本周已完成的关键工作"
2. **内容模糊时追问**：当用户输入过于笼统（如仅输入"写个总结"），引导用户提供更多细节
3. **多意图识别**：识别到多个潜在意图时，列出可能的功能选项供用户确认
4. **输出长度自适应**：根据用户提供的信息量自动调整输出详细程度，信息越多输出越详尽

## 输出质量标准

### 语言风格

- **正式严谨**：使用商务书面语，避免口语化、网络化表达
- **简洁干练**：段落精炼，无冗余修饰，每句话都有信息量
- **积极正向**：表述客观但不消极，突出解决问题的态度
- **逻辑清晰**：因果关系明确，层次分明，过渡自然

### 禁止内容

- 不生成娱乐化、低俗化文案内容
- 不包含违规职场话术或不当表述
- 不改写或扭曲用户提供的客观事实和数据
- 不添加与用户输入无关的推测性内容
- 不在公文中使用emoji、表情符号、网络流行语

### 格式统一标准

- 标题：使用标准公文标题格式，不加粗修饰符号
- 段落：段首不空格，段间空一行
- 层级编号：一级用"一、二、三"，二级用"（一）（二）（三）"，三级用"1. 2. 3."
- 日期：统一使用"20XX年X月X日"格式
- 数字：正文中涉及统计数据时使用阿拉伯数字，约数使用中文数字
