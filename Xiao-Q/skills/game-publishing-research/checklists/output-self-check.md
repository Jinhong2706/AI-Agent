# 产出自检清单

每次写完一份 MD / HTML 之前必跑此清单。逐项打钩，确保不漏。

## 通用自检（所有产出）

### 写作风格
- [ ] 一句话先行（每路径 / 每案例先一句话讲清楚）
- [ ] 中文名优先（所有海外游戏用中文名）
- [ ] 路径前缀（导航 / 标题用「路径 N · 游戏中文名」格式）
- [ ] 决策选项化（不开放式提问，给 A/B/C 选项）
- [ ] 反面教材只反衬（不单独立章）
- [ ] 写作偏正式汇报风（标题避免口语化表达）

### 真实性 / 来源
- [ ] 每个具体数字都有可点开的来源链接
- [ ] 来源优先级遵守（官方 IR > 平台官方 > 权威数据源 > 主流媒体 > 二级讨论）
- [ ] 二级讨论只作线索，不作定论
- [ ] 未确认归因已做真实性降级（"行业普遍说法是 X，但 Y 未公开确认"）
- [ ] 关键传播素材原话保留（HTML 用 `<span class="highlight">` 黄底高亮）



### 文件命名
- [ ] 文件名遵守命名规范（参考 `checklists/写作硬规则.md` 第 15 条）
- [ ] 包含 Agent 编号便于识别

---

## 单案分析专项自检

- [ ] 元信息头完整（厂商、品类、平台、首发时间、定价）
- [ ] 核心问题一句话
- [ ] 关键动作具体（不是模糊话）
- [ ] 数据点表格附来源
- [ ] 对发行方的参考意义已写（可借鉴 + 不可复制两点）
- [ ] 数据来源区块至少 3 条权威源

---

## 综合洞察专项自检

- [ ] 分类轴是从案例数据归纳的，不是套上一次的
- [ ] 速览表 + 详细展开两层都有
- [ ] 每条路径都有 2+ 案例支撑
- [ ] 每个案例的"关键动作"够具体
- [ ] 关键传播素材原话保留并黄底高亮
- [ ] 共通打法 5 条左右、不是 10+ 条
- [ ] 反面教材没有单独立章

---

## HTML 集合页专项自检

### 文件结构
- [ ] 单文件无外部依赖（CSS / JS / 图片都内嵌）
- [ ] 双击可直接打开
- [ ] 所有视图 ID 与导航 data-view 对应

### 视觉规范
- [ ] 微软雅黑 / Arial 字体
- [ ] 浅灰几何背景
- [ ] 暗色 sidebar
- [ ] Hero 渐变色按视图区分
- [ ] 路径卡彩色左边框
- [ ] 结论框绿底 + 💡
- [ ] 警告框红底 + ⚠️
- [ ] 关键素材黄底高亮
- [ ] 表格 hover 行高亮
- [ ] 案例库 emoji 区分（🎮 F2P / 🕹️ 付费 3A / 🎯 应用层）

### 交互
- [ ] 侧边栏切换视图正常
- [ ] 子锚点平滑滚动正常
- [ ] 移动端响应式（如要求）

### 案例视图独立页
- [ ] 每个案例都有 Hero 区
- [ ] 核心问题 + 策略总结表 + 关键动作详解 + 启示 + 数据来源 五段齐全
- [ ] 关键传播素材黄底高亮

## 用户复核节点

写完一个阶段不要立刻进入下一个阶段，让用户先过一遍：

- 立题阶段 → 用户确认研究边界
- 案例池 → 用户检查是否有遗漏 / 冗余
- 单案分析每 5-10 份 → 用户检查中文名 / 事实 / 风格
- 综合洞察速览表 + 路径 1 → 用户确认分类轴
- 综合洞察全完成 → 用户拍板进入交付物阶段
- 交付物 HTML / PPT → 用户最终验收

不要跳过复核节点——后期返工成本远高于前期对齐。
