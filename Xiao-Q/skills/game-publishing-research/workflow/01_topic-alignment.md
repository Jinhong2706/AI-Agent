# 立题清单（阶段 1）

研究开始前必须对齐以下 5 个问题。**不要在立题没对齐前进入案例池构建阶段**。

## 必问问题（按顺序追问，决策选项化呈现）

### 1. 研究目标：要回答什么问题？给谁看？

追问示例：
> 这次研究你想回答的核心问题是什么？以下三类常见目标供参考：
> A. 复盘型——回答"过去某些游戏怎么做火的"，用于发行方学习方法
> B. 决策型——回答"我们这款游戏该走哪条路径"，用于具体项目策略制定
> C. 分类型——回答"行业内哪些发行模式存在 / 各自适合什么类型产品"，用于建立行业心智地图

读者是谁？（发行 PM / leader / 跨部门同事 / 外部合作方？）—— 不同读者决定语言风格和详略。

### 2. 边界：哪些游戏 / 平台 / 时间窗口 / 地区算在内？

强制问清：
- **品类**：F2P / 付费 3A / 独立 / 手游 / 主机 / PC？还是某个具体子类（如 PvP 射击）？
- **平台**：PS5 / Xbox / Switch / 移动端 / Steam / 其他？
- **时间窗口**：近 3 年？近 5 年？某个具体起止时间？
- **地区**：海外 / 国内 / 全球？某个具体市场（日韩 / 欧美 / 东南亚）？
- **数量预期**：覆盖 5-10 个还是 30+ 个案例？

### 3. 分类轴假设：用户已有想法吗？

询问但不固化：
> 你心里已经有想用什么维度分类了吗？例如按平台分、按品类分、按发行动作类型分、按厂商规模分。
> 如果已有假设，先记录下来作为初版假设；但**最终分类轴必须在跑完案例池后从数据里归纳**，不一定保留你的初版假设。

### 4. 输出形式：MD / HTML / PPT / 智能表？

追问：
- 单份 MD 综合洞察？
- 多份单案 MD + 综合 MD？
- HTML 集合页（含左侧导航 + 多视图切换）？
- PPT 汇报稿？
- 智能表格（结构化字段）？

不同输出形式决定写作粒度和视觉投入。

### 5. 是否需要应用层（用研究结论给具体产品做策略建议）？

追问：
> 这次研究是否要落到一个具体产品的发行建议上？例如"基于研究结论给 X 游戏建议下个月该怎么做"。
> 如果是，需要单独跑一份"应用层策略文档"，参考 `templates/应用层策略模板.md`。

## 立题对齐输出物

立题阶段对齐后，必须输出一份《研究立题确认》MD（一页纸即可），包含：

```markdown
# 研究立题确认

## 1. 研究目标
（一句话回答要解决什么问题、给谁看）

## 2. 边界
- 品类：
- 平台：
- 时间窗口：
- 地区：
- 案例数量预期：

## 3. 分类轴假设（不固化）
（用户初版假设，仅作起点）

## 4. 输出形式
（MD / HTML / 数量与结构）

## 5. 是否含应用层
（是 / 否，如是则指明应用产品）

## 6. 阶段计划
- 阶段 2 案例池：预计 X 个案例
- 阶段 3 单案分析：预计 Y 个完整 MD
- 阶段 4 综合洞察：预计 Z 条路径或共性
- 阶段 5 交付物：（如有）
```

用户确认后再进入阶段 2。
