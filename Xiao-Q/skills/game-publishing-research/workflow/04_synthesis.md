# 综合洞察（阶段 4 · Agent 3 角色）

综合洞察是从多个单案里**归纳跨案规律**。这是分类轴诞生的阶段——也是最容易出错的阶段。

## ⚠️ 阶段 4 的最大陷阱：直接套用上一次的分类

**绝对禁止**直接拿上一次研究归纳出来的路径分类去套这次的案例。

正确的归纳流程：
1. **不带预设**地把所有单案分析摊开来对比
2. 观察哪些维度真正有差异（厂商规模？品类？平台？发行动作？传播驱动力？）
3. 选出 1-2 个最有解释力的维度作为分类轴**初版**
4. 用初版分类轴试着把案例归类——看能不能解释 80% 的案例
5. 如果不能，回到第 2 步重选维度
6. 第一版分类轴出来后，**敢于推翻重排**——分类轴可以多次迭代

## 分类轴迭代法（核心方法论）

历史经验：一次研究的分类轴往往要经过 2-3 次迭代才能稳定。例如某次主机推广研究项目里：
- 第 1 版：按平台分（Sony / Nintendo / Microsoft）—— 失败，平台间动作太多重叠
- 第 2 版：按资源类型分（线上商店 / 线下展会 / 硬件实体）—— 部分有用，但混淆了"曝光手段"和"策略选择"
- 第 3 版：按发行动作类型分（产品力驱动 / 长线打磨 / 第一方独占 / 文化输出 / 诚意驱动）—— 最终稳定，每条路径有明确判断标准

**敢于推翻重排是这个阶段最重要的能力**。

## 写作硬规则

### 一句话先行
每个路径、每个案例、每个共通打法都先一句话讲清楚是什么，再展开细节。读者瞄一眼就能抓住核心。

### 决策选项化呈现
对发行方建议时，不要开放式说"可以考虑做 XX"，而是给 A/B/C 选项让用户选。

### 反面教材只反衬
反面教材（翻车的案例）**不单独立章**。它们应该顺带出现在某条路径里作为反衬：
- 例如路径 1（产品力驱动）里反衬"超凡大师 PvP 多人付费 11 天关服"
- 例如定价杠杆共通打法里反衬"自杀小队预购 4 档 + 季票 = 信任崩塌"

如果反面教材太多需要专门讨论，可以放在最后一个章节叫"反面教材清单"，但不能与正面路径平级。

### 中文名 + 路径前缀
所有案例统一中文名。导航/索引中用"路径 N · 游戏中文名"格式。

## 综合洞察阶段的关键问题（自检）

写完后逐项检查：

- [ ] 分类轴是从案例数据归纳的，不是套上一次的吗？
- [ ] 每条路径都有 2+ 案例支撑吗？
- [ ] 速览表 + 详细展开两层都有吗？
- [ ] 每个案例的"关键动作"够具体（不是"做了平台合作"这种模糊话）吗？
- [ ] 关键传播素材原话保留并黄底高亮了吗？
- [ ] 共通打法 5 条左右、不是 10+ 条吗（太多就稀释）？
- [ ] 反面教材没有单独立章吗？
- [ ] 决策选项化、不是开放式建议吗？
- [ ] 中文名和路径前缀全统一吗？
- [ ] 来源链接齐全吗？

## 用户迭代节点

综合洞察容易和用户预期不一致——分类轴常常需要用户帮忙判断。建议节奏：
1. 写完速览表 + 路径 1 详细展开 → 让用户确认分类轴
2. 用户认可后再写其他路径
3. 全部写完后再让用户看共通打法
4. 最后用户拍板是否需要进入阶段 5（交付物视图）
