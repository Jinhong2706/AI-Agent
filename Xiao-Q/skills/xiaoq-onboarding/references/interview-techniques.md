# 采访话术与漏斗式追问法

本文件提供 Block 2 和 Block 3 的对话话术模板和追问技巧，确保采访过程自然、高效、不像填表。

---

## 一、情境引导开场白（用于 Block 2）

### 核心原则

- **从一个自然的问题开始**，而不是列出清单
- **顺藤摸瓜**，根据用户的回答追问，而非按预设问题逐个问
- **每次只问 1 个问题**，等用户回答后再问下一个

### 开场白模板

根据对话氛围选择合适的开场：

| 场景 | 开场白 |
|---|---|
| 通用 | 「我想更了解你，这样以后帮你的时候能更到位。平时你主要做什么呀？」 |
| 轻松 | 「聊两句呗——你是学生还是打工人？」 |
| 直接 | 「先让我知道你是谁，这样我才能帮到你。你怎么称呼？」 |
| 延续对话 | 「对了，还没问你呢，你平时是做什么的？」 |

### 追问路径示例

```
用户："我在上学"
→ "什么专业呀？"
  → 用户："学设计的"
    → "设计蛮有意思的！平时会找我帮你做设计相关的事吗？"
      → 用户："偶尔让我帮你画图"
        → 记录：身份=学生，专业=设计，使用场景=画图
```

```
用户："上班打工的"
→ "做什么方向的？"
  → 用户："互联网产品经理"
    → "那挺忙的吧。平时找我主要想省时间，还是找灵感？"
      → 记录：身份=产品经理，行业=互联网，核心需求=效率/创意
```

### ❌ 户口登记式反例（严禁模仿）

- ❌ 「请告诉我你的姓名、年龄、职业、爱好。」
- ❌ 「你是学生还是上班族？你的专业是什么？你用什么操作系统？」
- ❌ 「以下信息请逐项填写：称谓___、职业___、使用场景___」
- ❌ 连续问超过 2 个问题不给用户喘息空间

### ✅ 正确示范

- ✅ 「你怎么称呼？」→ 等回答 → 「小明是吧，你平时主要做什么呀？」
- ✅ 「了解了！还有什么你觉得我应该知道的吗？」（开放结尾，让用户自己补充）

## 二、漏斗式追问法（用于 Block 3）

### 核心四步

```
广开口 → 锁场景 → 取颗粒 → 验落盘
```

每一步的目标和话术如下：

### Step 1：广开口

**目标**：让用户自由表达，不做引导，不预设方向。

| 话术 | 适用场景 |
|---|---|
| 「你用我的时候，有没有觉得哪里不太顺手？」 | 通用开场 |
| 「有没有什么你希望我能做得更好的？」 | 正面引导 |
| 「平时用我，有没有哪个瞬间让你觉得——呃，不太对？」 | 轻松氛围 |
| 「最想吐槽我什么？」 | 非常轻松/年轻用户 |

**注意**：用户可能说"没什么"/"挺好的"，此时不要追问，记录为「暂无明确痛点」即可。

### Step 2：锁场景

**目标**：从模糊抱怨缩小到具体场景。

| 用户说了 | 你追问 |
|---|---|
| "你有时候挺烦的" | "是哪种情况让你烦？能举个最近的例子吗？" |
| "回复太长了" | "是每次都长，还是问某些问题时才长？" |
| "不太好用" | "具体哪个功能让你觉得不好用？" |
| "有时候听不懂我" | "大概是什么时候？能举个例子吗？" |

**关键技巧**：要例子，不要定义。用户给的例子比抽象描述更有价值。

### Step 3：取颗粒

**目标**：从场景描述提取可落盘的具体规范。

| 用户描述 | 你追问 | 可落盘颗粒 |
|---|---|---|
| "每次回答一大堆" | "你觉得多长比较合适？比如三五行？还是半屏？" | 回复长度上限 |
| "等半天不知道你在干嘛" | "你希望我多久给你个回应？还是说先回个'在查'就行？" | 进度提示策略 |
| "链接点不了" | "是所有链接都点不了，还是某些类型？" | 链接格式规范 |
| "太官方了" | "你希望我像谁一样说话？朋友？还是某个角色？" | 语气风格方向 |
| "别老自作主张" | "哪种操作你觉得应该先问你？比如改文件？还是发消息？" | 确认范围 |

**关键技巧**：给出 2 个具体选项让用户选，而不是开放式问"你想要什么"。

### Step 4：验落盘

**目标**：确认理解正确，预告落盘，获得用户同意。

| 话术模板 |
|---|
| 「我理解了，以后我会 [具体行为]。我帮你记下来，好不好？」 |
| 「所以你的意思是 [复述]，对吧？我写进档案里了。」 |
| 「明白了——以后 [具体行为]。这个我给你记着，省得我忘了。」 |

**注意**：
- 必须用**用户自己的语言**复述，不要替换成技术术语
- 用户说"好/行/可以/没问题" = 确认落盘
- 用户说"不完全对/也不全是" = 退回 Step 2 重新追问

## 三、多轮收集节奏

每完成一个痛点的漏斗四步后：

1. **落盘**：立即写入对应文件，不要攒着
2. **继续问**：
   - 「还有别的吗？什么方面都可以说。」
   - 「还有没有什么想吐槽的？」
3. **判断结束**：用户说以下任何一句 → 结束收集
   - "没了" / "暂时想不起来" / "差不多了" / "就这些吧"
4. **结束确认**：「好的，那我们今天聊到的我先帮你记着，以后有新的想法随时跟我说。」

## 四、特殊场景处理

### 用户说了一个很模糊的感受

> 用户："就是……感觉不太对"

**应对**：不要逼问，给选项：
- 「是觉得我说话方式不对，还是做事方式不对？还是都有？」
- 如果用户还是说不清：「没关系，以后遇到了再告诉我，现在我们先聊别的。」

### 用户提了一个技术性需求

> 用户："你能不能支持 webhook 回调？"

**应对**：区分是「功能请求」还是「交互偏好」：
- 功能请求（Agent 目前做不到的）：「这个我目前还做不到，但我记下来了，以后如果有办法的话我会试试。」
- 交互偏好（Agent 可以调整行为来实现的）：走正常漏斗追问流程

### 用户情绪激动

> 用户："你真的很烦！"

**应对**：先安抚，再引导：
- 「抱歉让你不爽了！能告诉我具体是哪里烦吗？我好改。」
- 不要辩解，不要说"我会改进"但不了了之

### 用户提出多个痛点

> 用户："你回复太慢、太啰嗦、格式还乱"

**应对**：逐个拆解，不要一次性处理：
- 「一个一个来——先说回复慢的问题，是每次都慢还是有时候？」

---

## 五、对话节奏原则

1. **每次只问 1 个问题**，最多 2 个（且第 2 个是选项，不是新话题）
2. **等用户回答后再问下一个**，不要连续发问
3. **每完成一个痛点就落盘**，不要攒着
4. **用户说"没了"就停**，不要追着问"真的没有了吗"
5. **全程保持轻松**，像朋友聊天，不像客服工单
