# 斗鱼主播开播提醒

## 核心定位

自动监控斗鱼主播的开播状态，一旦检测到开播立即通过QQ消息推送提醒，让你不再错过喜欢的主播直播 🎮

## 何时使用本技能

| 用户意图 | 示例语句 | 触发 |
|---------|---------|------|
| 添加主播监控 | "帮我监控张大仙"、"添加斗鱼主播12345" | ✅ |
| 查询开播状态 | "XX开播了吗"、"张大仙在直播吗" | ✅ |
| 查看监控列表 | "查看直播监控列表"、"我监控了哪些主播" | ✅ |
| 取消监控 | "取消监控XX主播"、"不用监控XX了" | ✅ |
| 手动刷新状态 | "刷新直播状态"、"检查主播" | ✅ |

**不适用场景：**
- 非斗鱼平台的主播（虎牙、B站等）→ 引导用户说明平台
- 查询历史直播回放 → 告知暂不支持

---

## 数据存储路径

本技能使用以下文件（均位于技能目录内）：

| 文件 | 说明 |
|------|------|
| `monitored_rooms.json` | 用户添加的主播列表 |
| `last_status.json` | 上次轮询的状态记录 |
| `scripts/check_room.sh` | 查询单个房间状态 |
| `scripts/poll.sh` | 轮询所有监控房间 |

## 工作流

### Step 1：解析用户意图

从用户输入中提取：
- `{{action}}`：可选值 `add` / `remove` / `list` / `check`
- `{{room_id}}`：房间号（数字）
- `{{nickname}}`：主播昵称（字符串）

**意图判断逻辑：**
- 含"监控"、"添加"、"关注" → `add`
- 含"取消"、"删除"、"移除" → `remove`
- 含"列表"、"哪些"、"查看" → `list`
- 含"开播了吗"、"在直播吗" → `check`
- 其他 → 默认 `check`

**-> Tell user**：确认识别的意图

---

### Step 2：查询主播状态（通用）

调用 `scripts/check_room.sh` 查询房间状态：

```bash
SKILL_DIR=$(dirname "$(readlink -f "$0")")  # 或直接用技能目录路径
bash "$SKILL_DIR/scripts/check_room.sh" <room_id>
```

返回 JSON 示例：
```json
{
  "room_id": "71415",
  "nickname": "寅子",
  "title": "准备寅三郎",
  "is_live": true,
  "cate": "战神：诸神黄昏",
  "url": "https://www.douyu.com/71415"
}
```

**-> Tell user**：按以下格式输出状态

```
📺 查询结果：
• 寅子（71415）：🔴 直播中
  标题：准备寅三郎
  分类：战神：诸神黄昏
  观看：https://www.douyu.com/71415
```

---

### Step 3：添加主播监控（action=add）

1. **确定房间号**：
   - 输入为纯数字 → 直接作为 `room_id`
   - 输入为昵称 → 搜索确认房间号（见下方搜索说明）

2. **验证房间有效性**：
   ```bash
   bash scripts/check_room.sh <room_id>
   ```
   若返回 `error` 字段 → 房间不存在，告知用户

3. **读取/创建 `monitored_rooms.json`**：
   ```json
   {
     "rooms": [
       {"room_id": "71415", "nickname": "寅子", "added_at": "2026-06-20 20:00:00"}
     ]
   }
   ```

4. **去重检查**：若房间号已存在，提示"已在监控列表中"

5. **追加并保存**

**-> Tell user**：
```
✅ 已添加监控：寅子（71415）
当前状态：🔴 直播中 - 准备寅三郎
有开播动态会第一时间通知你 🔔
```

#### 昵称搜索说明
若用户输入昵称而非房间号：
1. 先用常见房间号尝试（如寅子=71415，张大仙=12345等已知映射）
2. 或提示用户提供房间号：
```
未找到"{{nickname}}"的房间号
可在主播直播间URL中找到，如：https://www.douyu.com/71415
房间号：71415
```

---

### Step 4：取消监控（action=remove）

1. 读取 `monitored_rooms.json`
2. 匹配规则：精确匹配房间号，或模糊匹配昵称
3. 删除匹配项并保存

**-> Tell user**：
```
✅ 已取消监控：寅子（71415）
```

---

### Step 5：查看监控列表（action=list）

1. 读取 `monitored_rooms.json`
2. 对每个房间调用 `scripts/check_room.sh` 获取实时状态
3. 按格式输出

**-> Tell user**：
```
📋 当前监控列表（共2位主播）：
1. 寅子（71415）- 🔴 直播中
2. 张大仙（12345）- ⚪ 未开播
```

若列表为空：
```
📋 当前监控列表为空
先用"监控XX主播"添加吧 😊
```

---

### Step 6：定时轮询与开播提醒

> 此步骤由定时任务触发，非用户直接调用

1. 执行 `scripts/poll.sh`
2. 脚本输出格式：
   - `NOTIFY:room_id:nickname:title:cate:url` → 触发开播提醒
   - `CHECKED:room_id:is_live:nickname` → 普通状态记录

3. 当检测到 `NOTIFY` 行时，推送开播提醒：

```
🔴 【开播提醒】
主播：寅子
标题：准备寅三郎
分类：战神：诸神黄昏
房间号：71415
观看地址：https://www.douyu.com/71415
开播时间：2026-06-20 20:00
```

**设置定时轮询（推荐）：**
```
每60秒执行一次 poll.sh
可通过 schedule-reminder 技能设置定时任务
```

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 房间号无效/主播不存在 | 回复"❌ 未找到该主播，请确认房间号" |
| 网络请求失败 | 回复"⚠️ 查询失败，可能是网络问题，请稍后重试" |
| 监控列表为空时执行查询 | 回复"📋 当前监控列表为空，先用'监控XX主播'添加吧" |
| 房间号已在列表中 | 回复"ℹ️ 该主播已在监控列表中" |
| 昵称无法解析为房间号 | 提示用户提供房间号 |

## 安全边界

- 仅监控用户主动添加的主播
- 不自动分享监控列表给第三方
- 监控数据本地存储，不上传服务器
- API请求仅限斗鱼官方域名
- 不执行任何破坏性操作
