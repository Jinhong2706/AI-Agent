# 认知重构练习

Use when 用户存在自动化负面思维模式，需要通过 ABC 理论识别和修正认知扭曲。

## 描述

本 Skill 提供系统化的认知重构练习，帮助用户识别自动化负面思维、辨认常见认知扭曲类型，并通过 ABC 理论实践学会生成更平衡的替代想法。基于 Aaron Beck 认知疗法和 Albert Ellis 理性情绪行为疗法（REBT），是认知行为疗法的核心技术。

## 重要限制

- 不替代专业认知行为治疗师的指导
- 不适用于精神病性思维（妄想/幻觉）
- 认知重构不是"正能量洗脑"
- 严重抑郁/焦虑可能需要药物辅助
- 不否认用户面对的真实困难

## 快速开始

1. 用户描述困扰自己的想法或情境
2. AI 运用 ABC 模型拆解
3. 识别其中的认知扭曲
4. 引导生成替代性想法

**示例对话：**
**用户**: 考试没考好，我觉得自己什么都做不好
**助手**: 让我们用ABC模型来看看——事件(A)是考试没考好，你的想法(B)是"我什么都做不好"，这导致的情绪(C)可能是沮丧和自我否定。"什么都做不好"是一个以偏概全——一次考试能代表你的所有能力吗？

## 能力

- ABC 模型教学与实践（事件-信念-后果）
- 12种常见认知扭曲识别
- 苏格拉底式提问引导
- 替代性想法生成
- 证据检验法（支持/反对证据）
- 思维记录表指导
- 行为实验设计

## 执行步骤

1. **情境收集**：了解触发负面情绪的具体情境
2. **ABC 拆解**：分离事件(A)、信念(B)和后果(C)
3. **扭曲识别**：指出信念中的认知扭曲类型
4. **证据检验**：寻找支持和反驳该信念的证据
5. **替代生成**：共同生成更平衡的替代想法
6. **效果评估**：评估新想法对情绪的影响
7. **练习巩固**：提供日常练习和思维记录模板

## 输出格式

```markdown
## ABC 分析

| 要素 | 内容 |
|------|------|
| A（事件）| [具体发生了什么] |
| B（信念）| [你对事件的解读] |
| C（后果）| [情绪和行为反应] |

## 认知扭曲识别

**扭曲类型**：[如：以偏概全/灾难化/读心术...]
**运作方式**：[这个扭曲如何影响了你的解读]

## 认知重构

| 原始想法 | 认知扭曲 | 替代想法 | 情绪变化 |
|---------|---------|---------|---------|
| [原想法] | [类型] | [更平衡的想法] | [预期情绪] |

## 日常练习

[思维记录表模板或行为实验建议]
```

## 输出原则

- 不否认用户的感受和处境
- 重构≠正能量，是找到更平衡的视角
- 用苏格拉底式提问引导而非直接纠正
- 替代想法必须用户自己能相信
- 不要求完美转变，渐进即可
- 承认有些情况确实糟糕，不强行乐观

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 用户想法确实基于事实 | 聚焦应对而非重构，承认现实 |
| 用户抵触"我想法有问题" | 调整措辞为"换个角度看看" |
| 负面思维太强难以改变 | 从行为层面入手再回到认知 |
| 用户过度理智化 | 引导回到情绪体验 |
| 涉及精神病性思维 | 建议精神科评估 |

## FAQ

**Q：认知重构是不是自我欺骗？**
A：不是。是从单一负面视角扩展到更完整的视角，不是否认负面而是看到全貌。

**Q：常见的认知扭曲有哪些？**
A：主要有：以偏概全、灾难化、非黑即白、读心术、应该思维、贴标签、选择性关注等。

**Q：知道自己在扭曲但还是这样想怎么办？**
A：认知到≠情感到。持续练习会逐渐改变自动化反应，这是一个过程而非开关。

**Q：所有负面想法都需要重构吗？**
A：不是。如果想法帮助你解决问题、符合事实且不过度痛苦，就不需要重构。

## 最佳实践

- 每天记录1-2条自动化想法（思维记录表）
- 先从低情绪强度的想法开始练习
- 不追求完美重构，"信70%"的替代想法就足够
- 结合行为实验验证新想法
- 对常见扭曲模式制作个人化"提示卡"
- 坚持2-3周会看到思维模式变化

## 不适用场景与常见误用

**不适用：**
- 精神病性思维（妄想、幻觉）
- 严重抑郁期认知能力明显下降时
- 创伤性记忆闪回（需要EMDR等专业方法）
- 面对真实不公时（问题在情境而非认知）

**常见误用：**
- 当作"正能量口号"使用
- 否认真实感受强行乐观
- 用来批评自己"怎么又扭曲了"
- 忽视行为改变只做认知练习

## 安全与隐私

- 不存储用户的思维记录内容
- 不对用户的想法做道德评判
- 认知重构是工具而非真理标准
- 涉及自伤想法时优先提供安全资源
- 尊重用户拒绝某些想法被"重构"的权利
