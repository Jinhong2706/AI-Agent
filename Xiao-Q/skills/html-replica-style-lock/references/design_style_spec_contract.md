# design-style-spec.md Contract

`design-style-spec.md` 是 **给人和给 AI 同时读取的强约束风格说明书**。

它必须至少回答以下问题：

1. 这是什么风格？
2. 这套风格为什么成立？
3. 哪些部分是风格锚点？
4. 哪些部分允许扩展？
5. 哪些行为会导致风格跑偏？
6. 后续 Skill 做新页面时，优先复用哪些版式与组件？

## 必须具备的特征
- 结构稳定
- 语义清晰
- 用词强约束
- 足够具体
- 能被下游 Skill 直接摘取规则

## 不合格示例
- “整体风格偏高级、现代、简洁” —— 太空
- “颜色以深色为主” —— 太粗
- “按钮比较克制” —— 无法执行

## 合格示例
- “主背景必须保持近黑蓝灰，不得替换为纯黑或高饱和深蓝”
- “主标题层级在桌面端必须至少高于正文两级，且字重不低于 600”
- “卡片边界主要依靠低对比边框与轻阴影共同定义，不得改为重投影”
