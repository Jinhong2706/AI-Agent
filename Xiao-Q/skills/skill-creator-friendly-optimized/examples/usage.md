# Skill Creator 使用示例

## 示例 1：新手创建技能

用户：

> 我想做一个帮我写周报的技能，但不知道怎么描述。

执行方式：

1. 识别为新手引导模式。
2. 追问最少必要信息：使用对象、输入材料、输出格式、成功标准。
3. 根据回答生成 `SKILL.md`、示例和参考流程。
4. 校验 YAML front matter 和触发描述。
5. 打包输出。

## 示例 2：专业快速创建

用户：

> 创建一个合同审查 Skill，输入合同 docx，输出风险清单、修改建议和谈判提醒。

执行方式：

1. 识别为专业快速模式。
2. 自动生成技能目录。
3. 补齐 QA 清单和异常处理说明。
4. 校验并打包。

## 示例 3：优化已有技能包

用户：

> 这个技能能用，但异常处理一般，帮我优化。

执行方式：

1. 解压或读取已有技能目录。
2. 审计 `SKILL.md`、`AGENTS.md`、`references/`、`examples/`。
3. 检查是否缺少 YAML front matter。
4. 找出功能缺口：输入校验、错误提示、自纠错、进度保存、QA。
5. 在不破坏原核心能力的前提下优化。
6. 重新打包并输出变更摘要。

## 示例 4：故障修复

用户：

> 这个 Skill 安装失败，帮我查一下。

执行方式：

1. 识别为故障修复模式。
2. 检查 zip 根目录结构。
3. 检查 `SKILL.md` front matter。
4. 检查文件名、路径、编码、空文件。
5. 给出修复结果和可安装包。

## 示例 5：复杂需求的稳定执行

用户：

> 帮我把 8 个业务流程都封装成技能。

执行方式：

1. 先确认北极星目标和统一输出目录。
2. 拆分为多个独立单元。
3. 每个单元生成草稿。
4. 主流程统一合并、去重、校验和打包。
5. 保存进度，避免中断后丢失。

## 推荐提示词

```text
请使用 skill-creator 优化这个技能包：
- 保留原核心能力
- 补齐 YAML front matter
- 增加输入校验、异常处理、自纠错和 QA 清单
- 输出可安装 zip 包
- 给出变更摘要和未解决风险
```
