# 异常处理与自纠错协议

本协议用于提升 Skill Creator 在复杂输入下的稳定性。

## 错误分级

| 等级 | 类型 | 处理方式 |
| --- | --- | --- |
| E0 | 信息不足 | 只问必要问题，并给出默认方案 |
| E1 | 可自动修复 | 自动补齐缺失目录、front matter、示例或清单 |
| E2 | 可降级处理 | 先产出可用版本，标注假设与风险 |
| E3 | 高风险动作 | 停止并请求确认，例如覆盖源文件、删除、安装、发布 |
| E4 | 无法完成 | 说明阻塞点、尝试记录、原因和可选方案 |

## 自动修复规则

遇到以下问题时应优先自动修复：

- 缺少 `SKILL.md`：生成标准 `SKILL.md`。
- 缺少 YAML front matter：补齐 `name` 和 `description`。
- 缺少触发场景：根据用户目标生成明确触发条件。
- 缺少执行步骤：补充阶段式 workflow。
- 缺少异常处理：补充错误分级与恢复路径。
- 缺少 QA：补充质量门禁清单。
- 缺少示例：补充 2-4 个典型使用示例。

## 澄清策略

不要一次性提出大量问题。按以下顺序处理：

1. 能从上下文推断的，先推断并标注假设。
2. 可用默认值解决的，直接采用默认值。
3. 影响安全、覆盖、外部系统写入、用户身份或交付边界的，必须询问。
4. 每轮最多问 3 个关键问题。

## 重试策略

- 首次失败：检查路径、文件结构和权限。
- 第二次失败：换用更简单的实现路径，例如从完整重写降级为补丁式修改。
- 多次失败仍无进展：停止硬跑，汇报阻塞点和选项。

## 降级交付

当无法完整生成技能包时，应至少交付：

- 当前可用的 `SKILL.md` 草案；
- 已知缺口清单；
- 下一步修复建议；
- 不能完成的原因。
