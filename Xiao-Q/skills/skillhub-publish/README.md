# SkillHub Publish

> 一键检查 + 修复 + 生成发布物料，将你的 WorkBuddy skill 准备好发布到 SkillHub。

## 这是什么

如果你想把在 WorkBuddy 里打磨的 skill 发布到 SkillHub 市场，需要确保文件结构完整、描述清晰、版本一致、没有安全风险 —— 这些检查靠手工做既繁琐又容易遗漏。

**SkillHub Publish 就是来解决这个问题的。** 一句话触发，自动完成六维审计、与你讨论确认修复方案、在发布副本上执行修复、生成 README + 描述文本 + 版本说明，最后交付。原始文件仅在与你确认后才同步。

适合谁：
- 有 WorkBuddy skill 想发布到 SkillHub 的创作者
- 想确保 skill 文件干净、版本一致、无安全风险再上架
- 不想手工逐项检查，需要自动化审计 + 修复 + 打包一条龙

## 核心能力

| 能力 | 说明 |
|------|------|
| 六维审计 | 文件结构 / Frontmatter / 内容质量 / 版本一致性 / 安全 / 外部依赖 |
| 分级报告 | P0（必须修，无 checkbox）→ P1（推荐修）→ P2（锦上添花） |
| 先议后行 | 展示完整报告后与你讨论确认，不擅自动手 |
| 出发确认 | 🆕 Phase 0 必先出示完整"机票"（做什么/产出什么/放哪里/要确认几次），用户签字后才进入审计 |
| 副本优先 | 修复在桌面副本上执行，原始文件可选同步 |
| 零发现短路 | 审计零问题 → 跳过修复流程，直接生成物料 |
| 自动修复 | 补文件、改描述、对齐版本，每一步确认后执行 |
| 内容脱敏 | 自动替换发布包中的敏感词（人名/组织/政治触发词），避免上架后被报警 |
| AI 自举 | 首次运行自动用 LLM 扫描你的 skill，生成个性化脱敏规则，一次生成终身复用 |
| 反馈闭环 | 上架后收到安全报警？告诉 skill「XX 文件被报警了」，自动扫描新触发词并补入规则库 |
| 物料生成 | 自动生成 README.md + SkillHub 描述文本 + 版本更新说明 |
| 干净打包 | 自动排除 `.git`/`__pycache__`/`.DS_Store` 等垃圾文件 |

## 快速使用

对你的 WorkBuddy 说：

```
发布技能 xxx
```

或

```
把 xxx 准备好发布到 skillhub
```

系统自动进入完整流程，全程交互式引导。

## 工作流

```
Phase 0: 出发前确认（Pre-flight Checklist）
   │  └─ 向用户出示完整"机票"：要做的事、输出物、确认次数、输出位置
   │  └─ 用户签字确认后进入审计
   ▼
Phase 1: 六维审计（只读）
   │
   ├─ 零问题 → Phase 4 直接生成物料
   └─ 有问题 → Phase 2 讨论确认
                  │
                  ▼
Phase 2.5: 创建发布副本（排除垃圾文件，骨架化用户数据）
                  │
                  ▼
Phase 3: 在副本上修复（可选同步原文件）
                  │
                  ▼
Phase 3.5: 内容脱敏（AI 自举 → 敏感词替换 → 报告确认）
                  │
                  ▼
Phase 4: 生成 README.md + 描述文本 + 版本说明
                  │
                  ▼
Phase 5: 终检 → 最终交付
```

## 技术架构

纯指令式实现，所有逻辑在 SKILL.md 中定义，由 WorkBuddy agent 在对话中直接执行。

- **零 Python 脚本** — 不依赖任何外部代码
- **零外部依赖** — 不依赖其他 skill 或第三方库
- **零网络请求** — 所有操作在本地完成
- **先议后行** — 每步写操作前经用户确认
- **副本优先** — 所有修改在桌面副本上执行，原始文件可控
- **内容脱敏三层** — 内置通用规则（随 skill 发布）→ 用户私有规则（本地）→ skill 特有 override
- **AI 自举** — 首次运行自动扫描目标内容，生成个性化脱敏规则
- **反馈闭环** — 报警文件反馈后自动识别新触发词，规则库持续进化

详见 DESIGN.md（架构决策记录）。

## 环境要求

- WorkBuddy Agent（任意版本）
- 无其他依赖

## 安全与限制

- 仅限 WorkBuddy 平台使用
- 仅读取 `~/.workbuddy/skills/` 下的文件
- 写操作仅在用户明确确认后执行
- 不依赖其他 skill —— 安装一个即可用
- 遵循 MIT-0 许可证
