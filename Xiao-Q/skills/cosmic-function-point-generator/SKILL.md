# COSMIC 功能点度量表格自动生成 Skills

> **版本**：v1.0.0
> **适用范围**：适用于需求设计说明书/概要设计说明书/详细设计说明书已明确三级功能点的中小型软件项目（PBOSS/业务后台/工作流类），功能点数量在 5~50 之间，需要按 COSMIC ISO 19761 方法学进行功能点估算与拆分的场景。
> **优先级标注体系**：
>   - 🔴 **强约束 (Mandatory)**：必须严格遵守，违反直接判定输出不合格
>   - 🟡 **建议约束 (Recommended)**：默认遵守，若有充分理由可灵活调整
>   - 🟢 **参考建议 (Advisory)**：参考使用，智能体可按场景灵活处理
> **规则源**：
>   - 主规则：`references/cosmic-rules.md`（基于 `cosmic生成规则v2.1.md` 整合，含强约束）
>   - 辅助规则：`references/cosmic-spec-auxiliary.md`（基于 `COSMIC智能体自动生成规范.md` 提炼，含判定算法与示例）
>   - 可编辑规则文件：`references/cosmic-rules.md`（人工编辑入口）

## 2. 总体执行流程（14 步，不可逆）

```
[Step 1]  加载规则源
  └─→ 读取 references/cosmic-rules.md（强约束集）
[Step 2]  解析设计文档
  └─→ 解析 .docx 段落与表格，构建 DOM 树
[Step 3]  提取需求矩阵
  └─→ 抽取一级/二级/三级需求，建立索引
[Step 4]  定位功能实现章节（2.3 节）
  └─→ 收集每个三级功能点的功能描述原文
[Step 5]  前置分点扫描（强约束）
  └─→ 扫描数字分点/并列条件/独立动作，统计总量
[Step 6]  配比规划
  └─→ 按 2:7:1 分配"新增/复用/利旧"行号
[Step 7]  搭建骨架
  └─→ 写出 12 列固定前 3 行 + Sheet 名
[Step 8]  逐行拆分子过程（强约束）
  └─→ F 列 1:1 还原原文动作，G 列判定 E/R/W/X
[Step 9]  填充 H/I 列（强约束）
  └─→ 数据组与数据属性，含专属定语保证唯一性
[Step 10] 填充 A~E 列
  └─→ 客户需求/功能用户/二级需求/触发事件/功能过程
[Step 11] 填充 J/K 列
  └─→ 复用度 + CFP 计算
[Step 12] 物理合并
  └─→ A~E 列相邻同值行垂直合并，仅首行保留
[Step 13] 填 ΣCFP
  └─→ L 列仅首行填入 SUM(K 列)
[Step 14] 合规校验（15 项）
  └─→ 全部通过后输出 .xlsx
```

### 2.1 关键执行原则

- 🔴 **源数据唯一原则**：所有内容必须源自设计文档原文，禁止新增/修改/过度解读
- 🔴 **一一对应原则**：每一行业务数据在原文必须有唯一对应
- 🔴 **固定结构优先**：前 3 行 + 12 列结构不可修改
- 🔴 **执行顺序固定**：14 步严格按顺序执行，不可跳步/逆序

## 3. 关键规则速查（强约束）

### 3.1 数据移动类型 G 列强约束

| 类型 | 简称 | 出现位置 | 判定关键词 |
|:----:|:-----|:---------|:-----------|
| **E** | Entry 输入 | **每三级功能点首行（强制，仅 1 次）** | 解析入参、获取入参、接收推送、调用接口 |
| **R** | Read 读 | 中间行，可多次 | 通过…查询、读取、解析已落库数据、判断（须改写为"读取并判断"） |
| **W** | Write 写 | 中间行或末行 | 保存、写入、更新、生成+保存、记录日志 |
| **X** | eXit 输出 | **末行（强制 W/X 结尾）** | 反馈、返回、推送、发送、调用外部接口 |

### 3.2 行数与拆分强约束

- 🔴 **每三级功能点行数 ≤ 8**
- 🔴 **首行必须为 E，末行必须为 W 或 X**
- 🔴 同一三级功能点下 E 动作**仅允许首行出现 1 次**
- 🟡 **行数 > 8 拆分规则**：拆分为 2 个子功能过程，名称后追加 `（一）` `（二）` 或序号 1/2，各自满足 E→…→W/X 完整流程且 ≤ 8 行

### 3.3 复用度配比强约束

- 🔴 **新增:复用:利旧 = 2:7:1**
- 🔴 **新增占比 20%~25%**（下限 ceil(N×0.2)，上限 floor(N×0.25)）
- 🔴 **CFP 固定取值**：新增=1.00、复用=0.33(⅓)、利旧=0.00
- 🟢 **分配原则**（按业务重要性）：保存/写入/生成类 W 优先「新增」；调用既有接口/获取入参类 E/X 优先「利旧」；其余 R/中间 W 分配「复用」

### 3.4 字符长度与禁用词强约束

- 🔴 **F / H / I 列 ≤ 30 字符**（剔除所有空格，含中文/全角空格）
- 🔴 **F 列禁用单独"判断"**，必须改写为"读取…并判断…"
- 🔴 **F 列禁止单独出现**：封装、加密、redis 缓存、持久化数据
- 🔴 **F 列禁止过度精简**，必须逐行还原原文"动作+操作对象+业务目的"
- 🟡 超长降级策略：优先保留"动作+对象"，业务目的简写为 1~2 个核心词

### 3.5 字段唯一性强约束

- 🔴 **H 数据组全表唯一、同功能过程唯一**
- 🔴 **I 数据属性全表唯一、同功能过程唯一**
- 🔴 **必须加专属定语**（定语来源优先级：子过程动作动词 > 数据组定语 > 业务场景定语 > 触发事件定语 > 时间阶段定语）

### 3.6 分点拆分强约束

- 🔴 数字分点（1/2/3…、① ② ③）→ 每个分点独立成行
- 🔴 顿号/分号/"且/或"连接的平行条件 → 拆分为多行
- 🔴 多业务场景分支 → 拆分为多行
- 🔴 多维度查询（按地址/按订单号/按账号）→ 拆分为多行
- 🔴 多步骤写操作（生成+保存+更新）→ 拆分为多行
- 🔴 **禁止合并**："场景1查A、场景2查B" 合并为 "按场景查询" / "判断A、判断B" 合并为 "判断条件" / "读取X、读取Y" 合并为 "读取信息"

### 3.7 触发事件 5 类取值（强约束）

仅允许以下 5 类（按优先级匹配）：

| 顺序 | 取值 | 适用场景 |
|:-----|:-----|:---------|
| 1 | 原文触发动作 | 原文有"点击查询"等明确动作时原文照录 |
| 2 | `定时任务` | 原文为"定时…"/"每天…" |
| 3 | `流程自动流转` | 原文为"流程触发"/"流程引擎…" |
| 4 | `接口调用` | 原文为"调用方系统推送"/"外部系统请求" |
| 5 | `人工操作` | 兜底 |

### 3.8 功能用户格式（强约束）

**固定格式**：`发起者：XX系统/服务；接收者：XX系统/服务`

| 业务场景 | 发起者 | 接收者 |
|:---------|:-------|:-------|
| 页面可视化类 | 业务服务接口 | 业务 WEB 服务 |
| 流程自动执行类 | 流程引擎服务 | 业务服务接口 |
| 跨系统接口类 | 调用方系统服务 | 被调用方系统服务 |
| 定时任务类 | 流程引擎服务 | 业务服务接口 |
| 外部系统反馈类 | 外部系统 | 业务服务接口 |

## 4. 输出文件结构

### 4.1 12 列固定结构（不可篡改）

| 列 | 列名 | 数据类型 | 长度上限 | 来源 |
|:-:|:-----|:---------|:---------|:-----|
| A | 客户需求 | 文本 | 50 | 整表统一 = 核心需求全称（来自设计文档标题/1.2 节） |
| B | 功能用户 | 文本 | 60 | `发起者：XX；接收者：XX` |
| C | 功能用户需求 | 文本 | 50 | 二级需求全称 |
| D | 触发事件 | 文本 | 30 | 5 类取值之一 |
| E | 功能过程 | 文本 | 50 | 三级功能点全称 |
| F | 子过程描述 | 文本 | **30** | `动作+对象+目的`，E 开头、W/X 结尾 |
| G | 数据移动类型 | 枚举 | 1 | E/R/W/X |
| H | 数据组 | 文本 | **30** | 命名源自原文，全表唯一 |
| I | 数据属性 | 文本 | **30** | 字段集合（顿号分隔），全表唯一 |
| J | 复用度 | 枚举 | 4 | 新增/复用/利旧 |
| K | CFP | 数值 | - | 1.00 / 0.33 / 0.00 |
| L | ΣCFP | 数值 | - | 仅首行，SUM(K 列) |

### 4.2 前 3 行固定骨架

```
第 1 行：12 个空单元格（强制保留）
第 2 行：A:C 合并="度量策略阶段"，D:J 合并="映射阶段"，K:L 合并="度量阶段"
第 3 行：表头行（12 列名称严格按 A→L 顺序）
第 4 行起：业务数据行，按三级功能点分组
```

### 4.3 文件格式

- 文件格式：`.xlsx`
- Sheet 名：`2、功能点拆分表`
- 起始位置：A1
- 合并规则：A~E 列内容完全相同的相邻行 → 垂直合并
- 列宽建议：A=30 / B=22 / C=30 / D=12 / E=30 / F=32 / G=8 / H=30 / I=32 / J=8 / K=8 / L=8

## 6. 特殊场景处理指南

### 6.1 批量操作

| 场景 | 处理 | F 列模板 |
|:-----|:-----|:---------|
| 单次接口推送 N 条数据 | 单次 X，不重复计数 | `批量反馈【对象】至【对端】` |
| 批量查询后逐条处理 | 拆分：批量查询(R) + 逐条处理(W) | `批量查询【对象】获取【N】条记录` |
| 批量保存多条记录 | 单次 W，不重复计数 | `批量保存【对象】至【表】` |

### 6.2 复用组件

| 复用程度 | 复用度 | CFP | F 列模板 |
|:---------|:-------|:----|:---------|
| 全新开发 | 新增 | 1.00 | `根据【依据】生成【对象】并保存至【表】` |
| 复用公共逻辑 | 复用 | 0.33 | `调用【公共组件】处理【业务对象】` |
| 完全既有逻辑 | 利旧 | 0.00 | `调用【既有接口】获取【对象】` |

### 6.3 接口调用类

| 接口方向 | G 列 | F 列模板 |
|:---------|:-----|:---------|
| 接收外部入参 | E | `解析【接口名】入参` |
| 推送至外部 | X | `调用【对端系统】【接口名】推送【对象】` |
| 解析外部出参 | R | `解析【接口名】出参获取【对象】` |

### 6.4 定时任务类

- D = `定时任务`
- B = `发起者：流程引擎服务；接收者：业务服务接口`
- 首行强制 E：`获取定时任务配置参数`
- 末行强制 W：`记录定时任务执行日志` 或 `更新待执行状态`

### 6.5 流程自动触发类

- D = `流程自动流转`
- B = `发起者：流程引擎服务；接收者：业务服务接口`
- 首行强制 E：`获取流程触发的当前【对象】id`
- 末行强制 W：`更新当前【对象】流程实例状态`

### 6.6 日志记录类

- 视为单次 W 写入（不重复计数）
- F 列：`保存【对象】操作轨迹日志`
- H 列必须加专属定语
- 同一三级功能点下最多 1 条日志类 W

### 6.7 字段长度超限的降级策略

| 优先级 | 策略 | 示例 |
|:-------|:-----|:-----|
| 1 | 简化"业务目的"为 1~2 个核心词 | `保存退单信息至回访表` |
| 2 | 删除通用字段，保留业务标识字段 | 保留 `工单Id`，删除 `联系号码` |
| 3 | 使用缩略语（仅限域内通用） | IVR、PBOSS、CFP |
| 4 | 重新选择专属定语 | `退单场景_原因id` |

## 7. 执行脚本与自动化

### 7.1 推荐脚本（位于 `scripts/`）

| 脚本 | 功能 | 调用方式 |
|:-----|:-----|:---------|
| `extract_requirements.py` | 从 .docx 提取需求矩阵与功能描述 | `python extract_requirements.py <docx_path>` |
| `generate_cosmic.py` | 端到端生成 COSMIC 表格 .xlsx | `python generate_cosmic.py <docx_path> <output.xlsx>` |
| `validate_cosmic.py` | 执行 15 项合规性校验 | `python validate_cosmic.py <xlsx_path>` |

### 7.2 智能体手工执行模式

若脚本不可用（如仅文本环境），按以下伪代码执行：

```python
# 伪代码：端到端 COSMIC 表格生成
def generate_cosmic(docx_path, output_path):
    # Step 1: 解析文档
    doc = parse_docx(docx_path)

    # Step 2: 提取需求矩阵
    req_matrix = extract_requirements_matrix(doc)  # {一级, 二级, 三级}

    # Step 3: 定位功能实现章节
    func_desc = extract_function_descriptions(doc, section="2.3")

    # Step 4: 前置分点扫描
    sub_processes = scan_sub_processes(func_desc)  # 每个三级功能点 → N 个子过程

    # Step 5: 配比规划（新增:复用:利旧 = 2:7:1）
    n = len(sub_processes)
    plan = allocate_reuse(n)  # 标记每行复用度

    # Step 6-12: 逐行填充 12 列
    rows = build_rows(sub_processes, plan, req_matrix)

    # Step 13: 合并 A~E
    rows = merge_a_to_e(rows)

    # Step 14: 校验
    check_result = validate(rows)
    assert check_result.passed, check_result.errors

    # 输出
    write_xlsx(output_path, rows)
```
