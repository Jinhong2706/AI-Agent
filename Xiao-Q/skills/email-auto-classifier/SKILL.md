# Email Auto Classifier — 邮件自动分类

Use when the user's inbox is overwhelmed, needs help organizing emails by priority, wants a system for categorizing incoming messages, or needs email processing workflow advice.

NOT for: email marketing campaigns, spam filter configuration, email server setup, or writing/replying to emails.

## 描述

为收件箱爆炸的职场人提供邮件分类方案，按项目归属、紧急程度和操作类型自动分类，生成处理优先级清单，帮助从"每天淹没在邮件里"变为"10分钟清空收件箱"。

## 重要限制（请提前告知用户）

- **无法直接连接邮箱系统**，需用户描述分类需求或提供邮件列表
- **自动分类规则需要人工设定后执行**，本工具帮助设计规则而非自动执行
- **紧急程度判断基于规则**，可能遗漏上下文导致的隐含紧急性
- **不阅读或存储邮件内容**，仅基于描述的模式提供分类建议

## 能力

- 分类体系设计：根据用户工作场景设计个性化邮件分类规则
- 优先级排序：按紧急×重要矩阵划分邮件处理优先级
- 规则生成：输出可直接配置到邮件客户端的过滤规则
- 处理流程设计：设计批量处理邮件的SOP
- 模板推荐：为常见回复场景准备快捷回复模板
- 减负策略：识别可退订/取消通知的邮件来源

## 执行步骤（Step by Step）

### Step 1：邮件现状诊断
1. 了解日均收件量和主要来源
2. 识别邮件类别（项目通知/审批/知会/外部/广告）
3. 了解当前处理方式和痛点
4. 确认使用的邮件客户端

### Step 2：分类体系设计
1. 设计3-5个一级分类（如项目A/项目B/行政/外部/通知）
2. 为每个分类定义优先级规则
3. 标注"可不看"和"必须即时处理"的边界
4. 设计标签/文件夹/颜色标记体系

### Step 3：输出执行方案
1. 生成邮件客户端的过滤规则配置
2. 设计每日邮件处理SOP
3. 提供常见场景的快捷回复模板
4. 输出退订/取消通知建议清单

## 输出原则

1. 分类不超过5个层级，越简单越容易坚持
2. 规则设计遵循"20%邮件产生80%价值"原则
3. 必须为每个分类指定明确的处理时效
4. 提供可直接配置到邮件客户端的具体规则
5. 考虑减法思维——能不收的邮件就不收

## 错误处理

| 异常场景 | 提示语 |
|----------|--------|
| 用户不确定邮件来源类型 | "可以转发或截图您最近一天的收件列表（隐去敏感内容）吗？我来帮您分类" |
| 邮件客户端不支持规则 | "您的邮件客户端功能有限，我帮您设计手动分类的快速流程作为替代" |
| 用户邮件量极大（500+/天） | "这个量级建议先做大幅减法（退订+退群组），再建规则。我来帮您识别可砍掉的来源" |
| 分不清哪些邮件可以不看 | "我们用一个测试：如果这封邮件3天没读，会有什么后果？没后果的就是'可不看'" |
| 团队强制CC文化 | "CC泛滥是组织问题，个人层面建议设规则自动归档CC邮件，每周集中扫一次" |

## 常见问题（FAQ）

**Q：Inbox Zero真的能做到吗？**
A：可以，但目标不是"邮件数=0"，而是"知道每封邮件的去向"。归档≠处理完，但意味着你掌控了它。

**Q：会不会漏掉重要邮件？**
A：良好的规则设计会确保重要邮件被高亮。建议前两周每天检查一次"自动归档"文件夹，验证规则准确度。

**Q：手机上也要设分类吗？**
A：建议手机只处理🔴紧急邮件，其他留到电脑前处理。手机设VIP联系人通知即可。

**Q：邮件和即时通讯（飞书/微信）怎么配合？**
A：邮件用于正式/需留痕的沟通，IM用于快速协调。如果一件事24小时内需回复，IM更合适。

**Q：规则设好了但还是忍不住看每封邮件怎么办？**
A：这是习惯问题。建议关闭邮件弹窗通知，设定固定处理时间，前三天会不适应，一周后会习惯。

**Q：离职/转岗后分类规则要重新来吗？**
A：项目相关规则需要更新，但处理框架（优先级分层+固定处理时间）是通用的。

## 最佳实践

1. 邮件处理遵循"一次触碰原则"：打开一封邮件就决定它的去向（回复/委派/归档/删除）
2. 设定3个固定时间处理邮件，其余时间关闭邮件客户端
3. 紧急事项不应该依赖邮件传达——和团队约定紧急联系方式
4. 每月清理一次规则，删除过期规则，添加新需求
5. 善用快捷回复模板，常见回复不需要每次重新写

## 不适用场景

| 场景 | 原因 | 替代方案 |
|------|------|----------|
| 邮件营销/群发 | 非收件管理 | 使用邮件营销工具 |
| 垃圾邮件过滤 | 系统级功能 | 配置邮件服务商反垃圾策略 |
| 邮件服务器迁移 | IT基础设施 | 联系IT部门 |
| 邮件内容撰写 | 非分类范畴 | 使用写作助手工具 |

常见误用：
- 分类过于细致（20+文件夹反而找不到邮件）
- 设规则但从不检查规则效果
- 把所有沟通都用邮件解决（应该分渠道）

## 安全与隐私

- 不要求用户转发或提供邮件原文内容
- 分类规则设计基于模式描述，不涉及具体邮件内容
- 不会连接或访问用户的邮箱账户
- 建议用户在描述邮件模式时隐去发件人具体姓名
