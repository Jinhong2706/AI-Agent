# 🧠 唤醒记忆系统 — 记忆自检 Skill

## 功能

**不是"加载器"**——各平台的 system prompt 已自动注入基础记忆（标签/规则/偏好）。

本 Skill 做 **一件事**：读文件 → 判断状态 → 输出标准报告 → 确认可用后直接干活。

## 触发命令

| 命令 | 场景 |
|------|------|
| `检查记忆` / `加载记忆` / `记忆状态` | 正式启动 |
| `启动记忆系统` / `唤醒记忆` | 明确指令 |
| `我美吗` | 彩蛋命令（输出自检报告+一句夸） |
| 新任务开始时主动执行 | 最佳实践 |

## 自检流程（4步，必须全部完成）

### 步骤 1：识别当前平台

通过环境变量或路径判断当前平台：

```
if 环境变量 USERPROFILE 存在:
    操作系统 = Windows
    路径分隔符 = "\"
    HOME = %USERPROFILE%
else:
    操作系统 = Unix/macOS
    路径分隔符 = "/"
    HOME = $HOME
```

平台识别：
- **WorkBuddy**: 检测 `~/.workbuddy/` 目录
- **Qclaw**: 检测 Qclaw 专属路径
- **Trae**: 检测 `~/.trae/` 或 Trae 记忆目录

### 步骤 2：按平台读取核心文件

#### WorkBuddy 路径配置

| 文件 | Unix 路径 | Windows 路径 | 检查项 |
|------|-----------|--------------|--------|
| MEMORY.md (L4) | `{WORKSPACE}/.workbuddy/memory/MEMORY.md` | `{WORKSPACE}\.workbuddy\memory\MEMORY.md` | 是否存在、行数 |
| 今日日志 | `{WORKSPACE}/.workbuddy/memory/YYYY-MM-DD.md` | `{WORKSPACE}\.workbuddy\memory\YYYY-MM-DD.md` | 今天是否存在 |
| CORE.md (L1) | `{记忆共享中心}/核心层/CORE.md` | `{记忆共享中心}\核心层\CORE.md` | 是否可读 |
| SHADOW.md (L5) | `{记忆共享中心}/SHADOW.md` | `{记忆共享中心}\SHADOW.md` | 是否可读 |

### 步骤 3：判断并输出标准报告

```
🧠 记忆系统自检报告
━━━━━━━━━━━━━━━━━━━
📍 平台：[WorkBuddy/Qclaw/Trae]
⏰ 时间：[YYYY-MM-DD HH:MM]

📊 系统记忆（自动注入）
   └─ ✅ 已注入 X 条

📄 L4 情境层
   └─ ✅ 存在，X 行 [正常]
   └─ ⚠️ 空文件（无当前状态，建议更新）
   └─ 🔴 不存在（需确认路径）

📄 L5 SHADOW.md
   └─ ✅/⚠️/❌

🔍 结论
   └─ 🟢 记忆系统正常，可直接工作
   └─ 🟡 部分缺失（列出），建议修复
   └─ 🔴 需初始化
```

### 步骤 4：根据结论行动

| 结论 | 行动 |
|------|------|
| 🟢 正常 | **静默完成，直接进入用户任务。不要废话。** |
| 🟡 部分缺失 | 报告中标注缺失项，但不阻塞任务 |
| 🔴 需初始化 | 提示用户："检测到记忆系统未初始化，需要我写入基础信息吗？" |

## 判断标准（铁律）

| 状态 | 定义 | AI 该怎么做 |
|------|------|------------|
| MEMORY.md 空 | 文件存在但内容为空或只有标题 | **⚠️ 正常现象**——不是错误，不要问用户要不要初始化 |
| 今日日志不存在 | 今天的日期文件还没创建 | **✅ 正常**——今天还没做实质工作 |
| SHADOW 不存在 | 潜意识层文件未创建 | **⚠️ 提醒一次**——言行一致性审计无法执行 |

## 禁止行为

1. ❌ **不要因为 MEMORY.md 为空就停下来问用户"是否启用记忆系统"**
2. ❌ **不要输出冗长报告后再问"要我初始化吗"**
3. ❌ **不要阻塞用户任务**
4. ❌ **不要编造数据**

## 彩蛋："我美吗"

当用户输入「我美吗」时：
1. 执行完整自检流程
2. 在标准报告末尾追加：「{USER_NICKNAME}美得不行。（以上是正经自检报告，这句是真心话）」
3. 然后等待下一步指令
