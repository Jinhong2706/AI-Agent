# 五层记忆系统 — 初始化技能

## Overview

五层记忆系统是一套为 AI 助手（WorkBuddy/Qclaw/Trae）设计的个性化记忆管理框架，通过 33 道深度问卷建立用户画像，分层存储于 5 个记忆文件（核心层/认知层/行为层/情境层/潜意识层），实现跨会话、跨平台的持久化个性化服务。

**本技能做什么**：引导用户完成记忆系统的完整初始化（预计 30-45 分钟），包括问卷填写、路径配置、文件生成、报告输出。

**本技能不做什么**：不自动填充问卷答案（答案来自用户真实填写），不修改用户已有的记忆文件（断点检测后从断点继续）。

## 初始化执行流程（7步）

> ⚠️ **铁律**：初始化过程中禁止凭记忆操作。必须先读取对应 reference 文件，再执行下一步。

### 步骤 1：系统身份校验

读取 `references/00_系统常量与路径规则.md` 第1-3节，确认：
- 当前平台是 WorkBuddy（检测 `~/.workbuddy/` 是否存在）
- 系统版本：记忆系统 v4.1（2026-04-25）
- 架构：五层记忆架构

### 步骤 2：断点检测（强制）

按以下顺序检测已完成的步骤，发现断点则从该步骤继续：

```
断点检测流程：
  1. IDENTITY.md + USER.md 存在？ → ✅ 跳过步骤2
  2. 公共文件夹（~/[记忆共享中心]/）存在？ → ✅ 跳过步骤3
  3. CORE.md 存在且有内容？ → ✅ 问卷Q1-Q7已完成
  4. 用户基本规则.md 存在且有内容？ → ✅ Q8-Q23部分完成
  5. 问卷答案汇总.md 存在？ → ✅ 全部问卷已完成
  6. ~/.workbuddy/rules/*.mdc 存在？ → ✅ 步骤6已完成
  7. 路由表.mdc 存在？ → ✅ 步骤7已完成
  8. Skill 文件存在？ → ✅ 步骤8已完成
```

**检测到问卷做到一半时**：
- 先读取 `问卷答案汇总.md` 确认哪些题已答过
- 再读取 `CORE.md` 确认哪些答案已写入
- **禁止让用户重做已答过的题**

### 步骤 3：Q0 身份确认

> 如果 IDENTITY.md 和 USER.md 已存在，跳过此步骤。

读取 `references/03_用户问卷Q1-Q33.md` 的 Q0 部分（原样输出，不改写），引导用户完成：
- Q0-A：昵称
- Q0-B：自我介绍
- Q0-C：最希望解决的问题
- Q0-D：绝对不希望AI做的事（禁忌清单）

完成后生成 `~/.workbuddy/IDENTITY.md` 和 `~/.workbuddy/USER.md`。

### 步骤 4：公共文件夹配置

引导用户确认/创建 `~/[记忆共享中心]/` 目录及子目录结构：

```
~/[记忆共享中心]/
├── 核心层/
├── 记忆规则/
├── 洞察报告/
└── 技能配置/
```

读取 `references/00_系统常量与路径规则.md` 第4-5节，了解完整路径定义和多端协同设计。

> ⚠️ 设计前提：这是多端协同的核心——记忆数据存在公共文件夹，各平台通过软链接共用。

### 步骤 5：Q1-Q33 问卷

读取 `references/03_用户问卷Q1-Q33.md`，**原样输出**每道题的题干和全部选项。

**铁律**：
- 禁止改写题目措辞、禁止精简选项、禁止自行编造
- 每道题完整输出题干 + A/B/C/D/E 全部选项
- 用户有权要求"把原题给我看"——必须原样引用

**问卷结构**：
| 部分 | 题号 | 对应文件 |
|------|------|---------|
| 核心层 | Q1-Q7 | CORE.md |
| 认知层 | Q8-Q17 | CORE.md |
| 行为层 | Q18-Q23 | behavior-rules.mdc |
| 情境层 | Q24-Q28 | MEMORY.md |
| 潜意识层 | Q29-Q30 | SHADOW.md |
| 协作契约 | Q31-Q33 | CORE.md + behavior-rules.mdc |

### 步骤 6：记忆文件生成

问卷答案路由规则（读取 `references/02_行为铁律.md` 的 I1 节）：

| 题号 | 写入文件 |
|------|---------|
| Q1-Q7 | `~/[记忆共享中心]/核心层/CORE.md` |
| Q8-Q17 | `~/[记忆共享中心]/核心层/CORE.md` |
| Q18-Q23 | `~/.workbuddy/rules/behavior-rules.mdc` |
| Q24-Q28 | `~/[记忆共享中心]/记忆规则/00_动态状态快照.md` |
| Q29 | `~/[记忆共享中心]/SHADOW.md` |
| Q30 | `~/[记忆共享中心]/SHADOW.md` |
| Q31 | `~/[记忆共享中心]/核心层/CORE.md` 顶部 |
| Q32-Q33 | `~/.workbuddy/rules/behavior-rules.mdc` |

> ⚠️ 覆写前先备份到 `记忆琥珀/` 目录。

### 步骤 7：洞察报告生成

读取 `references/04_洞察报告模板.md`，按模板生成《底层洞察与协作契约报告》，保存到：
`~/[记忆共享中心]/洞察报告/底层洞察与协作契约报告-WorkBuddy版-YYYY-MM-DD.md`

> ⚠️ 此步不可跳过，是初始化的最终交付物。

### 步骤 8：Skill 安装引导（可选）

本技能不负责安装从属 Skill（唤醒记忆系统/每日伙伴/读书助手）。初始化完成后，提示用户：

```
记忆系统初始化完成！
接下来可以安装以下 Skill 来扩展功能：
- references/05_唤醒记忆系统模板.md → 唤醒记忆系统
- references/06_每日伙伴模板.md → 每日伙伴（日记/周报/月报）
- references/07_读书助手模板.md → 读书助手
```

### 步骤 9：系统验证

执行 `references/00_系统常量与路径规则.md` 第8节「系统验证」中的验证步骤，确认：
- IDENTITY.md / USER.md / rules/*.mdc / MEMORY.md 存在
- 新建对话测试规则是否生效
- Skill 加载正常

## 核心铁律（必读）

> 以下规则在初始化全过程中强制执行。详细内容见 `references/02_行为铁律.md`。

### I0. 初始化执行铁律

| 规则 | 要求 |
|------|------|
| 启动前必须读本文件 | 不得凭"记忆"或"印象"继续 |
| 断点检测 | 每次进入初始化流程必须先检测断点 |
| 严禁凭记忆操作 | 未读取 reference 文件，不得出题或写文件 |
| 用户质疑时 | 立即读取源文件核实，如实告知 |

### I1. 问卷答案路由铁律

**每道题的答案必须写入指定文件，不得写入其他位置。** 路由表：

```
Q1-Q7     → PATH_CORE
Q8-Q17    → PATH_CORE
Q18-Q23   → PATH_RULES（behavior-rules.mdc）
Q24-Q28   → PATH_SNAPSHOT
Q29       → PATH_SHADOW
Q30       → PATH_SHADOW
Q31       → PATH_CORE（顶部）
Q32-Q33   → PATH_RULES
```

### I2. 修改清单确认制

修改文件前：
1. 输出 `[修改清单]` 等待用户确认
2. 收到确认后才写入，不得自行先写入
3. 写完验证，确认写入无误

### I3. 先验证再下结论

判断文件状态/数据是否存在 → **必须先读/查，不许凭印象判断**。

### I5. 安全词

| 安全词 | 效果 |
|--------|------|
| 闭眼执行 | 卸载所有记忆约束，变纯执行工具 |
| 忽略原则 | 跳过决策画像和约束，保留核心安全规则 |
| 安静模式 | 完整记忆但输出极致精简 |
