# 招投标术语识别指南

本文件帮助识别招标文件中的关键术语和表述，用于第 3 步（解析招标指标）和第 4 步（评定）。

## 1. 要求级别识别：必须 / 应 / 宜

国内招标文件常用的三级要求措辞，**级别不同直接影响废标风险**：

| 级别 | 典型措辞 | 实际含义 | 不满足的后果 |
|---|---|---|---|
| **必须**（强制） | 必须、应当、需、要、确保、不得 | 硬性门槛 | 直接判为不实质响应，可能废标 |
| **应**（推荐） | 应、宜、建议、最好 | 强烈期望但非硬性 | 扣分，但通常不废标 |
| **宜**（可选） | 可、支持、能够、具备 | 一般性要求 | 小幅扣分或仅影响技术分排序 |

**识别规则**：
- 不是看单个字，要看**句式和上下文**。"系统应支持 xxx" 在正式招标文件里通常就是强制要求
- 见到 "必须"、"不得"、"不低于"、"至少" 这类词，按强制处理
- 在"★"标记项或"核心指标"章节下，即使没强制措辞，也按必须处理
- 模糊措辞（"最好"、"建议"、"优先"）按一般项处理

**在评估表"要求级别"列填什么**：统一用 `必须` / `应` / `宜` 三种标签（不用"强制"/"推荐"等同义词，保持一致）。

## 2. 重要标记：★ 和 ▲

招标文件会在重要指标旁加特殊标记，**务必识别**：

| 标记 | 常见含义 | 处理 |
|---|---|---|
| **★** | 关键指标 / 否决项 / 核心技术要求 | 按"必须"级别处理；单独统计满足率 |
| **▲** | 重要指标 / 加分项 / 重点关注 | 按"应"级别处理；单独统计 |
| **●** 或 **◆** | 各地差异，通常也是重要标记 | 在澄清任务阶段让用户确认含义 |
| 加粗 / 红字 | 非标准但常见 | 建议按"应"级别处理，在指标清单里标注 |

**注意**：
- 招标文件通常在前言或总则里定义 ★/▲ 的含义（比如"★ 为核心条款，不满足视为废标"），**扫描指标前先读一下总则确认定义**
- 有些招标文件用 "★" 表示加分项而非否决项，用法与上表相反 —— 一定要以招标文件的自定义为准
- 如果找不到定义，按上表默认理解，并在"第 1 步澄清任务"时跟用户确认

## 3. 偏离度与响应度

应答术语，写报告的"应答策略"部分时会用到：

| 术语 | 含义 |
|---|---|
| **正偏离** | 应答优于招标要求（如要求 500、应答 1000） |
| **无偏离** | 应答恰好等于招标要求 |
| **负偏离** | 应答低于招标要求（如要求 1000、应答 800） |
| **完全响应** | 所有指标均满足或优于要求（全部正偏离或无偏离） |
| **部分响应** | 部分指标满足、部分负偏离 |
| **不响应** | 未应答或明确声明不满足 |
| **实质性响应** | 对招标文件规定的"必须"条款全部满足（如缺失任一必须项，视为不实质响应） |
| **响应度** | 满足指标数 / 总指标数，常用百分比 |

**评定等级到应答措辞的映射**（便于在应答策略里用对词）：

| 评估等级 | 典型应答措辞 |
|---|---|
| 完全满足 | "完全满足"、"正偏离"、"优于要求" |
| 基本满足 | "满足"、"无偏离" |
| 部分满足 | "部分满足"、"负偏离，拟通过 xxx 补足" |
| 不满足 | "负偏离"、"不响应"（需决策是否仍应答） |

## 4. 指标分类

第 3 步"解析招标指标"时，要给每条指标打上类别标签，方便后续检索和统计。默认用下列 7 类：

| 类别 | 典型内容 |
|---|---|
| **功能** | 具体功能点、模块能力（防护、审计、识别、分析等） |
| **性能** | 量化指标：吞吐量、并发、时延、容量、响应时间、TPS/QPS 等 |
| **资质** | 认证证书：等保测评、商密认证、CCRC、3C、ISO 等；专利、软著 |
| **部署 / 兼容** | 操作系统、国产化适配、虚拟化、云环境、硬件平台 |
| **安全** | 产品自身安全性（权限、审计、加密、漏洞管理） |
| **服务** | 驻场、售后、培训、SLA、质保、升级服务 |
| **其他** | 不属于以上的（案例经验、本地化、商务条款等） |

类别不唯一时按**主要属性**判断，不要一条指标打多个类别（会让统计表混乱）。

## 5. 易混淆的招标措辞

下面这些表述在评定时容易看走眼，**见到要特别警觉**：

| 招标原文措辞 | 容易误判的理解 | 正确理解 |
|---|---|---|
| "具有提供 xxx 的能力" | 要求实际提供 xxx | 只要求"有能力"，不一定需要当前就有（但证据要能证明能力） |
| "支持不少于 3 种" | 有 3 种就行 | 至少 3 种，多多益善（4 种、5 种更好） |
| "不低于" / "≥" | 严格相等即满足 | 必须大于或等于，**略低就是不满足** |
| "优于" / ">" | 相等即满足 | 必须**严格大于**（有的招标要求会用这个区分） |
| "具有 xxx 证书 / 资质" | 只要有相关能力 | 必须有证书**实物**，不能用"支持 xxx 标准"等价 |
| "本地化" | 在中国提供 | 可能指"属地化服务"、"国产化适配"，需上下文判断 |
| "原厂" | 随便哪个厂商 | 特指**产品生产厂商**本身，不是集成商或代理商 |
| "7×24 响应" | 24 小时响应 | 一周 7 天、每天 24 小时均可响应（通常指服务值班）|

遇到这些措辞时，在"关键要点"里用 🔍 标注，并在评定时严格按原意处理。

## 6. 招标文件结构速览

国内常见技术招标文件的章节结构（帮助定位"技术指标要求"在哪一章）：

```
第一章 招标公告 / 投标邀请
第二章 投标人须知
第三章 评标办法（重要：看清技术分/商务分权重）
第四章 合同条款
第五章 采购需求（= 技术需求 / 技术规格书 / 技术参数）  ← 主要评估对象
第六章 投标文件格式
```

**第五章**通常细分为：
- 项目背景 / 建设目标
- 总体要求（总则级别的宏观要求）
- **功能性能要求**（最重要，绝大多数指标在这里）
- **资质和业绩要求**
- 服务要求
- 商务要求

评估时以"功能性能要求"和"资质和业绩要求"为主；"总体要求"里的条款通常比较宏观，也要扫一遍看有无隐含指标。

## 7. 评分办法速览（辅助定位重要指标）

第三章"评标办法"里通常会给出评分细则，指明哪些指标是**否决项**、哪些是**打分项**：

- 否决项（废标项）不满足 → 直接废标，务必识别
- 分值高的指标 → 即使没 ★ 标记，也应视为重要指标，评估时更严格
- 分值低的指标 → 加分项，负偏离风险小

**评估前建议**：如果用户提供了完整招标文件，**先扫一遍评标办法**，把否决项和高分项列出来，在结构化清单里额外标一列"评分权重"（如果有数据）。这能让"第三章：重大风险清单"做得更准。
